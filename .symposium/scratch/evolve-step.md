# Evolve-step — Cycle 1

## Seed v1
```yaml
goal: 전체 서비스를 로컬에서 실행해 핵심 정산 시나리오를 종단 간 검증하고, 발견된 코드·테스트·API·문서 불일치를 개선하여 실제 품질과 포트폴리오 전달력을 높인다.
constraints:
  - 포트폴리오 표현보다 실제 프로젝트 품질을 우선한다.
  - 전체 서비스의 로컬 실행 가능성을 검증 범위에 포함한다.
  - 구현, 테스트, API 명세, README 및 아키텍처 문서의 일관성을 확인한다.
acceptance_criteria:
  - 전체 서비스가 로컬에서 구동된다.
  - 핵심 정산 시나리오가 종단 간으로 완료된다.
  - 코드, 테스트, API 및 문서 사이에서 발견된 불일치가 해소되고 검증된다.
  - 검증된 개선 결과가 포트폴리오에서 명확히 드러난다.
```

## Blind Spot Candidates
### Q1. Missing user perspectives
- accepted — 핵심 시나리오의 정산 결과가 비즈니스 규칙과 일치하는지 검증
- accepted — 로그·메트릭·추적·장애 복구로 실패 원인을 진단할 수 있는지 검증
- accepted — 신규 개발자가 문서만으로 환경 구성, 실행, 테스트를 재현할 수 있는지 검증

### Q2. Cut-off possibilities
- accepted — Docker Compose와 Kubernetes 배포 환경에서도 실행 가능성 확인
- accepted — 중복 이벤트, 재시도, 부분 실패, 환불·조정 예외 흐름 검증
- accepted — 이번 점검을 CI에서 반복 가능한 자동 검증으로 전환
