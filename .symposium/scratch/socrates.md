## Cycle 1
### Wonder

- 포트폴리오 전달력 — 면접관에게 설계 역량과 구현 완성도가 명확히 보이는지 점검
- 실행 완전성 — 로컬에서 전체 서비스가 구동되고 핵심 시나리오가 실제 동작하는지 점검
- 산출물 일관성 — 구현, 테스트, API, README 및 아키텍처 문서가 서로 일치하는지 점검

### Reflect

- 포트폴리오 전달력
  - User's intended nuance: 채용 관점도 포함하지만 실제 프로젝트 품질이 우선이다.
  - Model's interpreted nuance: 면접관에게 설계 역량과 구현 완성도가 명확히 보여야 한다.
  - Same / different: 일부 동일하나 우선순위가 다름
- 실행 완전성
  - User's intended nuance: 로컬에서 전체 서비스가 구동되고 핵심 정산 시나리오가 끝까지 동작해야 한다.
  - Model's interpreted nuance: 로컬에서 전체 서비스가 구동되고 핵심 정산 시나리오가 실제로 끝까지 동작해야 한다.
  - Same / different: 동일
- 산출물 일관성
  - User's intended nuance: 구현, 테스트, API 명세, README, 아키텍처 문서가 현재 코드와 서로 일치해야 한다.
  - Model's interpreted nuance: 구현, 테스트, API 명세, README, 아키텍처 문서가 현재 코드와 서로 일치해야 한다.
  - Same / different: 동일

### Refine

shared meaning: 실제 품질을 최우선으로 전체 서비스를 실행 검증하고, 코드·테스트·문서의 불일치를 해소한 뒤 그 결과가 포트폴리오에도 드러나게 한다.

### Restate

goal: 전체 서비스를 로컬에서 실행해 핵심 정산 시나리오를 종단 간 검증하고, 발견된 코드·테스트·API·문서 불일치를 개선하여 실제 품질과 포트폴리오 전달력을 높인다.
