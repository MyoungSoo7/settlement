# Engagement Retro — 회고와 환류 (사이클의 관문)

회고는 감상문이 아니라 **진단 프레임워크 자체를 개선하는 산출물**이다.
이 플러그인의 에셋 보관함은 코드다: 임계값 기본값·프리셋(rationale)·판별 테스트 힌트·채점기 규칙.

## 절차

1. 사이클 기록(브리핑·이행 노트·델타 파일)만 근거로 `<엔게이지먼트폴더>/retro-cycle-N.md` 작성:

```markdown
# {고객} cycle {N} 회고
- 신호 품질: 오탐(발화했는데 실체 없음) / 미탐(델타 신규가 전 사이클에 조짐 있었나) 사례
- 권고 품질: blocked/미이행 액션과 그 원인 — 조치가 실행 가능하게 쓰였는가
- 환류 실행: (아래 2번의 실제 반영 내역 링크 — 반영 없이 "환류했다"고 쓰지 않는다)
- 다음 사이클에 바꿀 것 딱 1가지
```

2. **환류 실행** — 해당되는 것만, 반드시 실제 반영으로:
   - 임계값 과민/둔감 → `common/presets/` 프리셋 값 조정 제안 (**rationale 필수** — 테스트가 강제)
     + `bin/calibrate.mjs` 로 발화율 재측정해 근거 첨부.
   - 판별 테스트가 실행 불가능했음 → 해당 신호의 `checkHints` 문구 개선 제안.
   - 채점기가 놓친 패턴 → briefing-eval 회귀 테스트 케이스 추가 제안.
   - 코드 변경은 제안 + 검증(테스트)까지. **pwc 폴더는 커밋하지 않는다** (사용자 방침).
3. 완료 보고: 회고 파일 경로 + 환류 내역. 관리자가 이 경로를 next-cycle 의 `--retro` 로 쓴다.

## 경계

cycle 증가는 관리자의 `advance --to next-cycle` 만 한다. 새 브리핑 생성도 이 스킬 밖의 일.
