# CEO 브리핑 — 서명용 보고 형식

분석이 아니라 **전달**의 스킬이다. 아래 형식과 표현 규칙을 강제한다.

## 리스크 1건당 형식

```
### [순위]. [리스크 한 줄 제목 — 무엇이 얼마나 위험한가]

결론: (한 문장. 예: "제품C는 배부 기준을 자원 소비에 맞추면 분기 적자 전환 — 연 환산 약 8~11억 손실이 가려져 있습니다.")

증거: (숫자 최대 3개, 각각 출처 명시)
  - ...
  - ...

왜 문제인가: (인과 사슬을 자연어로 3~5문장. 숫자 나열 금지 — 숫자는 위 증거로 끝났다.)

확신도: [확인됨 | 가설] — 가설이면 확인에 필요한 데이터/절차를 명시
  (예: "가설 — A사 채권의 담보 설정 내역 확인 필요 (재무팀, 1일)")

권고 조치: (조치 + 담당 부서 + 기한. 회사의 기존 결재 라인(SOP)에 맞는 단위로 쪼갠다.)

서명: 검토 ______ / 승인 ______   ← 에이전트는 여기까지만. 실행은 승인 뒤 사람의 일.
```

## 표현 규칙

- **단정 대신 근거-결론**: "분식회계입니다" ❌ → "수익 인식 시점이 정책과 다를 가능성이 있으며 근거는 3가지입니다" ✅
- **금액은 범위로**: 점 추정 대신 하한~상한 + 산정 방식.
- **전문용어는 풀어서**: DSO ❌ → "매출이 현금으로 돌아오는 데 걸리는 날수(DSO)" ✅. 첫 등장에서 한 번 풀고 이후 약어 허용.
- **좋은 소식도 보고**: 검사했지만 문제없던 영역을 끝에 한 줄로 — "확인 범위" 절. 무엇을 안 봤는지도 명시(범위 밖 데이터).
- **분량**: 리스크당 반 페이지 이내. 상세 근거는 부록(break report 등)으로 분리하고 본문에서 참조.

## 문서 전체 구조

1. 헤드라인 요약 (리스크 3건 이내, 각 한 줄)
2. 리스크 상세 (위 형식 반복, 중요도 순)
3. 확인 범위와 한계
4. 부록 참조 목록
