# Company Identity — 기업 식별 입력 게이트

기업 분석은 **대상 식별**이 먼저다. 기업명이 비슷하거나 사업자번호가 틀리면 DART, 뉴스, 재무제표가
서로 다른 회사를 가리킬 수 있다. 이 스킬은 분석 전에 기업명과 사업자등록번호를 검증하고,
이후 도구가 사용할 식별자를 정리한다.

## 필수 입력 (특정 기업 CEO 컨설팅)

- `company_name`: 분석 대상 기업명
- `business_number`: 사업자등록번호

## 선택 입력

- `representative_name`: 대표자명. 국세청 진위확인에 사용.
- `opening_date`: 개업일자 YYYYMMDD. 국세청 진위확인에 사용.
- `stock_code`: 상장사 종목코드 6자리. DART 검색 보조.
- `dart_corp_code`: DART 고유번호 8자리. 이미 아는 경우 사용.
- `news_query_name`: 네이버 뉴스 검색 별칭. 법인명과 브랜드명이 다를 때 사용.

## 워크플로우

1. `company_identity_gate` 를 우선 호출한다. 이 도구는 아래 2~4단계를 한 번에 수행하고 분석 진입 여부를 반환한다.
2. `business_number_validate` 로 사업자등록번호 10자리 형식과 체크섬을 먼저 확인한다.
   - 실패하면 DART/뉴스/재무 분석에 들어가지 말고 입력 정정 요청 또는 데이터 품질 리스크로 보고한다.
3. `business_status_check` 로 국세청 사업자등록 상태를 확인한다.
   - 계속사업자가 아니거나 미등록이면 분석 결과에 식별 리스크를 최상단에 표시한다.
4. 대표자명과 개업일자가 있으면 `business_auth_check` 로 진위확인을 수행한다.
   - 진위확인 실패는 단정하지 말고 "입력값 불일치 또는 국세청 등록정보 불일치"로 표현한다.
5. 이후 분석은 정리된 식별자를 기준으로 수행한다:
   - DART: `dart_corp_search` 또는 `dart_corp_code`
   - 뉴스: `news_query_name`
   - 내부 CSV: 해당 법인 데이터인지 확인

## 산출물

```text
기업 식별 게이트:
- 기업명:
- 사업자등록번호: 형식/체크섬 통과 여부
- 국세청 상태: 계속/휴업/폐업/미등록/미확인
- 진위확인: 확인됨/불일치/입력 부족
- DART 식별자:
- 뉴스 검색명:
- 분석 진입 여부:
```

## 표현 규칙

- "가짜 사업자"처럼 단정하지 않는다.
- 체크섬 실패는 "사업자등록번호 형식 오류"로 쓴다.
- 국세청 API 키가 없거나 호출 실패 시 "외부 상태 미확인"으로 표시하고, 확인 필요 항목으로 남긴다.
- 휴업/폐업/미등록이면 재무·뉴스 분석보다 식별 리스크를 먼저 보고한다.
- 특정 기업 CEO 컨설팅에서 `company_name` 또는 `business_number` 가 없으면 후속 분석을 시작하지 말고 입력값 보완을 요청한다.
