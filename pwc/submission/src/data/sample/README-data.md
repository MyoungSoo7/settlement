# 샘플 데이터 설명 (합성 데이터, 단위: 백만원)

가상 제조·유통사 "(주)한빛커머스" 의 2026Q2 기준 데이터. 세 파일은 서로 교차 검증되도록 설계됨.

| 파일 | 내용 | 기간 |
|------|------|------|
| `trial_balance.csv` | 분기별 시산표 요약 (매출·채권·재고·선수금·영업이익·영업현금흐름·**변동금리 차입금·이자비용**) | 2025Q3~2026Q2 |
| `ar_aging.csv` | 분기×거래처별 매출채권 연령분석 — 각 분기 합계는 시산표의 해당 분기 매출채권과 일치 (2025Q4=3,400 / 2026Q1=4,900 / 2026Q2=7,200) | 2025Q4~2026Q2 (3개 분기 스냅숏) |
| `cost_allocation.csv` | 제품별 원가·공통원가 배부 — 매출 합계는 시산표의 2026Q2 매출(15,800)과 일치. 공통원가 총 3,000 을 매출 비중으로 배부 중 | 2026Q2 |

aging 을 3개 분기로 제공하는 이유: `cashflow-bottleneck` 스킬의 판별 테스트
("특정 거래처 집중인가, 전반적 드리프트인가 — 추세로 확인")를 데모 데이터로 실제 실행할 수 있게 하기 위함.
`variable_rate_debt`/`interest_expense` 컬럼은 `macro-exposure` 스킬(금리 노출 분석)의 입력이다.

정합성은 불변식 게이트로 기계 검증한다: `node src/bin/verify-books.mjs`

⚠️ 이 데이터에는 의도적으로 심어둔 이상 신호 4개가 있다 (정답지는 `submission/README.md` 의 데모 섹션 참조).
데모 목적: 에이전트가 "이쁜 대시보드"가 아니라 행간의 문제를 추론으로 찾아내는지 검증.
포착 여부 자동 채점: `node src/test/briefing-eval.mjs <생성된_브리핑.md>`
