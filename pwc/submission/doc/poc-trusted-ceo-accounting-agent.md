# Trusted CEO Accounting Agent PoC

## 1. 문제 정의

기업의 회계 데이터는 이미 ERP, 엑셀, 회계 시스템 안에 존재하지만 경영진이 실제로 놓치는 문제는 숫자가 없어서가 아니라 **숫자 사이의 관계를 늦게 읽기 때문**에 발생한다. 매출, 채권, 현금흐름, 원가 배부 데이터는 각각 따로 보면 정상처럼 보일 수 있지만, 서로 교차해서 보면 수익 인식, 회수 가능성, 제품 수익성 왜곡 같은 중요한 리스크가 드러난다.

이 PoC의 목표는 기업 내부의 정형 회계 데이터를 기반으로 CEO가 아직 질문하지 않은 리스크를 먼저 찾아내는 **Trusted CEO Accounting Agent**를 제안하고 검증하는 것이다.

Agent가 해결하려는 핵심 문제는 세 가지다.

| 문제 | 설명 | 경영 리스크 |
|---|---|---|
| 회계처리 이상 징후 | 매출, 채권, 계약부채, 현금흐름이 서로 맞지 않는 패턴 | 수익 조기 인식, cut-off 오류, 회계 추정 오류 |
| 현금흐름 병목 | 이익은 나지만 현금이 들어오지 않는 구조 | 운전자본 악화, 특정 거래처 신용 리스크 |
| 원가 배분 왜곡 | 공통원가 배부 기준이 실제 자원 사용량을 반영하지 못함 | 손실 제품을 이익 제품으로 오판 |

PoC의 핵심 가설은 다음과 같다.

> 기업 회계 Agent의 가치는 재무 데이터를 요약하는 데 있지 않고, 서로 다른 회계 데이터의 관계에서 경영진이 놓친 리스크를 근거와 함께 설명하는 데 있다.

## 2. 왜 기존 회계 보고로는 놓치는가

기존 회계 보고는 보통 보고서 단위로 분리되어 있다. 손익계산서는 매출과 이익을 보여주고, 재무상태표는 채권과 재고를 보여주며, 현금흐름표는 현금 유입과 유출을 보여준다. 각 보고서는 필요하지만, 보고서 하나만으로는 기업의 실제 위험을 충분히 설명하지 못한다.

예를 들어 매출과 영업이익이 증가하면 겉으로는 성과가 좋아 보인다. 하지만 같은 기간 매출채권이 더 빠르게 증가하고 영업현금흐름이 악화된다면, 그 매출은 아직 현금으로 회수되지 않은 매출일 수 있다. 이 경우 CEO에게 필요한 질문은 "매출이 얼마인가?"가 아니라 "그 매출이 실제 현금으로 전환되고 있는가?"이다.

또한 제품별 손익도 원가 배부 기준에 따라 전혀 다르게 보일 수 있다. 매출 비중으로 공통원가를 배부하면 판매 규모가 작은 제품은 비용을 적게 부담한다. 그러나 실제 생산시간이나 기계 사용량이 높은 제품이라면, 현재 손익은 실제 자원 소모를 과소 반영하고 있을 수 있다.

기존 보고가 놓치는 지점은 다음과 같다.

| 기존 보고 방식 | 놓치는 위험 |
|---|---|
| 매출 증가율 중심 보고 | 회수되지 않는 매출, 조기 인식 가능성 |
| 총액 기준 채권 보고 | 특정 거래처에 집중된 장기 미수 |
| 매출 비중 기준 원가 배부 | 실제 자원 사용량과 다른 제품 손익 |
| 표 중심 보고 | 원인 가설, 확인 절차, 책임 부서가 분리되지 않음 |

Trusted CEO Accounting Agent는 단일 보고서를 더 예쁘게 만드는 시스템이 아니다. 시산표, 채권 aging, 원가 배부 데이터를 서로 대조하여 "좋아 보이는 숫자 뒤에 무엇이 숨어 있는가"를 먼저 묻는 시스템이다.

## 3. Agent 아키텍처

PoC는 실제 ERP 연동 대신 샘플 CSV를 사용한다. 단, 구조는 향후 ERP, DW, 감사 시스템, MCP 서버로 확장 가능하도록 설계한다.

```text
사용자 요청
  예: "우리 회사 데이터에서 CEO가 놓친 리스크를 찾아줘"
        |
        v
CEO Risk Recon Agent
  - 데이터 인벤토리 확인
  - 분석 대상 파일과 grain 파악
  - 전문 Agent 실행 순서 결정
        |
        v
전문 분석 Agent
  1. Accounting Anomaly Agent
  2. Cashflow Bottleneck Agent
  3. Cost Allocation Audit Agent
        |
        v
교차 검증 Layer
  - 여러 데이터에서 같은 방향의 신호가 나오는지 확인
  - 금액 영향, 확신도, 시급성 기준으로 우선순위 산정
        |
        v
CEO Briefing Agent
  - 결론, 근거, 왜 문제인지, 확인 절차, 권고 조치로 정리
```

각 Agent의 책임은 명확히 분리한다.

| Agent | 입력 | 역할 | 출력 |
|---|---|---|---|
| CEO Risk Recon Agent | 전체 데이터 목록 | 분석 흐름 통제, 교차 검증 | Top risk 후보 |
| Accounting Anomaly Agent | 시산표, 기간별 계정 요약 | 회계처리 이상 징후 탐지 | break report |
| Cashflow Bottleneck Agent | 시산표, 채권 aging | 현금흐름 병목과 거래처 집중 탐지 | cashflow risk report |
| Cost Allocation Audit Agent | 제품별 원가 배부 데이터 | 배부 기준 왜곡 탐지 | allocation scenario report |
| CEO Briefing Agent | 분석 결과 전체 | 경영진 보고서 작성 | CEO briefing |

PoC에서 사용하는 샘플 데이터는 다음과 같다.

| 파일 | 내용 | 검증 목적 |
|---|---|---|
| `src/data/sample/trial_balance.csv` | 분기별 매출, 매출채권, 재고, 계약부채, 영업이익, 영업현금흐름 | 손익과 현금흐름 괴리 탐지 |
| `src/data/sample/ar_aging.csv` | 거래처별 채권 aging | 특정 거래처 장기 미수 집중 탐지 |
| `src/data/sample/cost_allocation.csv` | 제품별 매출, 직접원가, 배부 공통원가, 배부 기준, 실제 기계시간 | 원가 배부 왜곡 탐지 |

## 4. 샘플 데이터 기반 탐지 시나리오 3개

### 시나리오 1. 수익 인식 이상 가능성

`trial_balance.csv` 기준으로 2026Q1에서 2026Q2로 넘어가며 다음 변화가 발생한다.

| 지표 | 2026Q1 | 2026Q2 | 변화 |
|---|---:|---:|---:|
| 매출 | 14,200 | 15,800 | +11.3% |
| 매출채권 | 4,900 | 7,200 | +46.9% |
| 계약부채 | 340 | 180 | -47.1% |
| 영업이익 | 2,520 | 2,840 | +12.7% |
| 영업현금흐름 | 880 | -310 | 급락 |

매출과 영업이익은 증가했지만 매출채권은 매출보다 훨씬 빠르게 증가했고, 계약부채는 감소했으며, 영업현금흐름은 음수로 전환되었다. 이 조합은 단순한 성장보다 **현금으로 회수되지 않은 매출 증가** 또는 **수익 인식 시점 확인 필요성**을 시사한다.

Agent는 이를 단정하지 않고 다음과 같이 표현한다.

> 수익 조기 인식 또는 cut-off 이슈 가능성이 있습니다. 매출은 11.3% 증가했지만 매출채권은 46.9% 증가했고, 영업현금흐름은 880에서 -310으로 악화되었습니다. 기말 전후 매출 전표, 계약 진행률, 청구 조건 확인이 필요합니다.

### 시나리오 2. 현금흐름 병목과 거래처 집중 리스크

`ar_aging.csv` 기준 2026Q2 매출채권 합계는 7,200으로 시산표의 매출채권과 일치한다. 이 중 90일 초과 채권은 1,920이다.

| 항목 | 금액 | 비율 |
|---|---:|---:|
| 총 매출채권 | 7,200 | 100.0% |
| 90일 초과 채권 | 1,920 | 26.7% |
| 에이프릴리테일 90일 초과 채권 | 1,750 | 90일 초과 채권의 91.1% |

문제는 장기 미수의 대부분이 특정 거래처 하나에 집중되어 있다는 점이다. 전사적인 청구 프로세스 문제일 수도 있지만, 샘플 데이터에서는 특정 고객 신용 리스크 또는 담보·회수 조건 문제일 가능성이 더 크다.

Agent는 다음과 같이 보고한다.

> 현금흐름 병목은 전사 전체에 고르게 퍼진 문제가 아니라 특정 거래처에 집중된 리스크일 가능성이 큽니다. 90일 초과 채권 1,920 중 1,750이 에이프릴리테일에 집중되어 있어, 해당 거래처의 신용 상태, 담보, 회수 계획을 우선 확인해야 합니다.

### 시나리오 3. 원가 배분 왜곡으로 인한 보이지 않는 손실

`cost_allocation.csv` 기준 제품 C는 현재 매출 비중 15%를 기준으로 공통원가 450을 배부받고 있으며, 영업이익은 20으로 표시된다.

하지만 실제 기계시간 사용 비중은 45%다. 공통원가 총액 3,000을 실제 기계시간 기준으로 재배부하면 제품 C가 부담해야 할 공통원가는 1,350이다.

| 제품 | 현재 배부 공통원가 | 실제 기계시간 기준 공통원가 | 손익 영향 |
|---|---:|---:|---:|
| 제품 C | 450 | 1,350 | -900 |

제품 C의 현재 영업이익은 20이므로, 실제 자원 사용량 기준으로 보면 영업손익은 20 - 900 = **-880**으로 전환된다.

Agent는 다음과 같이 보고한다.

> 제품 C는 현재 손익상 소폭 흑자로 보이지만, 실제 기계시간 기준으로 공통원가를 배부하면 약 880의 손실 제품으로 전환됩니다. 이는 제품 가격, 생산 방식, 공통원가 배부 기준을 재검토해야 하는 신호입니다.

# CEO Briefing: 2026Q2 회계 리스크 진단

## Headline Summary

이번 분기 데이터에서 CEO가 우선 확인해야 할 리스크는 세 가지입니다.

1. 매출과 이익은 증가했지만 현금흐름이 악화되어 수익 인식 시점 또는 회수 가능성 확인이 필요합니다.
2. 90일 초과 채권이 특정 거래처에 집중되어 단일 고객 신용 리스크가 큽니다.
3. 제품 C는 현재 흑자로 보이나 실제 자원 사용량 기준으로는 손실 제품일 가능성이 있습니다.

## 1. 매출은 늘었지만 현금이 들어오지 않는 구조

**결론**  
2026Q2 매출과 영업이익은 증가했지만, 영업현금흐름은 음수로 전환되었습니다. 수익 인식 시점과 기말 전후 매출 처리를 확인해야 합니다.

**근거**

- 매출: 14,200에서 15,800으로 11.3% 증가
- 매출채권: 4,900에서 7,200으로 46.9% 증가
- 영업현금흐름: 880에서 -310으로 악화
- 계약부채: 340에서 180으로 감소

**왜 문제인가**  
매출이 실제 현금으로 전환되지 않으면 손익계산서상 성장은 기업의 지급 능력 개선으로 이어지지 않습니다. 특히 매출채권 급증과 현금흐름 악화가 동시에 발생하면 조기 수익 인식, 회수 지연, 계약 조건 변경 가능성을 함께 확인해야 합니다.

**확신도**  
가능성 높음. 단, 최종 판단을 위해 계약서, 진행률 산정표, 기말 전후 전표 확인이 필요합니다.

**권고 조치**  
재무팀과 영업관리팀이 1주 내로 2026Q2 말 전후 대형 매출 전표, 청구일, 검수일, 계약 진행률을 대조해야 합니다.

## 2. 장기 미수금이 특정 거래처에 집중

**결론**  
90일 초과 채권의 91.1%가 에이프릴리테일 한 곳에 집중되어 있습니다. 전사 평균 문제가 아니라 특정 거래처 신용 리스크일 가능성이 큽니다.

**근거**

- 총 매출채권: 7,200
- 90일 초과 채권: 1,920
- 에이프릴리테일 90일 초과 채권: 1,750
- 에이프릴리테일이 90일 초과 채권에서 차지하는 비중: 91.1%

**왜 문제인가**  
장기 미수금이 특정 거래처에 집중되면 해당 거래처의 지급 지연만으로도 회사 전체 현금흐름이 흔들릴 수 있습니다. 이 경우 일반적인 독촉 프로세스보다 신용 한도, 담보, 거래 조건 재검토가 우선입니다.

**확신도**  
확인됨. 채권 aging과 시산표 매출채권 합계가 일치합니다.

**권고 조치**  
영업관리팀과 재무팀이 해당 거래처의 미수 사유, 담보 설정, 회수 계획, 추가 출고 제한 여부를 즉시 검토해야 합니다.

## 3. 제품 C의 보이지 않는 손실

**결론**  
제품 C는 현재 공통원가 배부 기준에서는 20의 이익으로 보이지만, 실제 기계시간 기준으로 재배부하면 약 880의 손실 제품으로 전환됩니다.

**근거**

- 제품 C 매출 비중: 15%
- 제품 C 실제 기계시간 비중: 45%
- 현재 배부 공통원가: 450
- 실제 기계시간 기준 공통원가: 1,350
- 손익 영향: -900

**왜 문제인가**  
제품 C가 실제보다 수익성 높게 보이면 가격 정책, 생산 우선순위, 설비 투자 판단이 왜곡됩니다. 회사는 손실 제품을 성장 제품으로 착각할 수 있습니다.

**확신도**  
가능성 높음. 단, 기계시간 외에 인력시간, 설비 점유율, 불량률 데이터까지 확인하면 더 정확한 배부 기준을 설계할 수 있습니다.

**권고 조치**  
원가관리팀이 제품 C의 표준원가, 실제 기계시간, 생산 batch 크기, 불량률을 검토하고 공통원가 배부 기준을 재산정해야 합니다.

## 6. 기대 효과

Trusted CEO Accounting Agent의 기대 효과는 단순 자동화가 아니라 **회계적 판단의 품질 향상**이다.

| 기대 효과 | 설명 |
|---|---|
| 리스크 조기 발견 | 월말·분기말 보고 이후가 아니라 데이터 입력 직후 이상 징후 탐지 |
| 의사결정 속도 향상 | CEO/CFO가 바로 확인할 수 있는 결론, 근거, 조치로 요약 |
| 감사 대응력 강화 | 분석 근거와 확인 절차가 남아 감사 trail로 활용 가능 |
| 현금흐름 관리 개선 | 매출 성장과 회수 가능성을 함께 관리 |
| 제품 수익성 개선 | 원가 배부 왜곡을 찾아 가격·생산 전략 재검토 |
| 설명 가능한 AI | 숫자, 원인 가설, 추가 확인 데이터를 분리해 제시 |

일반적인 회계 자동화는 반복 입력, 전표 처리, 보고서 생성을 줄이는 데 집중한다. 이 PoC는 그보다 앞단에서 **"무엇을 확인해야 하는가"**를 먼저 제안한다. 따라서 회계 담당자에게는 검토 우선순위를 제공하고, CEO에게는 경영 리스크를 설명 가능한 언어로 전달한다.

## 7. 확장 방향

PoC는 CSV 기반이지만 실무 시스템으로 확장할 때는 다음 단계로 발전시킬 수 있다.

### 7.1 ERP/DW/MCP 연동

현재는 `trial_balance.csv`, `ar_aging.csv`, `cost_allocation.csv`를 사용하지만, 실제 환경에서는 ERP, 회계 DB, DW, 전표 시스템, 계약 관리 시스템과 연결한다. Agent는 직접 DB를 수정하지 않고 읽기 전용 조회 도구를 통해 데이터를 수집한다.

### 7.2 감사로그와 근거 보존

Agent가 어떤 데이터로 어떤 결론을 냈는지 저장한다. 보고서에는 다음 정보가 포함되어야 한다.

- 사용한 데이터 원천
- 기준 기간
- 계산식
- 탐지 규칙
- 원인 가설
- 사람이 확인해야 할 추가 데이터

### 7.3 승인 워크플로

Agent의 결과는 자동 조치가 아니라 사람의 검토를 거친다.

```text
Agent 탐지
  -> 회계 담당자 검토
  -> 재무팀장 승인
  -> CFO/CEO 보고
  -> 조치 결과 기록
```

### 7.4 실시간 모니터링

분기말 보고서 생성 이후에만 분석하는 것이 아니라, 전표 입력, 매출 인식, 채권 aging 업데이트, 원가 배부 실행 시점마다 이상 신호를 감지한다.

예시 알림은 다음과 같다.

> 이번 주 신규 매출 중 특정 거래처의 미청구/미회수 비중이 과거 평균을 초과했습니다. 기말 매출 인식 전 계약 조건 확인이 필요합니다.

### 7.5 감사·컨설팅 서비스화

회계법인 관점에서는 이 Agent를 고객사 진단 도구로 활용할 수 있다.

| 활용 영역 | 예시 |
|---|---|
| 재무 진단 | 손익과 현금흐름 괴리 분석 |
| 내부통제 진단 | 반복적인 cut-off 오류, 승인 누락 탐지 |
| 원가 컨설팅 | 제품·사업부 수익성 재산정 |
| 감사 사전 점검 | 고위험 계정과 거래처 우선순위 도출 |

## PoC 결론

Trusted CEO Accounting Agent는 회계 데이터를 단순 요약하는 챗봇이 아니라, 기업 내부 숫자의 관계를 읽고 CEO가 놓치기 쉬운 리스크를 먼저 제기하는 분석형 Agent다. 이 PoC는 세 개의 샘플 데이터만으로도 수익 인식 이상 가능성, 현금흐름 병목, 원가 배분 왜곡이라는 기업의 실제 의사결정 문제를 탐지할 수 있음을 보여준다.

최종 메시지는 다음 한 문장으로 요약할 수 있다.

> Trusted CEO Accounting Agent는 기업의 회계 데이터를 교차 분석해 CEO가 놓치기 쉬운 수익 인식, 현금흐름, 원가 왜곡 리스크를 선제적으로 탐지하고, 근거 기반의 설명 가능한 의사결정 보고서를 생성하는 AI Agent다.
