argument-hint: "[상품 ID 또는 조사 범위, 생략 시 코드 레벨 점검만]"

`review-integrity` skill 을 로드하고(skill 미지원 환경이면
`fashion-copilot/skills/review-integrity/SKILL.md` 를 직접 읽어라), 리뷰 스크리닝을 수행하라.
대상: $ARGUMENTS (비어 있으면 코드 레벨 무결성 점검만).

1. **코드 무결성**: `UNIQUE(user_id, product_id)` 하드 제약 존재, rating 1~5 도메인 검증,
   수정/삭제 본인 확인이 유지되는지 코드에서 확인하라. 소프트 체크만 믿는 신규 코드는 위반.
2. **구매 검증 공백 안내**: reviews↔orders 링크가 없음을 전제로, 어뷰징 조사는 skill 의
   패턴 카탈로그(비구매 리뷰/집중 작성/rating 편중/복붙 본문) 순서로 교차조회 절차를 제시하라.
   조회는 읽기 전용 SELECT 만 — 운영 DB 쓰기 명령은 절대 생성하지 마라.
3. 대상 상품이 지정되면: 해당 상품의 리뷰 분포(시간창·rating·작성자 가입 시점) 조사 쿼리를
   제안하고, 구매 이력 교차조회로 비구매 리뷰 비율을 산출하는 절차를 안내하라.

보고: 결론 한 줄 → 무결성 점검 결과 → 어뷰징 시그널 유무와 근거 → 권고 조치
(구매 검증 도입은 설계서 §8 Phase 2 로드맵을 인용해 제안).
