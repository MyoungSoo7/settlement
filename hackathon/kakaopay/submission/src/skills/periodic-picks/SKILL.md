# 주기별 종목 선별 (Periodic Picks)

목표: "종목 3개 + 매입가 + 매도가"라는 초보자의 요구를 **정직하게** 충족한다.
확률을 지어내지 않는다 — 같은 규칙을 누구나 재현할 수 있는 **스크리닝 산출물 + 진입·청산 규칙**으로 답한다.

## 절차 (순서 고정)

### ⓪ 기대치부터 세팅 (`backtest_stats`)
응답 첫 부분에 반드시 해당 주기의 실측 통계를 병기한다:
- "이 유니버스(66종목·10년·표본 약 7천 건)의 실측 승률은 월 51% / 분기 54% / 연 56% 수준이고,
  최악 5% 구간 손실은 연 기준 -42% 입니다. **어떤 방법도 90% 확률을 만들 수 없으며,
  그렇게 말하는 서비스는 피하세요.**"
- 생존 편향(현재 대형주 기준 유니버스)·배당 제외·거래비용 미반영 한계도 함께.
- **통계 인용 규칙 (강제)**: 수치와 주기 라벨(월/분기/연)은 `backtest_stats` 응답에서
  **그대로 옮겨 적는다** — 기억으로 재구성 금지. 주기 라벨 바꿔치기(실측 사고: 연 기준
  최악 5% 손실 -42% 를 "월 기준"으로 오기)는 그 자체가 지어낸 숫자다. 인용 전
  도구 응답의 해당 주기 필드와 대조하라.
- 사용자가 "90% 확률"을 요구하면: 그 숫자가 왜 존재할 수 없는지 이 통계로 설명하고,
  아래 절차가 그 욕구의 정직한 대체물임을 말한 뒤 진행한다.

### ① 예산·주기 확인 (질문 최대 2개)
총 투자 예산(잃어도 생활에 지장 없는 돈)과 주기(월/분기/연)를 확인한다.
관심 업종이 있으면 반영하되, 3종목이 같은 업종에 몰리지 않게 한다 (집중 리스크).

### ② 후보 좁히기 (`universe_list` → 10~12개)
유니버스에서 업종 분산을 고려해 후보 10~12개를 고른다. 사용자 관심사가 있으면 우선 반영.
유니버스 밖 종목을 사용자가 지명하면 같은 규칙으로 함께 점검한다.

### ③ 규칙 스크리닝 (후보별 라이브 확인 — 탈락 사유를 표로)
| 규칙 | 도구 | 탈락 조건 |
|---|---|---|
| R1 수익성 | `dart_financial_summary` | 최근 연간 영업이익 적자 |
| R2 성장 유지 | `dart_financial_summary` | 매출 2년 연속 역성장 |
| R3 악재 없음 | `news_search_risk` | 유상증자·횡령·배임·거래정지·상장폐지 보도 확인 |
| R4 추격 구간 아님 | `price_history` (streak) | 3거래일 이상 연속 상승 중 (진입 시점만 문제 — 종목 자체 탈락은 아님, "대기" 표기) |
| R5 52주 위치 | `price_quote` | 52주 고점 대비 -5% 이내면 "고점 부근" 감점 표기 (탈락 아님) |

모든 후보의 통과/탈락/대기를 표로 남긴다 — **왜 이 3개가 남았는지가 산출물의 핵심**이다.
남은 종목이 3개 미만이면 후보를 보충하고, 그래도 부족하면 부족한 대로 말한다
(3개를 채우려고 규칙을 느슨하게 하지 않는다).

### ④ 상위 3종목 발표 (종목당 4요소, trust-explainer 구조)
1. **남은 이유**: 통과한 규칙과 수치 (출처·시점 병기)
2. **반대 근거 1개 이상**: 이 선택이 틀릴 수 있는 조건
3. **매매 계획** (`plan_trade`, 종목당 예산 = 총예산 ÷ 3):
   3분할 매입 밴드(현재가/-5%/-10%) + 손절 -7% + 1차 익절 +20% 기준가·수량
4. **이 종목의 주기 체크포인트**: 다음 실적 발표 등 확인할 것 1개

### ⑤ 주기별 리밸런싱 체크리스트 (마무리)
- **월**: 종목당 비중 상한(총자산 20% — 전 스킬 공통 기준) 초과 여부 + 손절선 이탈 여부만 확인. 그 외엔 손대지 않는다.
- **분기**: 실적 발표 확인 — R1/R2 가 깨졌으면 교체 검토 (`trade-retrospective` 로 지난 주기 복기 권장).
- **연**: 포트 전체 재구성 — 같은 절차를 처음부터 반복. 이게 "스스로 반복할 수 있는 절차"의 완성이다.

## 산출물 저장

파일 쓰기가 가능한 환경이면 최종 리포트(④+⑤ 전체, 고지문 포함)를 작업 디렉터리의
**`outputs/result-<YYYY-MM-DD>.md`** 로 저장하고 응답 끝에 저장 경로를 알려준다
(제목: `# 주식 투자 리포트`, 생성 시각과 사용 프롬프트를 상단에 병기).
파일 쓰기가 불가하면 조용히 생략한다.

**저장 경로 규칙 (강제)**:
1. 프롬프트에 `[하네스 지시]` 로 저장 경로가 지정돼 있으면 **그 경로만** 쓴다.
2. 아니면 저장 직전에 존재 여부를 반드시 확인하고, 이미 있으면 `-2`, `-3` … 중
   **비어 있는 다음 순번**을 쓴다.
3. **기존 `outputs/result-*.md` 를 덮어쓰거나 수정하는 것은 어떤 경우에도 금지** —
   이전 리포트는 사용자의 기록이다 (실측 사고: 같은 날 재실행이 아침 리포트를 덮어씀).

## 금지 사항 (컴플라이언스)

- **확률·목표 수익률 수치 생성 금지**: "오를 확률 N%", "N% 수익 예상" 금지 — 인용 가능한 숫자는
  `backtest_stats` 실측치뿐이며 반드시 "과거 통계, 미래 보장 아님"을 병기.
- 지시형("사라") 금지 — "규칙 N개 중 M개 통과"로 말하고 최종 실행은 사용자 몫.
- 매매 계획의 가격은 예측이 아니라 규칙임을 매번 명시 (2·3차 밴드 미도달 시 미집행이 원칙).
- 레버리지·미수·신용 금지. FOMO 유발 표현 금지.
- 응답 끝 고지: "본 안내는 교육 목적의 정보 제공이며 투자자문·투자권유가 아닙니다.
  투자 판단과 그 결과(손실 포함)에 대한 책임은 투자자 본인에게 있습니다."
