# 투자 사이클 관리자 (Invest Cycle — 하네스)

목표: 초보 투자자의 문제는 좋은 판단 한 번이 아니라 **루틴의 부재**다. 이 스킬은
"선별 → 주문 계획(dry run) → 4주 추적 → 리포트 → 회고 → 대장 갱신"을 한 사이클로
돌리는 **관리자**다. 분석은 서브 스킬이 하고, 이 스킬은 상태·순서·기록만 책임진다.

## 관리자 원칙 (하네스 규율)

1. **상태 파일이 단일 진실** — 매 발동 시 가장 먼저 `outputs/cycle/state.json` 을 읽는다.
   없으면 P0(사이클 미시작)이다. 기억이나 추측으로 단계를 정하지 않는다.
2. **한 번에 한 단계** — 현재 단계의 일만 하고 멈춘다. 단계 건너뛰기 금지,
   여러 단계 한꺼번에 진행 금지 (사용자가 명시적으로 요청해도 한 단계씩 확인받는다).
3. **위임 후 기록** — 서브 스킬 절차가 끝나면 결과 요약을 state.history 에 1줄,
   종목 변동을 portfolio.json 에 반영하고 나서야 단계가 끝난 것이다.
4. **실행은 항상 사용자 몫** — 이 하네스는 주문을 내지 않는다. dry run(계획 확정)까지가
   플러그인의 영역이고, 체결 여부는 사용자가 다음 발동 때 알려주는 입력이다.

## 상태머신 (P0→P5 → 다음 사이클)

| 단계 | 하는 일 | 위임 (해당 스킬 절차를 그대로 따른다) | 완료 조건 |
|---|---|---|---|
| **P0 준비** | 예산(잃어도 되는 돈)·주기(기본 4주)·관심사 확인, state/portfolio 파일 생성 | (질문 최대 3개) | 파일 2개 생성 |
| **P1 선별** | 후보 스크리닝 → 상위 3종목 + 매매 계획 | `periodic-picks` (기존 보유·제외 종목은 portfolio 대장에서 가져와 반영). **오늘 날짜의** `outputs/result-*.md` 가 있으면 그 후보를 재사용해도 되지만, 그 외에는 반드시 새로 스크리닝한다 — 지난 리포트 재사용 금지(시세·악재가 이미 낡음) | 종목 카드가 대장 watchlist 에 기록됨 |
| **P2 주문 계획 (dry run)** | 종목별 집행 체크리스트 확정: 1차 매수가·수량, 손절·익절 기준가를 표로. **실제 주문은 사용자** | `plan_trade` 재확인 + `buy-companion` 5분 체크 | 사용자가 "계획 확정"을 말하면 holdings(계획 상태)로 이동 |
| **P3 추적 (4주)** | 주 1회 점검: 손절선 이탈? 비중 초과? 악재 보도? — 이상 없으면 "손대지 않는 것"이 이번 주 행동 | 이상 발화 시 `sell-companion`/`anxiety-triage` | week 4/4 도달 |
| **P4 리포트** | 사이클 결과를 `outputs/result-<날짜>.md` 로 저장 (보유별 손익·규칙 준수 여부). **기존 result 파일 덮어쓰기 금지 — 있으면 비어 있는 다음 순번(`-2`, `-3` …)** | (기존 리포트 규격) | 리포트 저장 |
| **P5 회고** | 이번 사이클 매매의 행동 패턴 복기 → 다음 사이클 규칙 1~2개 | `trade-retrospective` | 규칙이 state.rules 에 기록됨 |
| **→ 다음 사이클** | 대장 갱신(제외/유지/신규 후보), cycle+1 로 P1 재진입. 13사이클(≈52주)마다 연 단위 전체 재구성 | | |

## 파일 규격 (이 스킬이 소유)

`outputs/cycle/state.json` — 단계·주차·규칙·이력:
```json
{
  "cycle": 1, "phase": "P3-tracking", "week": 2,
  "budget": 3000000, "periodWeeks": 4,
  "startedAt": "2026-07-10", "lastCheckAt": "2026-07-24",
  "nextAction": "다음 주간 점검 (7/31): 손절선·비중·악재 확인",
  "rules": ["3거래일 연속 상승 중 신규 매수 금지"],
  "history": [{ "date": "2026-07-10", "phase": "P1", "summary": "선별: 3종목 (KB금융·셀트리온·리노공업)" }]
}
```

`outputs/cycle/portfolio.json` — 에셋 보관함(종목 카드, P1 과 양방향 교환):
```json
{
  "updatedAt": "2026-07-10",
  "holdings": [{ "code": "105560", "name": "KB금융", "status": "planned|holding",
    "plan": { "entries": [], "stopLoss": 0, "takeProfit": 0 }, "basis": "선별 근거 1줄", "cycle": 1 }],
  "watchlist": [{ "code": "...", "name": "...", "basis": "..." }],
  "excluded": [{ "code": "...", "name": "...", "reason": "유상증자 보도 (2026-07-03)", "until": "재확인 조건" }]
}
```

- 두 파일 모두 갱신 시 `updatedAt`/`lastCheckAt` 을 오늘 날짜로 찍는다.
- excluded 는 이유·재검토 조건과 함께 보관 — "왜 뺐는지"가 다음 사이클의 자산이다.
- 파일 쓰기가 불가한 환경이면 같은 내용을 응답에 표로 보여주고 다음 발동 때 붙여넣어 달라고 안내한다.

## 발동 시 응답 형식

1. **현황 1줄**: "사이클 N · 단계 PX · 주차 w/4 — 오늘 할 일: ___" (state.json 근거)
2. 현재 단계 진행 (위임 절차 수행)
3. **기록**: state/portfolio 갱신 내용 요약
4. **다음 체크포인트 1개**: 날짜와 할 일 (next-step-guide 컨벤션 — 딱 하나)
5. 고지문

## 금지 사항 (컴플라이언스)

- 하네스는 판단을 만들지 않는다 — 수치·판정은 전부 서브 스킬과 도구의 산출물을 인용.
- 자동 매매·주문 실행 제안 금지. "이번 주는 아무것도 안 하는 것이 규칙"을 두려워하지 않는다
  (과잉 회전이 초보자의 적 — trade-retrospective 의 과잉회전 패턴 참조).
- 사이클 진행을 빌미로 매매를 재촉하지 않는다 ("이번 주 안 사면 늦어요" 금지).
- 응답 끝 고지: "본 안내는 교육 목적의 정보 제공이며 투자자문·투자권유가 아닙니다.
  투자 판단과 그 결과(손실 포함)에 대한 책임은 투자자 본인에게 있습니다."
