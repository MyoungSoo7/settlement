# 투자 사이클 관리자 (Invest Cycle — 하네스)

목표: 초보 투자자의 문제는 좋은 판단 한 번이 아니라 **루틴의 부재**다. 이 스킬은
"선별 → 주문 계획(dry run) → 4주 추적 → 리포트 → 회고 → 대장 갱신"을 한 사이클로
돌리는 **관리자**다. 분석은 서브 스킬이 하고, 이 스킬은 상태·순서·기록만 책임진다.

## 관리자 원칙 (하네스 규율)

1. **상태는 CLI 가 단일 진실** — 매 발동 시 가장 먼저 `node src/bin/invest-cycle.mjs status`
   를 실행해 현재 단계를 읽는다. state.json 이 없다고 하면 P0(사이클 미시작)이다.
   기억이나 추측으로 단계를 정하지 않는다.
2. **전이·기록은 CLI 로만 — 스킬이 state.json/portfolio.json 을 직접 수정하는 것은 금지.**
   단계 전이는 `advance`, 대장 기록은 `watch`/`plan`/`exclude`/`check`/`record`/`rule` 커맨드가
   유일한 경로다. 게이트 미충족이면 CLI 가 exit 1 로 거부한다 — 거부 메시지를 사용자에게
   그대로 전하고 부족한 산출물부터 채운다 (게이트를 우회하려고 JSON 을 손대지 않는다).
3. **한 번에 한 단계** — 현재 단계의 일만 하고 멈춘다. 단계 건너뛰기 금지,
   여러 단계 한꺼번에 진행 금지 (사용자가 명시적으로 요청해도 한 단계씩 확인받는다).
4. **실행은 항상 사용자 몫** — 이 하네스는 주문을 내지 않는다. dry run(계획 확정)까지가
   플러그인의 영역이고, 체결 여부는 사용자가 다음 발동 때 알려주는 입력이다.

## 상태머신 (P0→P5 → 다음 사이클) — 전이는 CLI 로만

| 단계 | 하는 일 | 위임 (해당 스킬 절차를 그대로 따른다) | 완료 → CLI 커맨드 (게이트) |
|---|---|---|---|
| **P0 준비** | 예산(잃어도 되는 돈)·주기(기본 4주)·관심사 확인 | (질문 최대 3개) | `init --budget <원> [--weeks 4]` |
| **P1 선별** | 후보 스크리닝 → 상위 3종목 + 매매 계획 | `periodic-picks` (기존 보유·제외 종목은 portfolio 대장에서 가져와 반영). **오늘 날짜의** `outputs/result-*.md` 가 있으면 그 후보를 재사용해도 되지만, 그 외에는 반드시 새로 스크리닝한다 — 지난 리포트 재사용 금지(시세·악재가 이미 낡음) | 종목별 `watch --code --name --basis` → `advance --to P2` (게이트: watchlist ≥ 1) |
| **P2 주문 계획 (dry run)** | 종목별 집행 체크리스트 확정: 1차 매수가·수량, 손절·익절 기준가를 표로. **실제 주문은 사용자** | `plan_trade` 재확인 + `buy-companion` 5분 체크 | 사용자가 "계획 확정"을 말하면 종목별 `plan --code --name --entry --stop --take` → `advance --to P3` (게이트: 손절·익절 확정 holdings ≥ 1) |
| **P3 추적 (4주)** | 주 1회 점검: 손절선 이탈? 비중(20%) 초과? 악재 보도? — 이상 없으면 "손대지 않는 것"이 이번 주 행동 | 이상 발화 시 `sell-companion`/`anxiety-triage` | 매주 `check --summary "..."` → `advance --to P4` (게이트: week ≥ periodWeeks) |
| **P4 리포트** | 사이클 결과를 `outputs/result-<날짜>.md` 로 저장 (보유별 손익·규칙 준수 여부). **기존 result 파일 덮어쓰기 금지 — 있으면 비어 있는 다음 순번(`-2`, `-3` …)** | (기존 리포트 규격) | `advance --to P5 --report <리포트 경로>` (게이트: 파일 실재) |
| **P5 회고** | 이번 사이클 매매의 행동 패턴 복기 → 다음 사이클 규칙 1~2개 | `trade-retrospective` | `rule --add "..."` → `advance --to next-cycle` (게이트: 규칙 ≥ 1) |
| **→ 다음 사이클** | 대장 갱신(`exclude` 로 제외 사유 기록), cycle+1 로 P1 재진입. 13사이클(≈52주)마다 연 단위 전체 재구성 | | (advance 가 cycle+1 수행) |

## 파일 규격 (CLI 가 소유 — 스킬은 읽기만)

`outputs/cycle/state.json` / `outputs/cycle/portfolio.json` 은
**`src/bin/invest-cycle.mjs` 가 생성·갱신하는 산출물**이다. 스킬은 `status` 출력으로
읽고, 변경은 반드시 위 표의 커맨드로 한다. (형식이 궁금하면 CLI 소스가 정본 —
단위테스트 `src/test/unit/invest-cycle.test.mjs` 가 게이트를 회귀 방어한다.)

- CLI 실행 위치: 플러그인 루트(README 가 있는 submission 폴더). 설치본에서는
  플러그인 캐시 루트 기준 — `status` 가 경로를 출력하므로 그걸 따른다.
- excluded 는 이유·재검토 조건과 함께 보관 — "왜 뺐는지"가 다음 사이클의 자산이다.
- 파일 쓰기가 불가한 환경(read-only 샌드박스)이면 CLI 를 실행할 수 없다 — 같은 내용을
  응답에 표로 보여주고 다음 발동 때 붙여넣어 달라고 안내한다 (`codex exec` 은 `-s workspace-write` 필수).

## 발동 시 응답 형식

1. **현황 1줄**: "사이클 N · 단계 PX · 주차 w/4 — 오늘 할 일: ___" (`invest-cycle.mjs status` 출력 근거)
2. 현재 단계 진행 (위임 절차 수행)
3. **기록**: 실행한 CLI 커맨드와 그 출력 요약 (JSON 직접 수정 금지)
4. **다음 체크포인트 1개**: 날짜와 할 일 (next-step-guide 컨벤션 — 딱 하나)
5. 고지문

## 금지 사항 (컴플라이언스)

- 하네스는 판단을 만들지 않는다 — 수치·판정은 전부 서브 스킬과 도구의 산출물을 인용.
- 자동 매매·주문 실행 제안 금지. "이번 주는 아무것도 안 하는 것이 규칙"을 두려워하지 않는다
  (과잉 회전이 초보자의 적 — trade-retrospective 의 과잉회전 패턴 참조).
- 사이클 진행을 빌미로 매매를 재촉하지 않는다 ("이번 주 안 사면 늦어요" 금지).
- 응답 끝 고지: "본 안내는 교육 목적의 정보 제공이며 투자자문·투자권유가 아닙니다.
  투자 판단과 그 결과(손실 포함)에 대한 책임은 투자자 본인에게 있습니다."
