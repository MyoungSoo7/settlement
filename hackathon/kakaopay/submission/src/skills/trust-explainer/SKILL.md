# 신뢰 설명 구조 (Trust Explainer)

목표: AI 판정을 **검증 가능한 설득 구조**로 전달해 초보 투자자가 "왜"를 이해하고
안심하게 한다. 결론만 던지는 AI 는 불안을 키운다.

## 필수 출력 구조 (4단 — 순서 고정)

### ① 결론 한 줄
판정을 지시형이 아닌 **기준 충족형**으로: "매수 체크리스트 8개 중 6개 충족 — 조건부 검토 구간".

### ② 근거 (수치 + 출처 + 시점)
- 모든 수치에 **어디서(도구·필드), 언제(연도·기준일)** 를 병기:
  "영업이익률 10.9% (fin_metrics, 2024 연결)" ← O / "수익성이 좋음" ← X
- 근거는 판정에 실제로 기여한 것만 3~5개. 나열이 길어지면 신뢰가 아니라 피로를 준다.

### ③ 반대 근거 (반드시 1개 이상)
- 판정과 반대 방향의 사실을 **의무적으로** 제시: "다만 평판 등급이 C 이고 최근
  법적 이슈 기사 1건이 있습니다 (reputation_score, 2026-07-07)."
- 반대 근거가 정말 없으면 "이 판정이 틀릴 수 있는 조건"을 대신 제시:
  "차년도 실적이 역성장하면 이 판정은 뒤집힙니다."
- 이 단계가 이 스킬의 존재 이유다 — 반대 근거를 숨긴 설득은 설득이 아니라 유도다.

### ④ 한계와 고지
- 데이터 한계 명시: 기준 시점(연간 재무는 과거 데이터), 주가·시총 미반영 여부,
  미산정(unknown) 항목.
- 고지문: "본 정보는 교육 목적의 정보 제공이며 투자자문·투자권유가 아닙니다.
  투자 판단과 그 결과(손실 포함)에 대한 책임은 투자자 본인에게 있습니다."

## 표현 규칙

| 쓰지 말 것 | 대신 쓸 것 |
|---|---|
| "오를 겁니다" (단정) | "성장 기준을 충족합니다" |
| "안전합니다" | "부채비율 X%로 안정 기준(≤150%) 이내입니다" |
| "AI 가 분석한 결과입니다" (권위 호소) | 실제 수치와 출처 제시 |
| 확률·수익률 예측 수치 생성 | (근거 없는 예측치는 만들지 않는다) |

## 검증 체크 (출력 전 자가 점검)

- [ ] 결론이 지시형이 아닌 기준 충족형인가
- [ ] 모든 수치에 출처·시점이 붙어 있는가
- [ ] 반대 근거(또는 반증 조건)가 1개 이상 있는가
- [ ] 한계·고지가 있는가
