# 트렌드 체크 (Trend Check)

목표: 트렌드 소비의 진짜 불안은 "**지금 사면 한 철 입고 못 입게 되는 것 아닌가**"다.
느낌이 아니라 시계열로 답한다 — 그리고 트렌드 판정과 **구매 판단을 분리**한다
(유행이 정점이어도 내 옷장과 조합되면 사도 되고, 상승 중이어도 조합이 안 되면 낭비다).

## Step 1: 시계열 확보

- `buzz_trend(keyword, months=12~24)` — 월별 언급량, 방향, 최근3/직전3 비율.
- 비교 질문("A vs B")은 `trend_compare` 로 한 번에.
- `coverageComplete=false` 면 커버된 구간만으로 말하고 한계를 명시한다.
  null 인 달은 0이 아니라 미상 — 그래프를 그릴 때 구분한다.
- (데이터랩 권한이 있으면 `datalab_shopping_trend` 로 실클릭 시계열 교차 검증)

## Step 2: 트렌드 수명 분류

| 분류 | 판별 테스트 | 구매 관점 의미 |
|---|---|---|
| **상승 초입** | rising + 언급량 절대치 아직 낮음 | 선점 가치 있으나 검증 부족 — 소액/베이식 형태로 |
| **본격 유행** | rising + 언급량 높음 | 접근성 최고·가격 프리미엄 주의, 한 철 리스크 시작 |
| **정점/포화** | flat + 언급량 높음 (12개월+ 지속) | 클래식화 가능성 vs 급락 직전 — 지속 개월 수가 근거 |
| **식는 중** | falling | 세일이 많아지는 구간 — "싸서 사는" 함정 주의 |
| **판단 유보** | insufficient-data / 언급량 희소 | 데이터 없음을 명시하고 단정하지 않는다 |

- 아이템의 **형태**로 보정한다: 같은 트렌드라도 베이식 실루엣(스트레이트 데님)은 유행
  종료 후에도 입고, 시그니처 디테일이 강할수록 유행과 운명을 같이한다.

## Step 3: 판정 전달 (출력 — honest-stylist 4단 구조)

1. **결론**: "OO는 [분류]" 한 줄 + 최근 3개월 vs 직전 3개월 수치.
2. **근거**: 월별 시계열 (표 또는 간단 텍스트 그래프) + 관련 보도 1~2건.
3. **반대 근거·한계**: 언급량은 수요의 프록시일 뿐 매출이 아님, 커버리지 한계 명시.
4. **다음 단계 1개**: 살 생각이면 → style-match 로 "내 옷장과 3가지 조합이 되는가" 확인.

## 금지 사항

- "지금 안 사면 늦는다" 류 FOMO 조장 금지 — 시계열이 상승이어도 구매 재촉으로 번역하지 않는다.
- 시계열 없이 유행 단정 금지. 데이터가 얇으면 얇다고 말한다.
