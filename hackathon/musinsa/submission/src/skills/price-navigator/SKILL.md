# 프라이스 내비게이터 (Price Navigator)

목표: 패션 소비의 두 번째 마찰은 **가격 불신**이다 — "70% 할인"이 정가 뻥튀기인지,
지금 보는 세일가가 실제로 싼 건지 알 방법이 없다. 할인율(판매자의 언어)이 아니라
**시장 분포에서의 위치**(구매자의 언어)로 번역한다.

## Step 1: 분포 확보

- `price_band(query, referencePrice=지금 보는 가격)` — 시장 최저/25%/중앙값/75%/최고 +
  기준가의 퍼센타일·중앙값 대비 %.
- query 는 상품을 특정할 수 있게 "브랜드 + 상품명/종류" 로. 결과의 표본 수(sampled)와
  동일 상품 여부(제목 확인)를 점검한다 — 옵션/시즌이 다른 상품이 섞이면 그 사실을 명시.

## Step 2: 판정 (할인율 언어 → 분포 언어)

| 기준가 위치 | 판정 |
|---|---|
| 하위 25% 이내 | 실질적으로 싼 가격 — 분포 근거 제시 |
| 25%~중앙값 | 평균 이하 — 나쁘지 않으나 "역대급"은 아님 |
| 중앙값~75% | 표시 할인율과 무관하게 시장 기준 비싼 편 — 정가 앵커 의심 |
| 상위 25% | 명백히 비쌈 — 다른 판매처 확인 권고 |

- **정가 앵커 감지**: 표시 정가가 분포 최고가보다 높으면 "그 정가에 팔린 적이 있는지"
  의심 근거로 제시한다 (할인율 70%가 아니라 실판매가로 판단).
- 최저가 판매처가 낯선 몰이면: 가품 리스크 확인 절차로 연결 —
  `brand_risk_scan` 에 "가품" 보도가 있으면 공식 판매처 우선을 권고.

## Step 3: "지금 살까 기다릴까"

- 확정 답 대신 **판단 규칙**을 준다: 시즌오프 캘린더(1월/7월 전후 폭이 큼),
  기준가가 이미 하위 25% 이내면 기다림의 기대 이익이 작다는 산술,
  트렌드가 falling 이면 세일이 더 올 확률이 높다는 관측(trend-check 연계).
- 필요 시점(당장 입을 일)이 있으면 그것이 최우선 변수임을 인정한다.

## Step 4: 출력 (honest-stylist 4단 구조)

1. **결론**: "45,000원은 시장 분포 하위 N% — [판정]" 한 줄.
2. **근거**: 분포 수치(min/중앙값/max, 표본 수), 조회 시점.
3. **반대 근거·한계**: 옵션·사이즈별 가격 차이, 배송비 미포함, 시점 변동 가능.
4. **다음 단계 1개**: 사기로 했다면 → style-match 3-코디 확인 또는 size-safety 체크.

## 금지 사항

- "최저가 보장", "무조건 이득" 표현 금지 — 모든 가격 판정에 조회 시점을 붙인다.
- 구매 재촉 금지 — "품절 임박" 류 압박 화법을 쓰지 않는다.
- 특정 판매처 비방 금지 — 가격 사실과 리스크 보도만 전달한다.
