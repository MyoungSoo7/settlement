# 매수·매도 기준 (초보 투자자 표준 체크리스트)

핵심 철학: **"사라/팔아라"가 아니라 "기준 충족 여부"를 보여준다.**
판단은 데이터가, 결정은 투자자가 한다.

## 매수 체크리스트 (BUY-8)

정량 5개는 MCP 도구로 자동 판정, 거시 2개는 경제지표로, 정성 1개는 뉴스/평판으로 판정한다.

| # | 축 | 기준 | 데이터 출처 | 초보자 근거 |
|---|---|---|---|---|
| B1 | 성장성 | 최근 연도 매출액 YoY > 0 | `fin_metrics` revenueGrowthPct | 역성장 기업은 반등 논거가 따로 필요 |
| B2 | 수익성 | 영업이익률 ≥ 5% | `fin_metrics` operatingMarginPct | 본업으로 돈을 버는가 |
| B3 | 이익 실재 | 당기순이익 > 0 (흑자) | `fin_metrics` netIncome | 적자 기업은 초보자 회피 대상 |
| B4 | 재무 안정 | 부채비율 ≤ 150% (null=자본잠식 → 즉시 fail) | `fin_metrics` debtRatioPct | 금리 상승기 생존력 |
| B5 | 자본 효율 | ROE ≥ 8% | `fin_metrics` roePct (로컬 계산 — 서버엔 ROA 만 있음) | 자기자본으로 얼마나 버는가 |
| B6 | 금리 환경 | 기준금리 인하/동결 사이클 | `econ_series('BASE_RATE', 6개월)` 추이 | 인상기는 밸류에이션 압박 |
| B7 | 환율 안정 | USD/KRW 3개월 상승률 < +5% | `econ_series('USD_KRW', 3개월)` | 급등 = 외국인 수급 부담 (단, 수출주는 실적엔 우호 — 이중성 명시) |
| B8 | 평판 | grade C 이상 + negativeByCategory 에 LEGAL/FINANCIAL 없음 | `reputation_score` (latest=null 은 미산정 → unknown) + `news_recent` | 정성 리스크 필터 |

### 판정 규칙

- **7~8 충족**: 기준상 "적극 검토" 구간 — 그래도 분할 매수(3회)로 진입 제시
- **5~6 충족**: "조건부 검토" — 미충족 항목이 무엇인지, 왜 중요한지 설명 필수
- **4 이하**: "보류" — 미충족 항목이 해소되는지 관찰 포인트 제시
- B3(적자) 또는 부채비율 200% 초과는 **단독으로도 보류 사유** (하드 필터)

업종별 보정: 금융업은 부채비율 기준 부적용(사업 구조상 높음), 바이오/성장주는 B2·B3 미충족이
일반적임을 명시하고 "초보자 기준으로는 보류"로 판정하라 — 기준을 몰래 완화하지 마라.

## 매도 체크리스트 (SELL-7)

데이터로 판정 가능한 4개 + 투자자가 HTS 에서 확인할 3개로 구분해 제시한다.

**데이터 기반 (자동 판정)**
| # | 기준 | 판정 |
|---|---|---|
| S1 | 영업이익 적자 전환 or 매출 YoY -20% 이하 | 펀더멘털 훼손 — 매도 검토 신호 |
| S2 | 부채비율 200% 초과 or 전년 대비 +50%p 급등 | 재무 리스크 급증 |
| S3 | 평판 점수 급락 / 최근 30일 부정 뉴스 스파이크 | 정성 악재 — 원인 기사 확인 필수 |
| S4 | 거시 악화: 기준금리 급등 사이클 + CPI 급등 동시 | 시장 전체 위험 — 비중 축소 검토 |

**가격 기반 (투자자 확인 항목 — 데이터 없음, 질문으로 확인하라)**
| # | 기준 | 원칙 |
|---|---|---|
| S5 | 매수가 대비 -7% 도달 | 기계적 손절 — 예외 없이 실행이 초보자 원칙 |
| S6 | 목표 수익(+20~30%) 도달 | 분할 익절 (절반 매도 후 나머지 보유 등) |
| S7 | 매수 논거 소멸 | 살 때 이유가 사라졌으면 보유 이유도 없다 |

S1~S3 중 하나라도 해당하면 "물타기(추가 매수) 금지"를 명시하라 — 펀더멘털 훼손 시
평균단가 낮추기는 초보자의 대표 실패 패턴이다 (`risk-management` skill).

## 조사 절차 (도구 조합 순서)

```
1. company_search(기업명)          → stockCode 확정 (동명 기업 있으면 사용자에게 확인)
2. fin_statements(stockCode)       → 최근 2~3개년 원본 확보 (수치 인용 근거)
3. fin_metrics(stockCode)          → B1~B5 판정용 비율 (서버 계산 — 직접 나누지 마라)
4. econ_latest() / econ_series()   → B6~B7 (최근 3~6개월 추이로 사이클 판단)
5. reputation_score + news_recent  → B8, S3 (부정 기사는 제목·날짜 인용)
6. invest_signal(stockCode)        → 종합 점수 교차 검증 (계산 결과가 내 판정과 다르면 원인 명시)
```

## 보고 형식

1. **결론 한 줄**: "BUY-8 중 N개 충족 — [적극 검토/조건부/보류]" (매수) 또는 해당 매도 신호
2. 체크리스트 표: 항목별 충족 여부 + 실제 수치 (연도·기준일 병기)
3. 미충족 항목 해설: 왜 중요한지, 어떤 변화가 있으면 재평가할지
4. 리스크 요약 + 분할 매수/손절 규칙 (`risk-management`)
5. 필수 고지문 (AGENTS.md 컴플라이언스)
