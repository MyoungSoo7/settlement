# 재무제표 해석 (financial-statements-service)

이 서비스는 코스피 상장사의 **연간 요약 재무제표**(DART 수집)를 제공한다.
분기 데이터·주가·시가총액·발행주식수는 없다 — PER/PBR/목표주가는 계산 불가.

## 응답 필드 (fin_statements — 원 단위, 연도별)

- 원본 계정: `revenue`(매출) `operatingProfit`(영업이익) `netIncome`(당기순이익)
  `totalAssets`/`totalLiabilities`/`totalEquity`
- 서버 계산 비율(%): `operatingMargin` `netMargin` `debtRatio`(자본잠식 시 null)
  `equityRatio` `roa` — scale 2, HALF_UP
- `fiscalYear`(연도), `fsDivision`: **CFS(연결) 우선, OFS(별도)는 폴백** — 어느 쪽인지 항상 병기
- `source`: DART(공시 수집) / SEED(시드 폴백) — SEED 데이터면 그 사실을 명시
- **ROE 는 서버에 없다** — `fin_metrics` 의 `roePct`(순이익/자본총계 로컬 계산)를 쓰라

## 3축 지표 (fin_metrics 가 서버에서 계산)

### 성장성
- **매출 YoY** = (당해 매출 − 전년 매출) / 전년 매출
- 해석: > +10% 고성장 / 0~10% 완만 / < 0 역성장 (B1 미충족)
- 함정: 일회성 매각/합병으로 튄 성장은 뉴스로 교차 확인하라 (`news_recent`)

### 수익성
- **영업이익률** = 영업이익 / 매출액 — 본업의 이익 체력 (B2 기준 ≥ 5%)
- **순이익률** = 당기순이익 / 매출액 — 영업이익률과 크게 다르면 영업외 손익(이자·일회성) 확인
- **ROE** = 당기순이익 / 자본총계 (B5 기준 ≥ 8%)
- 함정: 순이익만 급증한 해는 자산 매각 등 일회성일 수 있다 — 영업이익 추이와 함께 보라

### 안정성
- **부채비율** = 부채총계 / 자본총계 (B4 기준 ≤ 150%, 200% 초과는 하드 필터)
- **자본잠식 체크**: 자본총계 < 0 이면 즉시 "초보자 투자 불가" 판정 — 다른 지표 무의미
- 함정: 금융업(은행·보험·증권)은 사업 구조상 부채비율이 높다 — 업종 명시 후 기준 부적용

## 다년 추이 원칙

- 단년 스냅샷으로 판단하지 마라 — **최근 2~3개년**을 함께 조회해 방향을 본다
  (개선 추세의 B2 4.8% 는 악화 추세의 5.2% 보다 낫다 — 다만 체크리스트 판정 자체는 기준값대로)
- 연도별 수치를 표로 병기하라: 매출/영업이익/순이익/부채비율 × 연도

## 숫자 다루기

- 금액 필드는 **원 단위 정수 문자열**로 취급하고, 표기는 억/조 단위로 환산해 병기하라
  (예: "매출 2,437,140억 → 243.7조"). 환산 시 반올림 자리수를 명시.
- 비율 계산은 `fin_metrics` 서버 계산값을 사용하라 — 직접 나눠서 다른 값을 만들지 마라.
  서버값과 원본으로 검산해 다르면 그 사실을 보고하라.
- 데이터 연도가 오래됐으면(최신 연도 < 작년) "수집 시점 한계"를 명시하라.
