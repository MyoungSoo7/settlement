## 사용자 입력
모바일 퍼즐 게임을 만들고싶어

## Cycle 1

### Wonder
- 의미 A — 핵심은 "게임 제작" 자체가 아니라 **수익을 만드는 것**. 게임은 수단.
- 의미 B — 그 수익은 **회사원 월급 외 부수입**. 본업은 유지하면서 별도 소득원을 만드는 것.
- 의미 C — 부수입의 모양은 **매달 습관적으로 쓰이는 돈**. 단발성 대박이나 매도 이벤트가 아니라 월 단위로 반복 유입되는 현금흐름.

### Reflect
- 의미 A
  - 사용자 결: 게임은 수단. 대체 가능하면 갈아탈 수 있다.
  - 모델 결: 동일 — 게임 제작 자체가 목적이 아니다. 수단.
  - 차이: **일치**.
- 의미 B
  - 사용자 결: 본업이 우선. 회사원 안정성을 해치지 않는 선에서만.
  - 모델 결: 동일 — 본업 시간/에너지를 크게 잠식하면 안 된다.
  - 차이: **일치**.
- 의미 C
  - 사용자 결: **1주마다 새 앱을 배포 → 4주마다 매출 체크 → 매출 나오는 서비스에 추가 투자**. 즉 포트폴리오 빠른 실험 모델.
  - 모델 결: 한 게임을 잘 만들어 운영하며 월 단위 반복 과금/광고 수익. 라이브 서비스 운영형.
  - 차이: **크게 다르다.** 모델은 "한 작품 깊은 운영"으로 봤으나, 본인은 "여러 후보를 빠르게 시장에 던져 살아남는 것에만 추가 자원을 붓는" 실험-측정-투자 사이클. "퍼즐 게임 한 개"가 아니라 1주짜리 배포 단위 중 하나에 가깝다.

### Refine
shared meaning: 회사원 본업을 유지한 채, 1주 단위로 새 모바일 앱(퍼즐 게임을 포함한 후보들)을 배포하고, 4주 단위로 매출을 체크해 매출이 나오는 것에만 자원을 추가 투입하는 식으로 월 단위 부수입을 만든다.

### Restate
goal: 본업(회사원)을 유지한 채, 1주 1개 모바일 앱 배포 · 4주 매출 체크 · 매출 발생 앱에만 추가 투자 사이클로 월 단위 부수입을 만든다.

## Cycle 2

### Wonder
- constraint 후보 1 — 주 1개 배포 일정 엄수. 완성도에 집착해 1주를 넘기지 않는다.
- constraint 후보 2 — 매출 제로 앱에는 미련/애정으로 추가 투자하지 않는다.
- acceptance 후보 — 단일 신호가 아니라 ①월 부수입 금액 ②배포 사이클 12주+ 유지 ③설치/장르 제증 세 가지를 동시 추적. 그리고 매 사이클 끝에 실패 앱의 원인·특징을 기록해 다음 사이클 판단 기준으로 쓰는 학습 루프 자체가 살아있어야 한다(컴파운드 엔지니어링).

### Reflect
- constraint 비교
  - 사용자 결: 회사 시간/자원 침범 금지 + 주 1개 일정 엄수 + 매출 제로 앱에 추가 투자 금지
  - 모델 결: 동일
  - 차이: 일치
- acceptance 비교
  - 사용자 결: 학습 누적 + 세 가지 신호(월 부수입 / 12주 사이클 유지 / 설치·장르 지표) 동시 추적
  - 모델 결: 동일
  - 차이: 일치

### Refine
shared meaning (보강): 회사 시간/자원을 침범하지 않으면서 주 1개 배포 일정을 지키고, 매 사이클 끝에 실패 앱의 원인·특징을 기록해 다음 사이클 판단 기준으로 쓰며, 매출 제로인 앱에는 추가 투자하지 않는다. 성공 신호는 월 부수입 금액 · 배포 사이클 12주+ 유지 · 설치/장르 지표 3가지를 동시 추적한다.

### Restate
goal (Cycle 1 그대로 유지): 본업(회사원)을 유지한 채, 1주 1개 모바일 앱 배포 · 4주 매출 체크 · 매출 발생 앱에만 추가 투자 사이클로 월 단위 부수입을 만든다.

constraints:
- 회사 업무 시간/자원/정보를 침범하지 않는다
- 주 1개 배포 일정을 엄수한다(완성도 집착으로 1주를 넘기지 않는다)
- 매출 제로인 앱에는 추가 투자하지 않는다(미련/애정으로 자원을 다시 쓰지 않는다)

acceptance_criteria:
- 매 사이클 끝에 실패 앱의 원인·특징을 기록해 다음 사이클 판단 기준으로 반영한다
- 월 부수입 누적 금액을 매월 측정한다
- 배포 사이클(주 1개) 자체를 12주 이상 끊김 없이 유지한다
- 설치 수 / 장르(카테고리) 지표를 매 앱 단위로 측정·기록한다
