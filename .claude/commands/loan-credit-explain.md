argument-hint: "<stockCode 또는 재무지표 (예: 부채비율=180 영업이익률=12 ROA=6 평판=B 자본총계=50000000000 기간=30)>"
---

`loan-domain-rules` 와 `money-safety` skill 을 로드하라. 대상: $ARGUMENTS

입력 해석:
- **6자리 stockCode** 면 기업 신용대출로 본다. 실행 중이면 loan-service 의 신용평가 엔드포인트
  (`EvaluateCorporateCreditUseCase` → `/api/corporate-loans/credit` 계열)로 조회하고, 없으면
  financial 공개 API 의 재무제표(부채비율·영업이익률·ROA·자본총계)를 근거로 skill 의 구간표로 직접 산정하라.
- **재무지표를 직접 준 경우**: skill 의 `CorporateCreditPolicy` 구간표로 결정적으로 계산하라. 추측 금지.
- "선정산" 이나 정산예정금이 언급되면 선정산 대출(`CreditPolicy`, LTV×평판 haircut)로 풀이하라.

기업 신용대출 출력 형식 (비개발자도 읽게):

```
안정성 (부채비율 180%)      30 / 40
수익성 (영업이익률 12%)     15 / 20
수익성 (ROA 6%)             15 / 20
평판 (등급 B)               15 / 20
────────────────────────────────
신용점수                    75 / 100  → 등급 B
한도  = 자본총계 500억 × 10% × 0.8(B) = 40억
수수료 = 원금 × 일할이율 × 30일 × 1.1(B)
판정                        승인 가능 (E등급/한도초과 아님)
```

- 반드시 **각 구간 경계**(예: 부채비율 ≤200 → 30점)를 근거로 제시하라 — "대략" 금지.
- **E등급(<35) 또는 신청액>한도 → 거절(422)** 임을 명시하라.
- null 지표(자본잠식·결측)는 0점 처리됨을 밝히고, 평판 미상은 10점(중립)임을 구분하라.
- 선정산 대출이면: `한도 = 미지급 정산예정금 × LTV × haircut(A·B1.0/C0.85/D0.70/E차단)`, 미상 등급은 1.0(fail-open) 로 풀이.
