description: 투자점수 산정 근거를 사람이 읽을 수 있게 풀이 (CEO 투자하기 문의 대응)
argument-hint: "<stockCode 또는 재무지표 (예: 영업이익률=12 ROA=6 부채비율=90 자기자본비율=55 매출YoY=15 순이익YoY=8)>"
---

`investment-domain-rules` skill 을 로드하라. 대상: $ARGUMENTS

입력 해석:
- **6자리 stockCode** 면: 실행 중이면 investment-service 투자점수 엔드포인트
  (`GetInvestmentScoreUseCase` → `/api/investments/scores/{stockCode}` 계열)로 조회하고,
  없으면 financial 공개 API 재무제표(+직전 연도)를 근거로 skill 의 구간표로 직접 산정하라.
- **재무지표를 직접 준 경우**: skill 의 `InvestmentScorePolicy` 구간표로 결정적으로 계산하라. 추측 금지.

출력 형식 (비개발자도 읽게):

```
수익성 (영업이익률 12%)      12 / 20
수익성 (ROA 6%)              6 / 15
안정성 (부채비율 90%)        16 / 20    ← 낮을수록 고득점
안정성 (자기자본비율 55%)    12 / 15
성장성 (매출 YoY +15%)       12 / 15
성장성 (순이익 YoY +8%)      9 / 15
────────────────────────────────────
투자점수                     67 / 100  → 등급 BBB
판정                         투자 적격 (≥60)
```

- 각 구간 경계를 근거로 제시하라(예: 부채비율 ≤100 → 16점). "대략" 금지.
- **투자 적격 = 총점 ≥60 (BBB 이상)**. <60 이면 부적격 → 주문 시 422(NotInvestableException) 임을 명시.
- **직전 연도 부재 → 성장성 중립 15점**(50%) 처리됨을 밝혀라. null 지표는 0점(보수적)임을 구분.
- 재원 문의가 섞이면: `가용 재원 = 확정 정산금 − EXECUTED 집행금` 공식과 집행 시 재검증을 함께 설명하라.
- **주가·PER/목표주가는 이 플랫폼에 없다** — 투자점수는 회계자료 기반임을 분명히 하고 가격 판단을 지어내지 마라.
