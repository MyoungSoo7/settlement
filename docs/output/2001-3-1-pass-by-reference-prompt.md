# 2001-3-1. Pass by Reference 프롬프트 (B안)

## Curriculum Mapping

- Module: 1-3. 분업형 하네스 기초
- Clip: 2001-3-1 — 메인 세션은 왜 쉽게 지치는가
- Key message: 메인 세션은 결정만 하고 탐색은 밖으로 보낸다

## 목적

Sub-agent의 결과를 **파일 시스템**에 영속화하고 메인 세션은 경로만 들고 종료시켜, 메인 컨텍스트가 거의 소비되지 않음을 시연한다. Pass by Value(A안)와 동일 리포지토리에서 동일 깊이로 비교하여 "탐색은 밖으로 보낸다"가 어떻게 메인 세션의 수명을 늘리는지 증명한다.

## 비교 변수

| 차원 | 본 실험(B) | 비교 실험(A) |
|---|---|---|
| Sub-agent 출력 채널 | 파일 작성 | 텍스트 반환 |
| 메인의 의무 | 파일 존재만 확인 | 사용자에게 풀어서 출력 |
| 정보 위치 | 디스크 | 메인 컨텍스트 안 |
| 다음 세션 재사용 | 가능 | 불가 |

## 측정 항목

- 메인 컨텍스트 사용량 (%)
- 누적 비용 ($)
- 메인 세션 답변 길이 (문자 수)
- 다음 세션이 정보 재사용 가능 여부 (Y/N)

## 프롬프트

```text
당신은 이 리포지토리의 강의자료 관리자입니다.

[작업 절차 — 반드시 이 순서로]
1. Sub-agent 1개를 소환해서 리포지토리 전체를 조사시키세요.
2. Sub-agent에게 다음을 직접 `output/analysis.md` 파일에 작성하라고
   지시하세요 (Write 권한 부여):
   - 디렉토리 구조 (모든 파일 경로 포함)
   - 6개 챕터별 핵심 내용 요약 (각 챕터당 200자 이상)
   - 실습 자료 29개 각각의 목적 (1줄씩)
   - 품질 감사 결과 요약
   - 다음 추천 작업 3가지
3. Sub-agent에게는 "작성 완료 후 파일 경로 한 줄만 반환"하라고
   명시적으로 지시하세요. 어떤 요약도 반환하지 말 것.
4. 메인 세션은 받은 경로로 `ls -la <path>` 만 실행해서
   파일 존재와 크기만 확인하고 종료하세요.

[제약]
- 메인 세션은 절대 파일을 Read하지 마세요.
- 메인 세션은 절대 리포지토리 파일을 직접 Read하지 마세요.
- 메인 세션의 최종 답변은 "✅ output/analysis.md 생성 완료
  (12,345 bytes)" 같은 한 줄이면 충분합니다.
- 사용자가 내용을 보고 싶다면 다음 세션에서 직접 파일을 열게 하세요.
```

## 예상 결과

- 메인 컨텍스트: **낮음** (경로 문자열 + ls 결과 한 줄)
- 단일 세션 비용: 비교적 비쌈 (sub가 풀텍스트 작성)
- 재사용성: artifact가 디스크에 영속 → 다음 세션이 그대로 소비

## 학습 포인트

비용은 1세션에 집중되지만, 산출물의 수명이 세션 수명을 넘어선다. N회 재사용에서 상각 곡선이 Pass by Value를 추월한다. 큰 작업에서는 *가능 vs 불가능*을 가르는 enabling pattern이다.

## 깨끗한 비교를 위한 가드레일

A안과 B안을 시연할 때 다음을 절대 어기지 말 것:

**A안에서 금지**: 파일 작성, 메인의 직접 Read
**B안에서 금지**: 메인의 파일 Read, 메인의 직접 리포지토리 탐색, sub의 요약 반환

이 가드레일이 무너지면 두 실험의 차이가 흐려진다.
