# 2001-2-3. 내 업무용 md 규칙서 + SKILL.md 초안 만들기

## Practice Task and Prompt

- Task: create `md 규칙서 초안 + SKILL.md 초안` for clip `2001-2-3` so the learner can apply the curriculum message `문서형 하네스는 규칙 저장소부터 시작한다` to their own work.
- Prompt to paste into AI:

```text
Fast Campus 하네스 엔지니어링 실습 2001-2-3을 도와줘.

Curriculum mapping:
- Module: 1-2. md 하네스 기초
- Clip: [실습] 내 업무용 md 규칙서 초안 만들기
- Key message: 문서형 하네스는 규칙 저장소부터 시작한다
- Required output: md 규칙서 초안 + SKILL.md 초안

Interview mapping:
- Use the interview framing that stable rules should reduce guessing by fixing role, principles, forbidden actions, and output expectations outside the one-off prompt.

Task:
내 입력을 바탕으로 `md 규칙서 초안 + SKILL.md 초안`을 작성해줘. 먼저 역할, 원칙, 금지사항을 규칙서 메모로 정리한 뒤, 최종 산출물은 바로 저장할 수 있는 `SKILL.md` 형태로 만들어줘. 아래 실습자료의 작성 템플릿 구조를 따르고, 모르는 정보는 추측하지 말고 "다시 물을 질문" 또는 "확인 필요"로 남겨.

Boundaries:
- 강의자료와 내 입력의 범위를 벗어난 외부 사례를 추가하지 않는다.
- 검증되지 않은 수치 및 주장을 넣지 않는다.
- 최종 책임 판단이 필요한 항목은 사람이 결정할 수 있게 분리한다.

My input:
[여기에 실패 원인 체크리스트에서 뽑았던 내 업무 상황, 원문, 제약, 기존 산출물을 붙인다]
```

## 목적

반복되는 업무를 골라 AI가 지켜야 할 역할, 원칙, 금지사항을 문서형 하네스로 고정한다. 최종 결과는 단순 규칙 목록이 아니라 다음 작업에 붙여 넣거나 파일로 저장할 수 있는 `SKILL.md` 초안으로 만든다.

## 권장 시간

25분

## 준비물

- 반복해서 AI에게 맡기고 싶은 업무 1개
- 매번 다시 설명했던 규칙이나 금지사항 목록
- 결과물이 지켜야 하는 출력 형식이 있다면 간단한 예시
- 이 스킬이 실행되어야 하는 상황과 실행되면 안 되는 상황

## Exercise Prompt

반복해서 AI에게 맡기는 업무 하나를 고르고, 다음 대화에서도 같은 출발점과 경계로 시작할 수 있는 `SKILL.md` 초안을 작성한다. 먼저 규칙서 메모로 역할, 원칙, 금지사항을 분리한 뒤, 그 내용을 스킬의 `name`, `description`, `When to Use`, `Workflow`, `Output Contract`, `Guardrails`로 옮긴다.

## Starter Context

아래 질문에 짧게 답한 뒤 규칙서 초안을 시작한다.

- 이 업무를 AI에게 다시 맡길 가능성이 높은가?
- 매번 프롬프트에 다시 적던 역할이나 원칙이 있는가?
- AI가 추측하면 안 되는 정보, 판단, 결론이 있는가?
- 결과물을 검수하기 쉽게 만드는 최소 출력 형식이 있는가?
- 이 문서를 `SKILL.md`로 저장한다면 언제 자동으로 불러와야 하는가?
- 반대로 이 스킬이 개입하면 안 되는 상황은 무엇인가?

## Expected Outcome

완성 산출물은 두 겹으로 구성한다. 첫째, `Role`, `Goal`, `Principles`, `Forbidden`, `Human Judgment`가 분리된 규칙서 메모다. 둘째, 이 메모를 바탕으로 바로 파일화할 수 있는 `SKILL.md` 초안이다. 초안만 읽어도 AI가 맡을 책임, 언제 실행되어야 하는지, 어떤 절차로 일해야 하는지, 어디서 멈추고 사람에게 확인해야 하는지가 보여야 한다.

## 단계별 진행

1. 반복 업무를 하나 고르고 AI에게 맡길 범위를 한 문장으로 쓴다.
2. AI의 역할을 "무엇을 책임지는가" 중심으로 작성한다.
3. 반복해서 지켜야 할 원칙을 3개 이내로 적는다.
4. AI가 추측하지 말아야 할 금지사항과 사람에게 되물어야 할 조건을 분리한다.
5. 이 규칙이 자동으로 적용되어야 하는 상황과 적용되면 안 되는 상황을 나눈다.
6. 규칙서 메모를 `SKILL.md` 구조로 옮긴다: frontmatter, When to Use, Inputs, Workflow, Output Contract, Guardrails, Done Check.
7. 순서 강제, 상태 관리, 재시도가 필요한 업무라면 이 `SKILL.md`만으로 끝내지 않고 뒤의 분업형 또는 MCP형 후보로 표시한다.

## Completion Checks

- 역할, 원칙, 금지사항이 서로 다른 줄이나 섹션에 분리되어 있다.
- AI가 추측하지 말고 질문해야 하는 조건이 1개 이상 적혀 있다.
- 사람이 최종 판단할 영역이 남아 있어 문서가 업무 판단을 과하게 대신하지 않는다.
- `SKILL.md`의 `description`이 언제 이 스킬을 써야 하는지 설명한다.
- `Workflow`가 최소 3단계 이상으로 적혀 있고, `Output Contract`가 검수 가능한 형태다.
- 출력 형식은 검수에 필요한 만큼만 적혀 있고, 불필요하게 긴 설명은 없다.

## 예시

```markdown
# 반복 보고서 초안 문서형 하네스 설계 메모

## Role
- 너의 역할: 흩어진 메모를 의사결정용 보고서 초안으로 정리하는 보조자
- 책임 범위: 입력된 메모를 구조화하고 빠진 질문을 표시한다.

## Goal
- 이 업무의 목표: 검토자가 바로 쟁점과 다음 행동을 볼 수 있게 만든다.
- 완료된 결과의 모습: 배경, 핵심 쟁점, 결정 필요 사항, 확인 질문이 분리된 초안

## Principles
- 원칙 1: 입력에 없는 사실을 확정된 것처럼 쓰지 않는다.
- 원칙 2: 핵심 쟁점은 3개 이내로 먼저 정리한다.
- 원칙 3: 문장은 짧게 쓰고 판단 근거를 함께 남긴다.

## Forbidden
- 하지 말아야 할 일: 없는 수치, 외부 사례, 확정되지 않은 결론을 추가하지 않는다.
- 추측하지 말고 질문해야 할 조건: 목표 독자나 제출 형식이 비어 있을 때

## Human Judgment
- 사람이 결정할 영역: 최종 우선순위와 의사결정 방향
- AI가 제안만 할 영역: 쟁점 후보와 확인 질문

## Output Format
- 배경
- 핵심 쟁점
- 결정 필요 사항
- 확인 질문

## Done Check
- [x] 역할이 분명하다.
- [x] 원칙과 금지사항이 분리되어 있다.
- [x] 사람이 판단할 영역이 남아 있다.

---

---

# 의사결정 보고서 초안 작성

## When to Use

- 사용해야 하는 상황: 회의 메모, 조사 메모, 이슈 목록을 의사결정용 보고서 초안으로 바꿀 때
- 사용하지 말아야 하는 상황: 최종 우선순위나 의사결정 방향을 AI가 확정해야 하는 상황

## Inputs

- 필수 입력: 원문 메모, 목표 독자, 원하는 출력 길이
- 선택 입력: 기존 보고서 양식, 금지 표현, 참고해야 할 내부 기준
- 부족하면 다시 물을 질문: 목표 독자, 제출 형식, 결정해야 할 주제가 비어 있는지 확인한다.

## Workflow

1. 입력 메모에서 배경, 사실, 추정, 질문을 분리한다.
2. 핵심 쟁점 후보를 3개 이내로 정리한다.
3. 입력에 없는 수치, 외부 사례, 확정되지 않은 결론은 추가하지 않는다.
4. 배경, 핵심 쟁점, 결정 필요 사항, 확인 질문 형식으로 초안을 작성한다.
5. 최종 우선순위와 의사결정 방향은 사람 판단 항목으로 남긴다.

## Output Contract

- 배경
- 핵심 쟁점
- 결정 필요 사항
- 확인 질문
- 사람 판단 항목

## Guardrails

- 입력에 없는 사실을 확정된 것처럼 쓰지 않는다.
- 핵심 쟁점은 3개 이내로 제한한다.
- 최종 우선순위와 의사결정 방향을 AI가 단정하지 않는다.

## Done Check

- [x] 배경, 쟁점, 결정 필요 사항, 확인 질문이 분리되어 있다.
- [x] 입력에 없는 사실이 추가되지 않았다.
- [x] 사람이 판단할 항목이 남아 있다.
```

## 완료 기준

- 역할, 목표, 원칙, 금지사항, 사람 판단 영역이 각각 분리되어 있다.
- `SKILL.md` 초안에 `name`, `description`, `When to Use`, `Inputs`, `Workflow`, `Output Contract`, `Guardrails`, `Done Check`가 있다.
- 반복 업무에 다시 붙여 넣거나 파일로 저장해도 같은 출발점과 경계가 보인다.
- AI가 추측하면 안 되는 조건이 명시되어 있다.
- 출력 형식은 필요한 만큼만 고정되어 있고 업무 판단까지 과하게 박제하지 않았다.

## 제출/검토 체크리스트

- [ ] 문서형 하네스가 규칙 저장소로 쓰일 수 있다.
- [ ] 최종 제출물에 `SKILL.md` 초안이 포함되어 있다.
- [ ] `description`만 읽어도 언제 이 스킬을 사용해야 하는지 알 수 있다.
- [ ] 역할과 원칙이 섞이지 않았다.
- [ ] 금지사항과 질문 조건이 구체적이다.
- [ ] 순서, 상태, 재시도 강제가 필요한 업무인지 별도로 표시했다.
- [ ] 새로운 외부 사례나 검증되지 않은 주장을 넣지 않았다.
