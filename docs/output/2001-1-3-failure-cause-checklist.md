최근 AI 작업 실패를 모델 문제가 아니라 구조 문제로 진단한다. 실패 원인을 맥락 소모, 방향 이탈, 검수 부재로 나누고 다음 시도에서 바꿀 구조를 남긴다.

1. 실패한 업무를 한 문장으로 적는다.
2. 원래 원했던 결과와 실제 나온 결과의 차이를 적는다.
3. 실패 원인을 `맥락 소모`, `방향 이탈`, `검수 부재` 중 하나 이상으로 표시한다.
4. 입력, 실행, 검수 중 어느 구간의 구조가 비어 있었는지 확인한다.
5. 다음 시도에서 추가할 구조를 한 줄씩 작성한다.

## 예시

```markdown
# 실패 원인 체크리스트

## 1. 실패 사례
- 업무: 반복 보고서 초안 작성
- 원했던 결과: 의사결정에 필요한 핵심 쟁점이 보이는 1페이지 초안
- 실제 결과: 문장은 길지만 무엇을 결정해야 하는지 흐려진 초안
- 어긋난 지점: 목표와 완료 기준이 대화 중간에 바뀌었다.

## 2. 원인 분류
- [x] 맥락 소모: 자료 탐색, 초안 생성, 수정 요청이 한 세션에 쌓였다.
- [x] 방향 이탈: "의사결정용"이라는 목표가 명확한 성공 기준으로 고정되지 않았다.
- [x] 검수 부재: 중간에 핵심 쟁점만 따로 확인하는 단계가 없었다.

## 3. 구조 진단
| 구간 | 비어 있던 구조 | 다음에 추가할 구조 |
|---|---|---|
| 입력 | 목적과 완료 기준 | "의사결정자에게 필요한 쟁점 3개"로 성공 기준 고정 |
| 실행 | 탐색과 작성 분리 | 자료 탐색은 보조 세션, 최종 판단은 메인 세션에 남김 |
| 검수 | 중간 확인 | 초안 전 쟁점 목록만 먼저 검토 |

## 4. 다음 시도 규칙
- 규칙: 목표, 제약, 완료 기준을 먼저 한 줄로 적는다.
- 분리할 역할: 탐색은 보조 세션으로 보내고 결정은 메인 세션에 남긴다.
- 확인 지점: 본문 작성 전에 쟁점 목록 3개를 확인한다.
```

## 완료 기준

- 최근 실패 사례가 1개 이상 구체적으로 적혀 있다.
- 실패 원인이 맥락 소모, 방향 이탈, 검수 부재 중 하나 이상으로 분류되어 있다.
- 입력, 실행, 검수 중 비어 있던 구조가 표시되어 있다.
- 다음 시도에서 바꿀 규칙, 분리할 역할, 확인 지점이 각각 한 줄 이상 작성되어 있다.

## 제출/검토 체크리스트

- [ ] 실패를 "AI가 못했다"로 끝내지 않고 구조의 빈칸으로 다시 설명했다.
- [ ] 다음 시도에서 바로 붙여 넣거나 확인할 수 있는 문장이 있다.
- [ ] 체크리스트만 봐도 같은 실패를 반복하지 않기 위한 조치가 보인다.
- [ ] 불필요한 외부 사례나 검증되지 않은 수치를 넣지 않았다.
