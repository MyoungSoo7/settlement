# 2001-3-3. 메인 세션 보호용 역할 분리표 만들기 + 기존 스킬에 적용

## Practice Task and Prompt

- Task: 학습자가 이미 가진 스킬 1개를 골라, `세션 역할 분리표`를 작성하고 그 결과를 바탕으로 그 스킬을 메인/보조(sub agent) 역할이 분리된 형태로 같이 고친다.
- 준비: 학습자가 이미 등록해 쓰고 있는 스킬 1개 (예: `~/.claude/skills/<skill-name>/SKILL.md`). 없으면 가장 자주 쓰는 슬래시 커맨드/프롬프트 묶음 1개로 대체한다.
- Prompt to paste into AI:

```text
Fast Campus 하네스 엔지니어링 실습 2001-3-3을 도와줘.

Curriculum mapping:
- Module: 1-3. 분업형 하네스 기초
- Clip: [실습] 메인 세션 보호용 역할 분리표 만들기
- Key message: 메인/보조 세션 역할을 먼저 나눈다
- Required output:
  1) 세션 역할 분리표
  2) 내가 이미 쓰고 있는 스킬 1개를 메인/보조 역할이 분리된 형태로 고친 변형안

Interview mapping:
- Use the interview framing that many AI agents without a harness diverge, while a main session should keep decisions and disposable side work should return only useful handoff context.

Task:
1. 내가 알려주는 기존 스킬의 SKILL.md 경로를 먼저 읽고, 그 스킬이 한 대화에서 어떤 일들을 섞어서 처리하는지 짧게 요약해줘.
2. 그다음 AskUserQuestion 도구로 다음을 물어봐:
   - 메인 세션이 끝까지 붙잡아야 할 정보(목표, 제약, 결정)
   - sub agent로 가장 먼저 분리하고 싶은 일 (탐색/리서치, 초안 생성·후보 비교, 독립 검수/리뷰, 반복 조회/도구 호출 중 다중 선택)
   - sub agent에서 돌려받을 handoff 형식 (요약+근거+선택지+실패조건 권장)
   - 메인 세션이 절대 보조로 넘기면 안 되는 최종 판단
   질문은 AskUserQuestion으로 한 번에 묶어 보내고, 모르는 항목은 추측하지 말고 "확인 필요"로 둔다.
3. 사용자의 답을 받은 뒤,
   - `세션 역할 분리표`를 아래 작성 템플릿으로 채우고
   - 같은 답을 근거로 기존 SKILL.md의 변형안을 제안한다. 변형안은 (a) 메인이 유지할 책임, (b) sub agent로 위임할 단계와 호출 시점, (c) handoff 포맷, (d) 새로 추가/삭제할 문장 패치 형태로 보여준다.
4. 변경은 사용자가 승인할 때까지 실제 파일에 쓰지 않는다.

Boundaries:
- curriculum.md와 interview.md의 범위를 벗어난 외부 사례를 추가하지 않는다.
- 검증되지 않은 수치, 가격, 할인, 마케팅성 주장을 넣지 않는다.
- 최종 책임 판단이 필요한 항목은 사람이 결정할 수 있게 분리한다.
- 학습자의 스킬 내용을 외부로 전송하거나 요약 외 형태로 변형해 저장하지 않는다.

My input:
- 기존 스킬 경로: [예: ~/.claude/skills/report-1pager/SKILL.md]
- 그 스킬이 한 대화에서 섞어 처리하는 일: [짧게]
- 메인 세션이 잃으면 안 되는 정보: [짧게]
- (선택) 추가 제약/맥락: [붙여넣기]
```

## 목적

메인 세션에 모든 일을 몰아넣지 않도록 메인 세션, sub agent(보조 세션), MCP/도구, handoff의 역할을 나눈다. 메인 세션은 목표·제약·현재 상태·결정 로그를 지키고, sub agent는 탐색·초안·비교·독립 검수처럼 버려도 되는 기억으로 처리한다. 그리고 그 원리를 **이미 사용 중인 스킬 1개에 실제로 적용**해서, 다음 실행부터 같은 흐름이 자동으로 굴러가게 만든다.

## 권장 시간

30분 (분리표 15분 + 스킬 변형안 15분)

## 준비물

- 본인이 이미 쓰는 스킬 1개의 `SKILL.md` 경로 (예: `~/.claude/skills/<name>/SKILL.md`). 없으면 자주 쓰는 슬래시 커맨드/프롬프트 묶음 1개.
- 그 스킬이 한 대화에서 섞어 처리하는 일 목록 (탐색, 초안, 비교, 검수, 실행 등)
- 메인 세션에 반드시 남아야 하는 목표·제약·현재 상태·결정 목록
- sub agent에서 결과를 받을 때 필요한 요약 형식 후보

## Exercise Prompt

본인이 이미 쓰고 있는 스킬을 그대로 두고 한 번 돌려보면, 메인 세션 안에서 탐색·초안·비교·검수가 어디서 섞이는지 보일 것이다. 그 지점을 끊어서 sub agent로 넘기는 변형안을 만들고, sub agent의 긴 대화 전문이 아니라 "요약+근거+선택지+실패조건" 형태의 handoff만 메인으로 회수하도록 스킬 본문을 수정한다.

## Starter Context

아래 질문에 짧게 답한 뒤 역할 분리표와 스킬 변형안을 작성한다. (AI가 AskUserQuestion으로 다시 물을 항목들이다.)

- 내가 적용할 기존 스킬은 무엇인가? (`SKILL.md` 경로)
- 그 스킬이 한 대화에서 어떤 일들을 섞어서 처리하고 있는가?
- 메인 세션이 잃으면 안 되는 목표·제약·현재 상태·결정은 무엇인가?
- 탐색/리서치, 초안 생성·후보 비교, 독립 검수/리뷰, 반복 조회/도구 호출 중 sub agent로 가장 먼저 분리하고 싶은 일은?
- sub agent에서 돌아올 결과는 어떤 형식이어야 다음 결정을 바로 할 수 있는가? (기본: 요약+근거+선택지+실패조건)
- 메인이 절대 sub agent에 넘기면 안 되는 최종 판단은 무엇인가?

## Expected Outcome

다음 두 산출물이 함께 나온다.

1. `메인 세션 / sub agent / MCP·도구 / handoff` 책임이 분리된 **세션 역할 분리표**.
2. 그 분리표를 적용한 **기존 스킬 변형안**: 메인이 유지할 책임, sub agent로 위임할 단계와 호출 시점, handoff 포맷, SKILL.md에 들어갈 추가/삭제 문장 패치.

표만 읽어도 무엇이 메인에 남고 무엇이 sub로 가는지 보여야 하고, 변형안만 보고도 다음 실행에서 어디서 sub agent를 호출할지 사람이 따라 할 수 있어야 한다.

## 단계별 진행

1. 적용할 스킬의 `SKILL.md`를 읽고, 한 대화에서 어떤 일이 섞여 있는지 한 문단으로 요약한다.
2. 메인 세션이 지켜야 할 목표·제약·현재 상태·결정 로그를 적는다.
3. 탐색·초안·비교·독립 검수처럼 맥락을 많이 써도 버릴 수 있는 일을 sub agent로 분리한다. (Claude Code의 Agent/서브에이전트 호출, Task 위임 등 실제 호출 경로를 같이 메모한다.)
4. 반복 조회·실행·상태 확인처럼 사람이 매번 기억하기 어려운 일은 MCP/도구 후보로 표시한다.
5. sub agent에서 돌아올 handoff 형식을 `요약 / 근거 / 선택지 / 실패 조건 / 메인 결정 요청`으로 정리한다.
6. 각 역할 옆에 기대 효과와 버릴 정보를 함께 적는다.
7. 위 결과를 바탕으로 기존 SKILL.md의 변형안을 만든다. 새로 추가할 섹션 후보:
   - "메인이 유지할 책임" (목표·제약·결정만)
   - "sub agent 호출 지점" (어떤 단계에서, 어떤 입력으로)
   - "Handoff 포맷" (요약+근거+선택지+실패조건)
   - "메인이 위임하지 않는 최종 판단"
8. sub agent의 긴 대화 전문을 그대로 메인 세션에 넣지 않아도 다음 결정을 할 수 있는지 변형안을 한 번 시뮬레이션한다.
9. 마지막에 "메인/보조 세션 역할을 먼저 나눈다"는 핵심 문장이 분리표와 변형안 양쪽에 드러나는지 확인한다.

## 예시 (스킬 변형안 포함)

```markdown
# 세션 역할 분리표 + 스킬 변형안

## 적용 대상 스킬
- 스킬 이름/경로: ~/.claude/skills/report-1pager/SKILL.md
- 현재 한 대화에 섞이는 일: 자료 탐색, 쟁점 후보 생성, 임원 예상 질문 도출, 1페이지 본문 작성, 자체 검수
- 완료 기준: 임원이 1페이지로 의사결정할 수 있는 보고서 + 쟁점 3 + 예상 질문 3

## 보호할 메인 맥락
- 목표: 반복 주기로 진행 현황을 1페이지로 전달
- 제약: 외부 공개용 카피·검증되지 않은 수치 금지
- 현재 상태: 자료 수집은 끝났고 쟁점 후보 단계
- 결정 로그: 독자는 마케팅 임원, 톤은 보고체, 분량 A4 1쪽 고정
- 메인이 위임하지 않는 최종 판단: 어떤 쟁점 3개를 본문에 올릴지 / 임원 질문 3개의 채택 여부

## 역할 분리
| 구분 | 맡길 일 | 남길 정보 | 버릴 정보 | 기대 효과 |
|---|---|---|---|---|
| 메인 세션 | 목표·제약·결정·쟁점 채택 | 독자, 톤, 결정 로그, 채택된 쟁점/질문 | 자료 탐색 중 긴 대화 | 핵심 맥락 보호 |
| sub agent | 자료 요약, 쟁점 후보 생성, 임원 예상 질문 후보, 본문 초안 검수 | 후보안, 장단점, 빠진 질문 | 실패한 후보 생성 과정 | 병행 탐색·검수 분리 |
| MCP/도구 | 깃 로그/티켓 조회, 파일 상태 확인 | 확인 결과, 실패 여부 | 실행 로그 전문 | 사람이 반복 기억할 일 감소 |
| handoff | sub 결과를 메인에 전달 | 요약, 근거, 선택지, 실패 조건 | sub 대화 전문 | 버릴 기억과 남길 기록 분리 |

## Handoff Format
- 현재 목표: 1페이지 보고서용 쟁점 3 채택
- sub agent가 한 일: 쟁점 후보 5개 생성 + 임원 예상 질문 5개 매칭
- 근거: 최근 2주 커밋·티켓·회의록 요약
- 선택지: 쟁점 후보 A/B/C/D/E (각 1줄)
- 실패 조건: 검증되지 않은 수치/가격/마케팅성 주장이 후보에 섞임
- 메인 세션이 결정해야 할 것: 쟁점 3개 채택 + 본문 톤 확정

## 스킬 변형안 (SKILL.md 패치)
- 추가할 섹션:
  - "메인이 유지할 책임": 독자·톤·분량·쟁점 채택·최종 본문 결정
  - "sub agent 호출 지점":
    1) 자료 요약 단계 → sub에 위임, handoff만 회수
    2) 쟁점/질문 후보 생성 단계 → sub에 위임, 채택은 메인
    3) 본문 초안 자체 검수 → 별도 sub에 위임 (편향 분리)
  - "Handoff 포맷": 요약 / 근거 / 선택지 / 실패 조건 / 메인 결정 요청
  - "메인이 위임하지 않는 최종 판단": 쟁점 3개 확정, 임원 질문 3개 확정, 본문 최종 결재
- 수정/삭제할 문장: 본문에서 "한 번에 자료 정리부터 검수까지 수행" 류의 문구는 "sub agent에 위임하고 handoff로 회수"로 교체
- Trigger 문구는 그대로 두는가? 예 — 트리거는 사용자 진입점이고 분업은 본문 흐름에서만 적용한다.

## 다음 실행 규칙
- 메인에 남길 것: 독자, 톤, 분량, 결정 로그, 채택된 쟁점/질문
- sub agent로 보낼 것: 자료 요약, 쟁점/질문 후보 생성, 본문 자체 검수
- handoff로만 받을 것: 요약, 근거, 선택지, 실패 조건
```

## Completion Checks

- 메인 세션이 지켜야 할 정보와 sub agent로 보낼 일이 서로 섞이지 않았다.
- sub agent의 결과는 긴 대화 전문이 아니라 handoff 형식으로 돌아온다.
- MCP/도구는 필요할 때만 후보로 표시했고 모든 업무의 기본값으로 두지 않았다.
- 분리표 결과가 실제 SKILL.md 변형안의 추가/삭제 문장으로 이어졌다.
- 다음 실행 때 어디서 sub agent를 호출할지 사람이 따라 할 수 있는 규칙이 남아 있다.

## 완료 기준

- 메인 세션, sub agent, MCP/도구, handoff의 책임이 각각 적혀 있다.
- 메인 세션에 남겨야 할 목표·제약·결정·현재 상태가 보인다.
- sub agent로 보낼 일과 그 기대 효과가 분리되어 있다.
- handoff에 요약·근거·선택지·실패 조건 중 필요한 항목이 포함되어 있다.
- 기존 스킬의 SKILL.md 변형안에 "메인 책임 / sub 호출 지점 / handoff 포맷 / 위임 금지 판단" 4개 섹션이 들어 있다.
- 사용자 승인 전에는 SKILL.md 파일을 실제로 수정하지 않는다.

## 제출/검토 체크리스트

- [ ] 메인 세션이 모든 일을 직접 처리하지 않는다.
- [ ] sub agent의 결과를 버려도 메인 세션의 결정 맥락은 남는다.
- [ ] handoff만 읽어도 다음 결정을 이어갈 수 있다.
- [ ] MCP/도구가 필요한 일과 필요하지 않은 일이 구분되어 있다.
- [ ] AskUserQuestion으로 받은 답이 분리표와 SKILL.md 변형안 양쪽 근거로 쓰였다.
- [ ] 변형안은 추가/삭제 문장 단위 패치로 제시되었고, 실제 파일 수정은 사용자 승인 뒤로 미뤘다.
- [ ] 새로운 외부 사례, 검증되지 않은 수치, 가격 또는 마케팅성 정보를 넣지 않았다.
