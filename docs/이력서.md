## 한 줄 소개

**주문·결제·정산을 헥사고날 아키텍처 기반 MSA로 분리하고, 분산 트랜잭션 없이 Outbox+Kafka 멱등 메시징으로 데이터 정합성을 보장한 금융 도메인 백엔드.**

## 핵심 기여 (이력서 bullet)

- **모놀리스 → MSA 분리 + 코드 경계 강제**: order-service/settlement-service를 분리하고, **Read-only Projection 패턴**(`@Immutable` 엔티티로 동일 테이블 read 매핑)으로 두 서비스 간 **코드 의존성을 0으로 유지**. 의존 방향은 **ArchUnit 테스트로 컴파일 단계에서 강제**.

- **분산 트랜잭션 없이 데이터 정합성 확보**: 결제→정산 전파를 **Transactional Outbox + Kafka**로 구현하고, `outbox_events.event_id(UNIQUE)` → `processed_events(PK)` → `settlements.payment_id(UNIQUE)` **3단 멱등 방어**로 메시지 중복·재처리에도 정산 1건을 보장(정확히 한 번 효과).

- **Outbox 발행 처리량 개선**: 단일 인스턴스 직렬 폴링(ShedLock)을 **`SELECT … FOR UPDATE SKIP LOCKED` claim 기반 다중 워커 배치 비동기 발행**으로 전환. 워커 간 무경합 작업 분배 + 리스(claim) 만료 자동 회수로 워커 장애에도 유실 없음. **Testcontainers 동시성 통합테스트로 disjoint 분배 검증**.

- **DB 인덱스 최적화로 풀스캔 제거**: 리포지토리 쿼리 패턴과 기존 인덱스를 전수 대조해, Seq Scan/불필요 정렬이 발생하던 지점에 **복합·부분 인덱스(B+Tree) 보강**(예: `(status, created_at)` partial). 과도한 인덱스는 배제하고 근거 있는 항목만 추가.

- **검색·집계용 Elasticsearch 프로젝션**: Settlement·Order·Payment·Refund를 **비정규화 통합 문서**로 색인(한글 **Nori** 형태소 분석, 금액 `scaled_float`로 부동소수 오차 제거). **이벤트 기반 비동기 색인 + 실패 시 DB 재시도 큐**로 ES 장애에도 색인 유실 방지. 정합성=RDB / 검색=ES **CQRS 읽기 분리**.

- **동시성·정합성 설계**: 상황별 락 전략 적용 — 정산/지급 **낙관적 락(@Version)**, 환불 **비관적 락 + Idempotency-Key**(중복 환불 차단), 재고 차감 **원자적 조건부 UPDATE**(초과판매 방지, **동시 100스레드 IT로 검증**).

- **관측성 3축 + AOP 횡단 추적**: Micrometer/Prometheus 메트릭, `traceId` MDC 주입(결제→정산→환불 체인 추적), OTel 분산 트레이싱(**Outbox가 `traceparent` 영속화 → 비동기 경계에서도 trace 단절 없음**). **Spring AOP로 로깅/실행시간/트랜잭션 경계를 헥사고날 레이어에 횡단 적용**.

- **회복탄력성·보안**: PG 연동에 **Resilience4j 서킷브레이커**(외부 장애 격리), `@Scheduled` 분산 락(**ShedLock**), **Bucket4j 토큰버킷 Rate Limiting**, JWT(HS256)·BCrypt(cost 12)·PII 마스킹·감사 로그.

## 트러블슈팅 / 문제 해결 사례 (면접 어필)

| 문제 | 원인 | 해결 |
|------|------|------|
| 서비스 간 강결합 위험 | settlement이 order 코드를 직접 참조하면 MSA 경계 붕괴 | **Read-only Projection**(동일 테이블 read 매핑) + ArchUnit으로 의존 방향 강제 |
| 결제 이벤트 중복 처리 | 메시지 재전송/컨슈머 재시작 시 정산 중복 생성 | **3단 멱등 방어**(UNIQUE + 멱등 PK + 멱등 키) |
| Outbox 단일 폴러 병목 | ShedLock으로 1 인스턴스만 직렬 발행 | **SKIP LOCKED claim 다중 워커 배치 발행**으로 병렬화 |
| 카테고리 하위탐색 N+1 | 노드마다 `findByParentId` 재귀 호출 | 전체 1회 로드 + **인접맵 + Queue(BFS)** 로 쿼리 N+1 → 1 |
| 인덱스 없는 풀스캔 | 정렬·필터 컬럼에 인덱스 부재 | 쿼리↔인덱스 전수 대조 후 **B+Tree 복합/부분 인덱스** 보강 |
| 금액 계산 오차 | 부동소수 사용 시 통화 반올림 오류 | 전 구간 **`BigDecimal` + HALF_UP scale 2**, ES는 `scaled_float` |

## 아키텍처 하이라이트

- **헥사고날(Ports & Adapters)**: 도메인=순수 POJO, 애플리케이션=포트 인터페이스 의존, 어댑터(web/kafka/persistence/search/pdf)가 바깥. 인프라 교체·테스트가 자유롭고 ArchUnit으로 경계 강제.
- **복식부기 원장(Double-Entry Ledger)**: 차변/대변 쌍으로 회계 항등식 유지, 역분개로 정정 — 금융 정합성.
- **DB-backed 큐**: Outbox·인덱스 재시도·원장 발행을 테이블 큐로 구현(재기동·다중워커에도 유실 없음).
- **품질 게이트**: JaCoCo 커버리지(핵심 도메인 INSTRUCTION 80% 강제) + ArchUnit + Testcontainers 통합/동시성 테스트.

## 한 줄 정리
