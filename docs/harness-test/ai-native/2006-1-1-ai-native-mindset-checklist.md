## 예시

```markdown
# AI Native 마인드셋 체크리스트

## 1. 업무와 결과
| 항목 | 내용 |
|---|---|
| 점검할 업무 | 반복 고객 문의 답변 초안 작성 |
| 최종 결과 | 담당자가 검토할 답변 초안과 판단 근거 |
| 비즈니스 임팩트 | 누락 없는 답변과 빠른 검토 |
| 현재 AI 사용 방식 | 초안 작성에만 사용하고 검수 기준은 매번 다시 설명한다 |

## 2. 코드가 아닌 파이프라인
| 단계 | 현재 방식 | AI가 맡을 일 | 사람이 판단할 일 | 산출물 |
|---|---|---|---|---|
| 입력 | 문의 원문만 붙인다 | 문의 의도와 제약을 분리한다 | 답변 정책이 충분한지 판단한다 | 입력 요약표 |
| 실행 | 초안을 바로 쓴다 | 정책 기준으로 답변 초안을 만든다 | 민감한 표현을 허용할지 판단한다 | 답변 초안 |
| 검수 | 담당자가 눈으로 확인한다 | 누락, 금지 표현, 남은 질문을 표시한다 | 발송 여부를 결정한다 | 검수 결과 |
| 공유 | 개인 메모로 남긴다 | 반복 규칙을 템플릿으로 정리한다 | 팀 규칙으로 둘지 정한다 | 공유 체크리스트 |

## 3. 검수와 운영 기준
| 기준 | 내용 |
|---|---|
| 성공기준 | 문의 의도, 정책, 남은 질문이 모두 보인다 |
| 실패 신호 | 정책에 없는 약속을 하거나 확인 질문 없이 추측한다 |
| 중단 기준 | 답변 정책이 없거나 담당자 판단이 필요한 민감 사안이다 |
| 재질문 기준 | 원문에서 요청, 제약, 기대 결과가 분리되지 않는다 |
| 사람이 최종 책임질 판단 | 실제 고객에게 발송할지 여부 |

## 4. 팀으로 넘길 항목
- 공유할 하네스/규칙: 문의 답변 입력 요약표와 검수 기준
- 공유할 실행 기록: 누락 항목과 재질문 사유
- 팀과 맞출 검수 기준: 금지 표현과 최종 승인 책임
- 다음 실습으로 넘길 요구사항: 공유, 관측, 리뷰 방식

## 5. 다음 행동
- 다음 업무에서 바로 바꿀 행동: 초안 작성 전 입력 요약표를 먼저 만든다.
- 바꾸지 않을 것: 최종 발송 판단은 담당자에게 남긴다.
- 확인할 사람 또는 리뷰 방식: 팀 리드가 검수 기준을 한 번 확인한다.
```
