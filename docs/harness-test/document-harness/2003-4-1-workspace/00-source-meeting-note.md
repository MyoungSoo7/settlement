# 00. 샘플 회의록 원문

> 4-1 실습 사슬 전체에서 같은 입력으로 쓰는 회의록. 1-3·2-3·3-3·4-1·4-3 모두 이 원문을 척추로 삼는다.

## 회의 정보
- 일시: 2026-05-22 (목) 14:00 ~ 15:10
- 참석: 박PM, 김디자이너, 이FE, 정BE, 한CS
- 작성: 박PM
- 목적: 사내 비개발자 교육 신청 페이지 개편 방향 정리

## 회의록 원문

박PM: 지난 분기 사내 교육 신청 페이지에서 신청 후 환불·일정 변경 문의가 CS로 너무 많이 들어왔다. 페이지 자체를 다시 정리해야 한다.

한CS: 이번 분기 문의 중 60% 정도가 일정 변경 가능한지, 환불 기준이 뭔지를 묻는 문의다. 정확한 수치는 다시 뽑아 와야 한다. 신청 직후에 안내가 부족하다는 인상은 분명하다.

김디자이너: 신청 페이지에 일정·환불 안내가 작은 글씨로 페이지 맨 아래에 있어서 사실상 안 읽힌다. 위치를 위로 올리거나 신청 직전에 한 번 더 강조해야 한다.

이FE: 디자인 시안이 확정되면 작업은 가능하다. 다만 결제 모듈은 외부 SDK를 쓰고 있어서 신청 직전 모달 위치는 우리 마음대로 못 옮긴다.

정BE: 신청 후 자동 안내 메일에는 일정 변경·환불 기준을 넣을 수 있다. 메일 템플릿을 손보는 작업으로 처리 가능하다.

박PM: 이번 개편 목표는 "신청 직후 환불·일정 문의를 줄인다"로 좁히자. 페이지 전체 리뉴얼은 이번 범위가 아니다.

한CS: 그러면 신청 직전에 핵심 정보를 한 번 더 보여 주는 화면과, 자동 안내 메일에 환불·일정 기준을 명확히 적는 두 가지가 우선 같다.

김디자이너: 신청 직전 강조 화면 시안은 다음 주 화요일까지 만들 수 있다.

정BE: 안내 메일 템플릿 수정은 시안 확정 후 3일이면 된다.

박PM: 일정과 담당자 최종 확정은 다음 주 월요일 정기 미팅에서 다시 잡는다. 오늘은 방향만 합의한다.

한CS: 한 가지 더, 환불 기준 자체가 명문화돼 있지 않다. 운영팀과 별도로 확인이 필요하다.

박PM: 그건 이번 페이지 개편 범위 밖이고, 운영팀과 따로 확인하자.

## 회의에서 확정된 내용
- 개편 목표: 신청 직후 환불·일정 문의를 줄인다.
- 이번 범위에 들어가는 것: (1) 신청 직전 핵심 정보 강조 화면, (2) 자동 안내 메일에 환불·일정 기준 추가.
- 이번 범위에 들어가지 않는 것: 페이지 전체 리뉴얼, 결제 모듈 모달 위치 변경, 환불 기준 자체 명문화.

## 회의에서 결정되지 않은 내용
- 최종 담당자와 일정 (월요일 정기 미팅에서 확정).
- CS 문의 60% 수치의 정확한 출처.
- 환불 기준 명문화는 운영팀 별도 확인 필요.
