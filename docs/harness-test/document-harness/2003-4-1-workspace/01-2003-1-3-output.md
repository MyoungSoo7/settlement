# 01. 문서 하네스 체크리스트 (2003-1-3 산출물)

> 입력: `00-source-meeting-note.md`. 회의록 → 1page 기획안 업무를 대상으로 문서로 고정할 것과 사람이 남겨 둘 판단을 나눈다.

## 입력표

| 항목 | 내용 |
|---|---|
| 대상 업무 | 사내 교육 신청 페이지 개편 회의록을 1page 기획안 초안으로 정리 |
| 업무 목적 | 회의에서 합의된 방향과 결정되지 않은 항목을 한 페이지에서 동시에 보이게 한다 |
| AI가 맡을 역할 | 회의록과 확인된 요구사항만으로 1page 기획안 초안을 구조화 |
| 반복해서 지킬 목표 | 배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목을 매번 같은 칸으로 분리 |
| 금지해야 할 추측 | 회의록에 없는 수치(예: 60%), 확정되지 않은 일정·담당자, 환불 기준 자체 |
| 사람이 판단할 영역 | 최종 일정·담당자 확정, 환불 기준 명문화 여부, 페이지 전체 리뉴얼 범위 결정 |
| 문서만으로 부족할 수 있는 신호 | 확인 질문 답변이 여러 차례 오갈 때 상태 추적, 입력 누락 시 재질문 |

# 문서 하네스 체크리스트

## 대상 업무
- 업무명: 사내 교육 신청 페이지 개편 회의록을 1page 기획안 초안으로 바꾸기
- 입력: 회의록 원문(`00-source-meeting-note.md`)과 확인된 요구사항
- 출력: 제목·배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목이 있는 1page 기획안 초안
- 업무 목적: 합의된 방향과 결정되지 않은 항목을 한 페이지에서 같이 보이게 한다.

## 문서로 고정할 것
| 구분 | 고정할 내용 | 이유 |
|---|---|---|
| 역할 | 회의록과 확인된 요구사항 안에서 1page 기획안 초안을 구조화하는 보조자 | AI의 책임 범위를 좁힌다 |
| 목표 | 배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목을 분리한다 | 출력 기준을 매번 같이 맞춘다 |
| 금지사항 | 회의록에 없는 수치·일정·담당자·환불 기준을 추가하지 않는다 | 검증 안 된 내용을 결론처럼 쓰지 않는다 |
| 출력 형식 | 제목·배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목 7칸 | 사람이 검수하기 쉽다 |
| 예시 기준 | 짧은 항목형 문장, 회의록 표현을 그대로 인용 | 톤과 길이를 맞춘다 |

## 문서로 고정하지 않을 것
| 구분 | 남겨둘 판단 | 사람이 봐야 하는 이유 |
|---|---|---|
| 예외 | "CS 문의 60%"처럼 수치 출처가 확정되지 않은 항목 | 운영팀 확인이 먼저 필요하다 |
| 창의적 선택 | 해결 방향의 최종 표현과 강조점 | 의사결정자 톤이 반영돼야 한다 |
| 도메인 판단 | 결제 모듈 모달 위치 변경 가능 여부, 환불 기준 명문화 | 외부 SDK·운영팀 조건이 필요하다 |
| 추가 질문 | 결정권자·일정·제외 범위가 비어 있는 경우 | AI가 임의로 채우면 안 된다 |

## 문서 밖 실행 구조 후보
- 순서 강제가 필요한 부분: 회의록 원문 보존 → 확정 내용 분리 → 미결정 항목 분리 → 1page 칸으로 옮기기.
- 상태 추적이 필요한 부분: 확인 질문에 대한 답변이 회차별로 오갈 때.
- 실패 후 재질문 또는 재시도가 필요한 부분: "반드시 보존할 표현"이 누락된 출력이 나왔을 때.
