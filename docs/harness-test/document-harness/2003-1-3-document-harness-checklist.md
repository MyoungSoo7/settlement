# 2003-1-3. 문서형 하네스 설계 체크리스트 만들기

## 이 산출물
반복 업무에서 문서로 고정할 것과 사람이 남겨 둘 판단을 한 장으로 나눈 체크리스트이며, `2003-2-3` 6칸 md 템플릿의 입력이 된다.

## 목적

문서형 하네스가 맡을 범위를 먼저 정한다. 반복해서 고정할 역할·목표·금지사항과 사람이 남겨 두어야 할 판단 영역을 분리해 한 장으로 만든다.

## 권장 시간

25분

## 준비물

- 반복해서 AI에게 맡기고 싶은 업무 1개
- 인터뷰하네스 실습에서 만든 한 줄 명세 또는 업무 목적 문장 1개
- 매번 다시 설명했던 역할·목표·금지사항 목록
- 사람이 직접 판단해야 하는 예외·창의적 선택·도메인 감각 목록

## 입력

아래 입력을 먼저 채운다.

| 항목 | 내용 |
|---|---|
| 대상 업무 |  |
| 업무 목적 |  |
| AI가 맡을 역할 |  |
| 반복해서 지킬 목표 |  |
| 금지해야 할 추측 |  |
| 사람이 판단할 영역 |  |
| 문서만으로 부족할 수 있는 신호 |  |

## AI에게 시키기

위 입력표를 채워 그대로 붙인 뒤, 한 줄을 덧붙인다.

> 이 입력으로 문서 하네스 체크리스트를 작성해줘. 아래 작성 템플릿 구조를 따르고, 모르는 값은 추측하지 말고 "확인 질문"으로 남겨. 입력에 없는 사실이나 검증 안 된 수치는 넣지 마.

## 단계별 진행

1. 대상 업무를 하나 고르고 문서형 하네스가 받을 입력과 만들 출력을 한 문장으로 쓴다.
2. AI가 맡을 역할을 "무엇을 책임지는가" 중심으로 적는다.
3. 반복해서 지켜야 하는 목표와 금지사항을 분리한다.
4. 사람이 판단해야 하는 예외·창의적 선택·도메인 감각을 별도 칸에 남긴다.
5. 각 항목을 "문서로 고정할 것"과 "문서로 고정하지 않을 것" 두 열로 옮긴다.
6. 순서 강제, 상태 추적, 실패 후 재시도가 필요한 항목은 문서 밖 실행 구조 후보로 표시한다.
7. 체크리스트를 읽었을 때 문서형 하네스의 경계가 한눈에 보이는지 확인한다.

## 예시

```markdown
# 문서 하네스 체크리스트

## 대상 업무
- 업무명: 회의록을 1page 기획안으로 바꾸기
- 입력: 회의록 원문과 확인된 요구사항
- 출력: 1page 기획안 초안
- 업무 목적: 회의 내용을 결정 가능한 기획안 구조로 정리한다.

## 문서로 고정할 것
| 구분 | 고정할 내용 | 이유 |
|---|---|---|
| 역할 | 회의록을 읽고 기획안 초안을 구조화하는 보조자 | AI의 책임 범위를 좁힌다 |
| 목표 | 배경, 문제, 제안, 남은 질문을 분리한다 | 출력 목적을 반복해서 맞춘다 |
| 금지사항 | 회의록에 없는 사실과 확정되지 않은 결론을 추가하지 않는다 | 추측을 막는다 |
| 출력 형식 | 제목, 배경, 문제, 제안, 확인 질문 | 매번 같은 검수 기준을 만든다 |
| 예시 기준 | 짧은 항목형 문장으로 쓴다 | 길이와 톤을 맞춘다 |

## 문서로 고정하지 않을 것
| 구분 | 남겨둘 판단 | 사람이 봐야 하는 이유 |
|---|---|---|
| 예외 | 회의에서 충돌한 의견의 최종 우선순위 | 담당자가 결정해야 한다 |
| 창의적 선택 | 제안의 최종 표현과 강조점 | 맥락에 따라 달라진다 |
| 도메인 판단 | 실제 실행 가능성 판단 | 현장 정보가 필요하다 |
| 추가 질문 | 결정권자, 일정, 제외 범위가 비어 있는 경우 | AI가 임의로 채우면 안 된다 |

## 문서 밖 실행 구조 후보
- 순서 강제가 필요한 부분: 회의록 원문 보존 후 추출해야 한다.
- 상태 추적이 필요한 부분: 확인 질문 답변이 여러 번 오가면 별도 관리가 필요하다.
- 실패 후 재질문 또는 재시도가 필요한 부분: 핵심 입력이 비어 있으면 다시 물어야 한다.

## 적용 범위 점검
- [x] 역할·목표·금지사항이 분리되어 있다.
- [x] 판단과 창의성을 전부 문서에 고정하지 않았다.
- [x] 문서만으로 부족한 신호가 표시되어 있다.
```

## 완료 기준

- 문서 하네스 체크리스트가 작성되어 있다.
- 역할·목표·금지사항·출력 형식이 "문서로 고정할 것"에 들어 있다.
- 예외·창의적 선택·도메인 판단·추가 질문이 "문서로 고정하지 않을 것"에 들어 있다.
- 순서·상태·재시도처럼 문서만으로 약한 신호가 별도로 표시되어 있다.
- 문서형 하네스가 맡을 범위와 사람이 남겨 둘 판단이 동시에 보인다.

## 제출/검토 체크리스트

- [ ] 문서형 하네스의 범위를 먼저 정한다는 목적이 보인다.
- [ ] 고정할 규칙과 남겨둘 판단이 두 열로 분리되어 있다.
- [ ] 금지사항이 목표나 원칙과 섞이지 않았다.
- [ ] 사람이 판단할 영역을 AI가 임의로 채우지 않도록 남겼다.
- [ ] 새로운 외부 사례, 검증되지 않은 수치, 가격 또는 마케팅성 정보를 넣지 않았다.
