# 2002-3-3. Seed v1 → v2 — evolve_step 손 버전

## Practice Task and Prompt

- Task: 2002-2-3에서 떨어진 **Seed YAML(goal + 제약 + 성공기준)을 입력**으로 받아, AI에 사각지대를 묻고, 채택된 사각지대를 반영해 **Seed v2**를 만든다. Ouroboros의 evolve_step을 손으로 한 번 흉내내는 실습.
- Prompt to paste into AI:

```text
Fast Campus 하네스 엔지니어링 실습 2002-3-3을 도와줘.

Curriculum mapping:
- Module: 2-3. Define — 공감에서 한 줄로 모이기 (evolve_step의 손 버전)
- Clip: [실습] 정의를 다른 시각으로 다시 본다 (재정의는 AI가)
- Key message: 한 번 모인 Seed의 사각지대를 다시 보는 일은 AI에 위임한다
- Required output: Seed v1 한 묶음 + 사각지대 응답 + 채택 표시 + Seed v2 한 묶음 + 옮겨진 의미 한 줄

Reference:
- Input Seed: 2002-2-3 Socrates 오케스트레이터가 떨어뜨린 YAML
- Ouroboros evolve_step: 한 세대 Seed → 다음 세대 Seed (ontology similarity ≥ 0.85에서 수렴)

Task:
2002-2-3에서 떨어진 Seed YAML을 입력으로 받아 다음 셋을 차례로 진행한다.

1) v1 보존: Seed YAML을 그대로 v1 자리에 옮긴다 (수정 금지).

2) 사각지대 두 질문: Seed v1을 보고 AI에 다음 두 질문을 한 번에 묶어 던진다.
   - 이 정의가 놓친 사용자 시각 세 가지
   - 이 정의로 잘려 나간 가능성 세 가지

3) v2 도출: 받은 여섯 개 답에 본인이 ☑ / ☒ 표시. 채택된 의미를 반영해 Seed v2 (YAML)를 다시 쓴다.
   v1과 v2를 나란히 두고, 어느 칸의 어느 의미가 옮겨졌는지 한 줄로 적는다.

Boundaries:
- v1을 그대로 보존한다 (덮어쓰지 않는다).
- v2는 본인이 채택한 의미만 반영한다. 추천을 자동 채택하지 않는다.
- 채택이 없으면 v1 = v2로 두고 "옮겨진 의미 없음"으로 기록한다.

My input:
[2002-2-3에서 떨어진 Seed YAML 한 블록]
```

## 목적

한 번 모인 Seed는 본인의 시야 안에서 만들어진 한 점이다.
그 점 바깥에 어떤 의미가 잘려 나갔는지 본인 혼자 다시 보는 건 시간이 많이 든다.
사각지대 검토는 AI에 위임하고, 채택은 본인이 한다.
이게 chapter 2의 척추 한 줄(**온톨로지 재정의는 AI에**)을 실습으로 옮긴 것이다 — Ouroboros가 evolve_step으로 자동화한 작업의 손 버전.

## 권장 시간

15분. v1 옮기기 3분, 사각지대 두 질문 5분, v2 도출 7분.

## 준비물

- 2002-2-3에서 떨어진 Seed YAML 한 블록
- 사각지대 두 질문을 던질 도구 (Claude Code, ChatGPT, Gemini 등 어느 것이든)

## 단계별 진행

1. **v1 옮기기** — Seed YAML을 그대로 v1 자리에 옮긴다 (수정 없이 보존).
2. **사각지대 두 질문 던지기** — 위 프롬프트의 두 질문을 한 번에 묶어 던진다 (응답 두 묶음, 각 세 개씩).
3. **채택/거절 표시** — 받은 여섯 개 답에 본인이 ☑ / ☒ 를 붙인다. 추천을 그대로 받지 않는다. 한 개도 채택 안 해도 된다.
4. **v2 다시 쓰기** — 채택된 의미만 반영해서 Seed v2 (YAML)를 새로 쓴다. v1과 형식은 같게 (goal / constraints / acceptance_criteria 세 칸).
5. **옮겨진 의미 한 줄** — v1 → v2 사이에 어느 칸의 어느 의미가 옮겨졌는지 한 줄로 적는다 (예: "독자 범위가 임원 두 명에서 임원 + 후속 실무자로 넓어졌다"). 채택이 없었으면 "옮겨진 의미 없음"으로.
6. **다음 단계 입력 확인** — Seed v1, Seed v2, 옮겨진 의미 한 줄. 이 셋이 2002-4-3 인터뷰 하네스의 Stage 3 입력이 된다.


## 예시

````markdown
# Seed v1 → v2

## Seed v1
```yaml
goal: 마케팅 PM이 다음 분기 예산 결정을 위한 캠페인 ROI 한 페이지 요약을 임원 두 명에게 만든다
constraints:
  - 한 페이지 분량을 넘기지 않는다
  - 외부 사실(미공개 마진 등)을 인용하지 않는다
acceptance_criteria:
  - 임원이 추가 질문 없이 결정으로 넘어간다
```

## 사각지대 두 질문 — 응답
### Q1. 놓친 사용자 시각
- ☑ 후보 1 — 후속 캠페인 실무자가 ROI 산출 근거를 추적할 수 있어야 한다
- ☒ 후보 2 — 외부 파트너에게 공유 가능한 톤으로 작성한다
- ☒ 후보 3 — 재무팀이 회계 마감과 대조할 수 있어야 한다

### Q2. 잘려 나간 가능성
- ☒ 후보 1 — 다음 분기 시나리오 두 가지를 함께 제시한다
- ☑ 후보 2 — "결정으로 넘어간다"의 의미가 임원마다 다를 수 있다
- ☒ 후보 3 — 디자인 정리에 시간을 쓴다

## Seed v2
```yaml
goal: 마케팅 PM이 임원 두 명과 후속 캠페인 실무자가 함께 볼 수 있는 캠페인 ROI 한 페이지 요약을 만든다
constraints:
  - 한 페이지 분량을 넘기지 않는다
  - 외부 사실을 인용하지 않는다
acceptance_criteria:
  - 임원 두 명이 같은 결정 항목에 합의한다
  - ROI 산출 근거가 후속 실무자에게 추적 가능하다
```

## 옮겨진 의미 (v1 → v2)
독자 범위가 "임원 두 명"에서 "임원 + 후속 캠페인 실무자"로 넓어졌고, 완료 신호가 "추가 질문 없이"에서 "임원 두 명이 같은 결정 항목에 합의한다"로 또렷해졌다.
````

## 완료 기준

- Seed v1을 YAML 한 블록으로 그대로 보존했다.
- 사각지대 두 질문에 각 세 개씩, 총 여섯 개 답이 있다.
- 채택/거절 표시가 모두에 붙어 있다.
- Seed v2가 채택된 의미만 반영해 새로 적혀 있다 (또는 "v1 = v2"로 명시).
- 옮겨진 의미 한 줄이 어느 칸의 어느 의미가 어떻게 바뀌었는지 가리킨다.

## 제출/검토 체크리스트

- [ ] v1을 그대로 보존했다 (덮어쓰지 않았다).
- [ ] 사각지대 두 질문을 본인이 직접 던졌고, 응답 여섯 개를 받았다.
- [ ] 채택은 본인이 했다. 추천을 자동 채택하지 않았다.
- [ ] v2 한 묶음이 v1과 같은 YAML 형식(goal / constraints / acceptance_criteria)이다.
- [ ] 옮겨진 의미 한 줄이 어느 칸의 어느 의미가 어떻게 바뀌었는지 가리킨다.
- [ ] 새로운 외부 사례, 검증되지 않은 수치, 가격 또는 마케팅성 정보를 넣지 않았다.
