# Handoff — L6 · review 세션 → 메인

> L6 도메인 제약 위반 검사 2회차(guard=false-negative). 1회차는 성공 기준 목표/지표 미결정으로 ask-back.
> 이번엔 의사결정자가 성공 기준을 측정 지표로 확정(다음 정기 미팅까지 약 4주 15% 감소, 2026-06-04)했고,
> build가 그 확정 지표를 성공 기준 칸에 반영한 반영본만 재검한다.
> 검수 대상은 L5 evidence 통과본 `handoffs/handoff-L3L4-build.md`. 근거 대조 원문은 `inputs/meeting.md`(37줄).
> 보존 인용 일치(evidence)는 L5가 이미 봤으므로 중복 verdict 없음.

## From task
- 목적: 도메인 제약 4항목 재검 — (a) 외부 사실 단정 (b) 미확인 수치 본문 진입 (c) 범위 밖 항목 유입 (d) 미결정 결론 톤
- 입력: `handoffs/handoff-L3L4-build.md`(성공 기준 지표화 반영본) + `inputs/meeting.md` + `inputs/review-gate.md`(§3 결정론 체크·§5 false-negative 보정)
- 범위 (하지 않을 일 포함): 통과/위반 항목과 근거 줄, 다음 route만. 본문 6칸 재작성 금지
- 금지: 본문 대신 고치기. evidence(보존 인용 일치)는 L5 몫이라 중복 verdict 금지
- 완료조건: 도메인 제약 4항목을 각각 통과/위반으로 판정하고 verdict 한 단어와 다음 행동을 낸다
