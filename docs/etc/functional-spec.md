# 기능 명세서

## 1. 회원 (User)

### 1.1 회원가입

- 이메일, 비밀번호, 이름 등 필수 정보 입력
- 비밀번호는 BCrypt(cost=12)로 해싱하여 저장
- 가입 시 기본 역할: `USER`

### 1.2 로그인

- 이메일/비밀번호 기반 인증
- 인증 성공 시 JWT(HS256) 토큰 발급
- 토큰에 사용자 ID, 역할 정보 포함

### 1.3 비밀번호 재설정

- 비밀번호 재설정 요청 처리
- 새 비밀번호 BCrypt 해싱 후 저장

### 1.4 역할 관리

| 역할 | 설명 |
|------|------|
| `USER` | 일반 사용자. 주문, 결제, 리뷰 작성 가능 |
| `ADMIN` | 관리자. 전체 시스템 관리 권한 |
| `MANAGER` | 매니저. 정산 확정, 상품 관리 등 운영 권한 |

## 2. 주문 (Order)

### 2.1 주문 생성

- 상품, 수량, 쿠폰 정보로 주문 생성
- 주문 생성 시 상태: `CREATED`

### 2.2 주문 조회

- 주문 ID로 단건 조회
- 사용자별 주문 목록 조회

### 2.3 주문 취소

- `CREATED` 상태의 주문만 취소 가능
- 취소 시 상태: `CANCELED`

### 2.4 주문 상태 관리

```
CREATED → PAID → REFUNDED
              → CANCELED
```

- `CREATED`: 주문 생성됨
- `PAID`: 결제 완료
- `REFUNDED`: 환불 완료
- `CANCELED`: 주문 취소

## 3. 결제 (Payment)

### 3.1 Toss Payments PG 연동

- Toss Payments API를 통한 결제 처리
- 멱등성 키(Idempotency Key)를 사용하여 중복 결제 방지

### 3.2 결제 상태 흐름

```
READY → AUTHORIZED → CAPTURED → REFUNDED
```

- `READY`: 결제 준비 상태
- `AUTHORIZED`: 결제 승인 완료 (카드사 승인)
- `CAPTURED`: 결제 캡처 완료 (실제 매출 확정)
- `REFUNDED`: 환불 처리 완료

### 3.3 결제 승인 (Authorize)

- Toss Payments API로 결제 승인 요청
- 승인 성공 시 상태: `AUTHORIZED`

### 3.4 결제 캡처 (Capture)

- 승인된 결제를 매출로 확정
- 캡처 성공 시 상태: `CAPTURED`

### 3.5 환불 (Refund)

- 캡처된 결제에 대해 환불 처리
- Toss Payments API로 환불 요청
- 환불 성공 시 상태: `REFUNDED`
- 관련 정산 데이터 환불 조정 처리

### 3.6 멱등성 키

- 각 결제 요청에 고유한 멱등성 키 부여
- 동일 키로 중복 요청 시 기존 결과 반환

## 4. 정산 (Settlement)

### 4.1 일일 배치 정산 생성

- Spring Batch 기반 일일 정산 배치 실행
- 캡처 완료된 결제 건을 대상으로 정산 데이터 생성
- 정산 정책: **D+7** (결제일로부터 7일 후 정산)
- 수수료율: **3%**
- 생성 시 상태: `REQUESTED`

### 4.2 정산 확정

```
REQUESTED → PROCESSING → DONE
                       → FAILED
```

- `REQUESTED`: 정산 요청됨
- `PROCESSING`: 정산 처리 중
- `DONE`: 정산 완료
- `FAILED`: 정산 실패

### 4.3 환불 조정

- 환불 발생 시 관련 정산 건의 금액 조정
- 이미 확정된 정산의 경우 차액 정산 처리

### 4.4 PDF 생성

- iText PDF 라이브러리를 사용하여 정산 내역서 PDF 생성
- 정산 기간, 금액, 수수료, 순정산액 등 포함

### 4.5 Elasticsearch 인덱싱

- 정산 데이터를 Elasticsearch에 인덱싱
- 기간별, 상태별, 금액 범위별 검색 지원

## 5. 상품 (Product)

### 5.1 상품 CRUD

- 상품 등록: 이름, 가격, 설명, 카테고리, 태그 정보 입력
- 상품 조회: 단건 조회, 목록 조회
- 상품 수정: 이름, 가격, 설명 등 변경
- 상품 삭제

### 5.2 이미지 업로드

- 상품 이미지 파일 업로드 지원
- 다중 이미지 등록 가능

### 5.3 카테고리/태그

- 상품별 카테고리 지정
- 태그 기반 상품 분류 및 검색

---

## 6. 쿠폰 (Coupon)

### 6.1 쿠폰 유형

| 유형 | 설명 |
|------|------|
| 정액 할인 | 고정 금액 할인 (예: 5,000원 할인) |
| 정률 할인 | 비율 기반 할인 (예: 10% 할인) |

### 6.2 최대 할인 상한

- 정률 할인 시 최대 할인 금액 상한 설정 가능
- 상한 초과 시 최대 할인 금액까지만 적용

### 6.3 유효기간

- 쿠폰별 사용 가능 시작일/종료일 설정
- 만료된 쿠폰은 사용 불가

---

## 7. 리뷰 (Review)

### 7.1 상품 리뷰 CRUD

- 리뷰 작성: 주문 완료 후 상품에 대한 리뷰 작성
- 리뷰 조회: 상품별 리뷰 목록 조회
- 리뷰 수정: 본인 작성 리뷰 수정
- 리뷰 삭제: 본인 작성 리뷰 삭제
