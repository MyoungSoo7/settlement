## 2. 프로젝트 목적 및 정당성

### 2.1 프로젝트 목적
전자상거래 환경에서 주문 생성부터 결제 승인, 정산 완료까지의 전 과정을 자동화하는 통합 시스템을 구축하여, 운영 효율성 향상 및 정산 투명성을 확보한다.

### 2.2 사업 정당성
- 전자상거래 거래량 증가에 따른 수작업 정산의 한계 도달
- 결제 오류 및 정산 지연으로 인한 판매자 불만 증가
- 통합 시스템 부재로 인한 데이터 불일치 및 관리 비용 증가

### 2.3 기대 효과

| 효과 영역 | 정량 목표 |
|-----------|----------|
| 정산 처리 시간 | 수작업 대비 90% 단축 |
| 결제 오류율 | 1% 미만 |
| 운영 비용 | 연간 30% 절감 |
| 판매자 만족도 | 80% 이상 |

## 3. 프로젝트 범위

### 3.1 포함 범위 (In-Scope)

| 도메인 | 주요 기능 |
|--------|-----------|
| 사용자 관리 | 회원가입, 로그인, JWT 인증/인가, 비밀번호 재설정 |
| 상품 관리 | 상품 CRUD, 이미지 업로드, Elasticsearch 검색 |
| 주문 관리 | 주문 생성, 상태 변경, 취소, 환불 |
| 결제 관리 | Toss PG 연동, 결제 승인/매입/취소 |
| 정산 관리 | 자동 배치 정산, Elasticsearch 검색 |
| 카테고리 | 다계층 카테고리 관리 |
| 쿠폰 | 쿠폰 발급/사용/만료 |
| 리뷰 | 상품 리뷰 CRUD |
| 프론트엔드 | React 18 SPA (관리자/판매자 대시보드) |

### 3.2 제외 범위 (Out-of-Scope)
- 게임 도메인 (오목/바둑) — 미완성, 향후 검토
- 물리 서버 / 네트워크 인프라 구축
- 기존 데이터 마이그레이션
- 모바일 앱 개발
