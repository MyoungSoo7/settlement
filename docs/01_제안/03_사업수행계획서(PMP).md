## 3. 인력 관리 계획

### 3.1 조직 구성

```
프로젝트 스폰서
    └── PM (프로젝트 관리자)
         ├── PL (백엔드) ── 백엔드 개발자
         ├── PL (프론트엔드) ── 프론트엔드 개발자
         ├── DA (데이터 아키텍트)
         └── QA (품질 관리)
```

### 3.2 역할 및 책임 (RACI)

| 활동 | PM | PL | DA | 개발자 | QA |
|------|----|----|----|----|----|----|
| 요구분석 | A | R | C | I | I |
| 아키텍처 설계 | A | R | R | C | I |
| DB 설계 | I | C | R | I | I |
| 개발 | A | R | I | R | I |
| 테스트 | A | C | I | C | R |
| 배포 | A | R | C | R | C |

> R: Responsible, A: Accountable, C: Consulted, I: Informed

## 5. 품질 관리 계획

### 5.1 품질 목표

| 품질 지표 | 목표 | 측정 방법 |
|----------|------|-----------|
| 코드 커버리지 | 70% 이상 | JaCoCo (Backend), Vitest (Frontend) |
| 결함 밀도 | 0.5건/KLOC 이하 | 결함 추적 시스템 |
| 코드 리뷰율 | 100% | PR 기반 코드 리뷰 |
| 정적 분석 | Critical 0건 | SonarCloud |
| 보안 취약점 | High 이상 0건 | Snyk |

### 5.2 코드 리뷰 기준
- PR 생성 후 최소 1인 이상 리뷰 필수
- 헥사고날 아키텍처 준수 여부 확인
- domain 레이어 외부 의존성 금지 확인
- 테스트 코드 포함 여부 확인

## 6. 형상 관리 계획

### 6.1 형상 관리 도구

| 도구 | 용도 |
|------|------|
| Git | 소스코드 버전 관리 |
| GitHub | 원격 저장소, PR, 이슈 관리 |
| GitHub Actions | CI/CD 파이프라인 |
| Docker / GHCR | 컨테이너 이미지 관리 |

### 6.2 브랜치 전략
- `main`: 프로덕션 브랜치
- `develop`: 개발 통합 브랜치
- `feature/*`: 기능 개발 브랜치
- `hotfix/*`: 긴급 수정 브랜치

### 6.3 CI/CD 파이프라인
- Backend: Gradle 빌드 → JaCoCo → SonarCloud → Snyk → GHCR push
- Frontend: TypeScript 체크 → ESLint → Vite 빌드 → Snyk → GHCR push

## 7. 변경 관리 계획

### 7.1 변경 관리 프로세스

```
변경 요청 접수 → 영향도 분석 → CCB 심의 → 승인/반려 → 변경 이행 → 결과 검증
```

### 7.2 변경통제위원회 (CCB)

| 역할 | 구성원 | 권한 |
|------|--------|------|
| 위원장 | 발주기관 담당자 | 최종 승인 |
| 위원 | PM | 영향도 분석, 일정 조정 |
| 위원 | PL | 기술적 영향도 평가 |

### 7.3 변경 등급

| 등급 | 기준 | 승인 권한 |
|------|------|-----------|
| A (중대) | 일정/비용/범위 변경 | CCB |
| B (일반) | 설계 변경, 기능 수정 | PM |
| C (경미) | UI 수정, 문구 변경 | PL |

## 9. 산출물 관리 계획

### 9.1 단계별 산출물

| 단계 | 산출물 | 필수 |
|------|--------|------|
| 제안 | RFP, 제안서, PMP, 프로젝트 헌장, WBS, 투입인력계획서 | 필수 |
| 분석 | 현행시스템 분석서, SRS, RTM, BPD, 유스케이스 명세서 | 필수 |
| 설계 | 아키텍처 설계서, UI/UX 설계서, ERD, 테이블 정의서, API 명세서 | 필수 |
| 구현 | 소스코드, 단위테스트 결과, 코드 인스펙션 결과 | 필수 |
| 테스트 | 통합테스트 계획서/결과서, 성능테스트 결과 | 필수 |
| 이행 | 운영 매뉴얼, 교육 자료, UAT 결과서 | 필수 |

### 9.2 산출물 검토 및 승인
- 모든 필수 산출물은 작성 후 검토 → 승인 프로세스를 거침
- 승인 전 사내 리뷰 1회 이상 수행
- 발주기관 승인 필요 산출물은 별도 검토 일정 협의
