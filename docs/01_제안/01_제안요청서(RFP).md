## 2. 사업 목적 및 배경

### 2.1 추진 배경
- 전자상거래 거래량 증가에 따른 주문·결제·정산 프로세스 자동화 필요
- 기존 수작업 정산 방식의 오류 빈도 증가 및 처리 지연
- 다중 PG사 연동 및 통합 결제 관리 요구
- 판매자 정산 자동화 및 투명성 확보 필요

### 2.2 사업 목적
- 주문 생성부터 결제 승인, 정산 완료까지의 End-to-End 프로세스 자동화
- Toss PG 연동 기반 통합 결제 시스템 구축
- 일별/주별/월별 자동 정산 배치 시스템 구현
- 판매자/관리자 대시보드를 통한 정산 투명성 확보

## 3. 사업 범위

### 3.1 시스템 구축 범위

| 구분 | 세부 내용 |
|------|-----------|
| 사용자 관리 | 인증/인가, JWT, 비밀번호 재설정 |
| 상품 관리 | 상품 CRUD, 이미지 관리, Elasticsearch 검색 |
| 주문 관리 | 주문 생성/상태 변경/취소/환불 |
| 결제 관리 | Toss PG 연동, 결제 승인/매입/취소, 환불 |
| 정산 관리 | 정산 자동화, 배치 처리, Elasticsearch 검색 |
| 카테고리 관리 | 다계층 이커머스 카테고리 |
| 쿠폰 관리 | 쿠폰 발급/사용/만료 |
| 리뷰 관리 | 상품 리뷰 |
| 프론트엔드 | React 18 기반 SPA (관리자/판매자 대시보드) |

### 3.2 기술 요구사항

| 구분 | 기술 스택 |
|------|-----------|
| 백엔드 | Spring Boot, Java 21 |
| 프론트엔드 | React 18, Vite, TypeScript |
| 데이터베이스 | PostgreSQL 17 |
| 검색엔진 | Elasticsearch 8.x (Nori 한글 분석기) |
| 아키텍처 | 헥사고날 (Ports & Adapters) |
| CI/CD | GitHub Actions, Docker, GHCR |

### 3.3 제외 범위
- 하드웨어 구매 및 설치
- 네트워크 인프라 구축
- 기존 데이터 마이그레이션 (별도 협의)

## 6. 제안 요청 내용

### 6.1 제안서 구성
- 기술 제안: 시스템 아키텍처, 기술 스택, 개발 방법론
- 관리 제안: 프로젝트 관리 방안, 일정, 투입 인력
- 가격 제안: 세부 비용 산출 내역

### 6.2 제안 자격 요건
- SI 구축 경험 3년 이상

- Spring Boot, React 기반 개발 역량 보유
- PostgreSQL, Elasticsearch 운영 경험
