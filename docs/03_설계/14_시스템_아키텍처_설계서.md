## 3. 헥사고날 아키텍처

### 3.1 패키지 구조 (모든 도메인 공통)

```
github.lms.lemuel.{domain}/
├── adapter/
│   ├── in/
│   │   ├── web/              # REST Controller
│   │   │   └── dto/          # Request/Response DTO
│   │   └── batch/            # Spring Batch (정산)
│   └── out/
│       ├── persistence/      # JPA Entity, Repository, Mapper
│       └── search/           # Elasticsearch Adapter
├── application/
│   ├── port/
│   │   ├── in/               # UseCase 인터페이스
│   │   └── out/              # Port 인터페이스
│   └── service/              # UseCase 구현체
├── domain/                   # 순수 도메인 객체 (POJO)
└── exception/                # 도메인 예외
```

### 3.2 의존성 규칙

| 레이어 | 의존 가능 | 의존 불가 |
|--------|----------|-----------|
| domain | 없음 (순수 POJO) | Spring, JPA, 외부 라이브러리 |
| application | domain | adapter, 프레임워크 직접 참조 |
| adapter | application, domain | - |

### 3.3 도메인-엔티티 변환
- **MapStruct 1.6.3**: `@Mapper(componentModel = "spring")`
- Mapper는 `adapter/out/persistence/` 패키지
- **Lombok**: `@Getter`, `@Builder`, `@NoArgsConstructor` 등

## 4. 도메인 설계

### 4.1 도메인별 핵심 정보

| 도메인 | 상태 Enum | 컨트롤러 | 핵심 서비스 |
|--------|----------|---------|-----------|
| user | UserRole (USER/ADMIN/MANAGER) | AuthController, UserController | LoginService, PasswordResetService |
| product | ProductStatus (ACTIVE/INACTIVE/OUT_OF_STOCK/DISCONTINUED) | ProductController + 3 | CreateProductService, ManageProductStatusService |
| order | OrderStatus (CREATED/CONFIRMED/CANCELLED/REFUNDED) | OrderController | CreateOrderService, ChangeOrderStatusService |
| payment | PaymentStatus (READY/AUTHORIZED/CAPTURED/REFUNDED/PARTIALLY_REFUNDED/CANCELED) | PaymentController | TossPaymentService |
| settlement | SettlementStatus (REQUESTED/PENDING/CONFIRMED/FAILED/CANCELED) | SettlementController + Search | CreateDailySettlementsService, GenerateSettlementPdfService |
| category | - | AdminEcommerceCategory + Public | EcommerceCategoryService (다계층, Slug) |
| coupon | CouponType (PERCENTAGE/FIXED_AMOUNT) | CouponController | CouponService |
| review | - | ReviewController | ReviewService |

### 4.2 핵심 도메인 간 흐름

```
User → Order(CREATED) → Payment(READY→AUTHORIZED→CAPTURED)
                                          ↓
                              Settlement(REQUESTED→PENDING→CONFIRMED)
                                          ↓ (환불 시)
                              RefundPayment → settlement_adjustments
```
