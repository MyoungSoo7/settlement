## 2. 상태 코드 정의 (실제 Enum 기준)

### 2.1 주문 상태 (OrderStatus) — `order/domain/OrderStatus.java`

| 코드 | 한글명 | 설명 | 전이 가능 상태 |
|------|--------|------|--------------|
| CREATED | 주문 생성 | 주문 생성됨, 결제 전 | CONFIRMED, CANCELLED |
| CONFIRMED | 주문 확정 | 결제 완료 후 확정 | REFUNDED |
| CANCELLED | 주문 취소 | 결제 전 취소 | - (최종 상태) |
| REFUNDED | 환불 완료 | 환불 처리 완료 | - (최종 상태) |

### 2.2 결제 상태 (PaymentStatus) — `payment/domain/PaymentStatus.java`

| 코드 | 한글명 | 설명 | 전이 가능 상태 |
|------|--------|------|--------------|
| READY | 결제 준비 | 결제 요청 생성 | AUTHORIZED, CANCELED |
| AUTHORIZED | 승인 완료 | PG 승인 완료 | CAPTURED, CANCELED |
| CAPTURED | 매입 완료 | 결제 매입 완료, 정산 대상 | REFUNDED, PARTIALLY_REFUNDED |
| REFUNDED | 전체 환불 | 전체 금액 환불 | - (최종 상태) |
| PARTIALLY_REFUNDED | 부분 환불 | 일부 금액 환불 | REFUNDED |
| CANCELED | 취소 | 결제 취소 | - (최종 상태) |

**추가 필드:**
- `refunded_amount`: 누적 환불 금액 (0 ≤ refunded_amount ≤ amount)
- `captured_at`: 매입 처리 시각

### 2.3 정산 상태 (SettlementStatus) — `settlement/domain/SettlementStatus.java`

| 코드 | 한글명 | 설명 | 전이 가능 상태 |
|------|--------|------|--------------|
| REQUESTED | 요청됨 | 정산 요청 접수 | PENDING |
| PENDING | 대기 | 배치에 의해 생성, 확정 대기 | CONFIRMED, FAILED |
| CONFIRMED | 확정 | 배치에 의해 확정 완료 | - (최종 상태) |
| FAILED | 실패 | 처리 중 오류 | - (failureReason 기록) |
| CANCELED | 취소 | 정산 취소 | - (최종 상태) |

**정산 금액 계산 (Settlement.java):**
- 수수료율: **3%** (하드코딩)
- fee = amount × 0.03
- net_amount = amount - fee
- settlement_date = 정산 생성일

### 2.4 상품 상태 (ProductStatus) — `product/domain/ProductStatus.java`

| 코드 | 한글명 | 설명 |
|------|--------|------|
| ACTIVE | 판매 중 | 정상 판매 |
| INACTIVE | 비활성 | 임시 비활성 |
| OUT_OF_STOCK | 품절 | 재고 소진 |

### 2.5 쿠폰 유형 (CouponType) — `coupon/domain/CouponType.java`

| 코드 | 한글명 | 계산 방식 |

### 2.6 환불 상태 (Refund) — `refunds` 테이블

## 3. 핵심 프로그램 상세 정의

### 3.1 TossPaymentService — Toss PG 결제 연동

| 항목 | 내용 |
|------|------|
| 프로그램 ID | BE-PAY-010 |
| 프로그램명 | TossPaymentService |
| 분류 | Service |
| 패키지 | payment.application.service |
| 관련 요구사항 | FR-PAY-001 ~ FR-PAY-007 |
| 외부 연동 | Toss Payments API (`https://api.tosspayments.com/v1/payments/confirm`) |

**주요 메서드:**

| 메서드 | 엔드포인트 | 설명 |
|--------|----------|------|
| confirmPayment() | POST /payments/toss/confirm | Toss 결제 확인 (TossConfirmRequest) |
| confirmCartPayment() | POST /payments/toss/cart/confirm | 장바구니 일괄 결제 확인 |

**처리 흐름 (Toss 결제 확인):**

```
confirmPayment(TossConfirmRequest)
  ├── 1. paymentKey, orderId, amount 검증
  ├── 2. Toss API 호출 (POST /v1/payments/confirm)
  │     └── Basic Auth: TOSS_SECRET_KEY
  ├── 3. Payment 상태: READY → AUTHORIZED → CAPTURED
  ├── 4. captured_at 시각 기록
  ├── 5. Order 상태: CREATED → CONFIRMED
  └── 6. 이벤트 발행 (PublishEventPort)
```

### 3.2 RefundPaymentUseCase 구현 — 환불 처리

| 항목 | 내용 |
|------|------|
| 관련 테이블 | payments, refunds, settlement_adjustments |
| 동시성 제어 | PESSIMISTIC_WRITE Lock |
| 멱등성 | Idempotency-Key 헤더 |

**처리 흐름:**

```
refundPayment(paymentId, RefundRequest, Idempotency-Key)
  ├── 1. refunds 테이블에서 idempotency_key 중복 조회
  │     └── 중복 → 기존 Refund 결과 반환 (멱등성)
  ├── 2. Payment 조회 (PESSIMISTIC_WRITE Lock)
  ├── 3. 환불 가능 금액 검증 (amount - refunded_amount ≥ refundAmount)
  ├── 4. Toss PG 환불 API 호출 (PgClientPort)
  ├── 5. refunds 레코드 생성 (COMPLETED)
  ├── 6. Payment.refunded_amount += refundAmount
  ├── 7. Payment 상태 변경 (REFUNDED / PARTIALLY_REFUNDED)
  ├── 8. Order 상태 → REFUNDED
  └── 9. settlement_adjustments 레코드 생성
```

### 3.3 Spring Batch 정산 배치 — SettlementJobConfig

| 항목 | 내용 |
|------|------|
| 위치 | settlement.adapter.in.batch |
| 스케줄러 | Spring Batch (JobRepository 중복 방지) |
| 모니터링 | Prometheus 메트릭 + Slack 알림 |

#### Job 1: createSettlementJob (매일 02:00)

```
CreateSettlementsTasklet
  ├── Reader: 전날 CAPTURED 결제 조회 (LoadCapturedPaymentsPort)
  ├── Processor: 수수료 3% 계산, 정산 금액 산출
  └── Writer: Settlement 레코드 생성 (status: PENDING)
```

#### Job 2: confirmSettlementJob (매일 03:00)

```
ConfirmSettlementsTasklet
  ├── Reader: PENDING 정산 조회
  └── Writer: status → CONFIRMED
```

#### Job 3: adjustmentSettlementJob (매일 03:10)

```
AdjustSettlementForRefundService
  └── settlement_adjustments 미처리 건 확정
```

**배치 헬스**: `SettlementBatchHealthIndicator` → `/actuator/health`
