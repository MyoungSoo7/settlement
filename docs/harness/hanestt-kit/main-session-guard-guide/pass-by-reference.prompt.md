# Pass by Reference 프롬프트 (B안)

## 한 줄

sub-agent를 **저자**로 쓴다. 메인은 경로만 가져오고 인덱서/오케스트레이터가 된다.

## 목적

Sub-agent의 결과를 **파일 시스템**에 영속화하고 메인 세션은 경로만 들고 종료시켜, 메인 컨텍스트가 거의 소비되지 않음을 시연한다. Pass by Value(A안)와 동일 리포지토리에서 동일 깊이로 비교한다.

## 측정 항목

- 메인 컨텍스트 사용량 (%)
- 누적 비용 ($)
- 메인 세션 답변 길이 (문자 수)
- 다음 세션이 정보 재사용 가능 여부 (Y/N)

## 프롬프트

```text
당신은 이 리포지토리의 강의자료 관리자입니다.

[작업 절차 — 반드시 이 순서로]
1. Sub-agent 1개를 소환해서 리포지토리 전체를 조사시키세요.
2. Sub-agent에게 다음을 직접 `output/analysis.md` 파일에 작성하라고
   지시하세요 (Write 권한 부여):
   - 디렉토리 구조 (모든 파일 경로 포함)
   - 6개 챕터별 핵심 내용 요약 (각 챕터당 200자 이상)
   - 실습 자료 29개 각각의 목적 (1줄씩)
   - 품질 감사 결과 요약
   - 다음 추천 작업 3가지
3. Sub-agent에게는 "작성 완료 후 파일 경로 한 줄만 반환"하라고
   명시적으로 지시하세요. 어떤 요약도 반환하지 말 것.
4. 메인 세션은 가져온 경로로 `ls -la <path>` 만 실행해서
   파일 존재와 크기만 확인하고 종료하세요.

[제약]
- 메인 세션은 절대 파일을 Read하지 마세요.
- 메인 세션은 절대 리포지토리 파일을 직접 Read하지 마세요.
- 메인 세션의 최종 답변은
  "✅ output/analysis.md 생성 완료 (12,345 bytes)" 같은 한 줄이면 충분합니다.
- 사용자가 내용을 보고 싶다면 다음 세션에서 직접 파일을 열게 하세요.
```

## 예상 결과

- 메인 컨텍스트: **낮음** (경로 문자열 + ls 결과 한 줄)
- 단일 세션 비용: 비교적 비쌈 (sub가 풀텍스트 작성)
- 재사용성: artifact가 디스크에 영속 → 다음 세션이 그대로 소비

## 학습 포인트

비용은 1세션에 집중되지만, 산출물의 수명이 세션 수명을 넘는다. N회 재사용에서 상각 곡선이 Pass by Value를 추월한다. 큰 작업에서는 *가능 vs 불가능*을 가르는 enabling pattern이다.

sub의 산출 깊이가 메인 RAM과 분리된다 — 메인이 가져오는 것은 경로 한 줄뿐이므로 sub는 자기검열 없이 적는다.
