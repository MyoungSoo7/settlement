# 실습 가이드 — 두 안을 같은 깊이로 돌려본다

## 권장 시간

30~40분. A안 10분, B안 10분, 측정·정리 10~20분.

## 준비물

- 본인이 자주 쓰는 리포지토리 1개 (또는 본 강의 리포지토리 그대로 사용)
- 프롬프트 두 개: `pass-by-value.prompt.md`, `pass-by-reference.prompt.md`
- 측정 기록 표

## 단계별 진행

1. **세션 분리** — A안과 B안을 *별도 세션*에서 돌린다. 한 세션에 두 패턴이 섞이면 컨텍스트 측정이 오염된다.
2. **A안 실행** — `pass-by-value.prompt.md`를 그대로 붙여넣고 한 번 돌린다. 측정 4개를 기록한다.
3. **B안 실행** — 새 세션에서 `pass-by-reference.prompt.md`를 그대로 붙여넣고 돌린다. 측정 4개를 기록한다.
4. **재사용 검증** — B안이 만든 `output/analysis.md`를 또 다른 새 세션에서 그대로 열어 본다. 정보가 살아 있는지 확인한다.
5. **가드레일 점검** — 비교표.md 마지막 절의 금지 항목을 어기지 않았는지 본다.

## 측정 기록 표

| 항목 | A안 (Pass by Value) | B안 (Pass by Reference) |
|---|---|---|
| 메인 컨텍스트 사용량 (%) | | |
| 누적 비용 ($) | | |
| 메인 답변 길이 (문자 수) | | |
| 다음 세션 재사용 가능 (Y/N) | | |
| 단일 세션 비용 체감 | | |

## 가드레일 (다시 한 번)

**A안에서 금지**
- sub가 파일을 작성한다 → A안의 정의가 깨진다.
- 메인이 리포지토리를 직접 Read한다 → sub 사용량이 줄어 비교가 흐려진다.

**B안에서 금지**
- 메인이 파일을 Read한다 → 메인 컨텍스트로 본문이 들어와 B안의 핵심 효과가 사라진다.
- sub가 요약을 반환한다 → 메인 컨텍스트가 부풀어 A안과 구분이 안 된다.

## 회고 — 한 줄씩 적는다

- 본인 업무에서 *값*으로 가져와야 하는 산출물은 무엇인가?
- 본인 업무에서 *주소*로 가져와 디스크에 남겨야 하는 산출물은 무엇인가?
- 현재 본인이 메인 세션에 몰아넣고 있어서 컨텍스트가 가장 빨리 부풀어오르는 작업은 무엇인가?
