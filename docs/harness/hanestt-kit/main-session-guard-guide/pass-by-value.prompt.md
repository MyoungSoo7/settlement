# Pass by Value 프롬프트 (A안)

## 한 줄

sub-agent를 **검색 함수**로 쓴다. 메인이 결과를 그대로 가져와 종합·판단한다.

## 목적

sub-agent의 결과를 **텍스트 반환값**으로 가져와 메인 세션 컨텍스트가 어떻게 부풀어오르는지 본인 눈으로 확인한다. Pass by Reference(B안)와 같은 리포지토리에서 같은 깊이로 비교한다.

## 측정 항목

- 메인 컨텍스트 사용량 (%)
- 누적 비용 ($)
- 메인 세션 답변 길이 (문자 수)
- 다음 세션이 정보 재사용 가능 여부 (Y/N)

## 프롬프트

```text
당신은 이 리포지토리의 강의자료 관리자입니다.

[작업 절차 — 반드시 이 순서로]
1. Sub-agent 1개를 소환해서 리포지토리 전체를 조사시키세요.
2. Sub-agent에게 다음을 텍스트로 "전부 반환"하라고 지시하세요:
   - 디렉토리 구조 (모든 파일 경로 포함)
   - 6개 챕터별 핵심 내용 요약 (각 챕터당 200자 이상)
   - 실습 자료 29개 각각의 목적 (1줄씩)
   - 품질 감사 결과 요약
   - 다음 추천 작업 3가지
3. Sub-agent의 반환값을 가져와서, 메인 세션이 직접 사용자에게
   잘 정리된 보고서를 출력하세요. 표·섹션·요약을 포함해서.

[제약]
- 파일은 만들지 마세요.
- Sub-agent의 반환값을 그대로 보존해서 메인 세션 답변에 녹여내세요.
- 정보 손실 없이 풀어서 보여주세요.
- 메인 세션이 직접 리포지토리 파일을 Read하지 마세요.
  모든 정보는 sub-agent를 거쳐야 합니다.
```

## 예상 결과

- 메인 컨텍스트: **높음** (sub 풀 텍스트 + 메인의 가공된 출력 누적)
- 단일 세션 비용: 비교적 저렴
- 재사용성: 세션 종료 시 휘발

## 학습 포인트

이 패턴이 *가능한* 작업의 상한은 메인 컨텍스트 한계와 같다. 작업이 커질수록 메인이 먼저 죽는다.

sub는 자기검열한다 — "메인이 가져올 수 있는 분량으로 정리"가 sub의 기본 동작이 되므로, sub의 산출 깊이가 메인 RAM에 묶인다.
