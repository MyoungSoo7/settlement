# Evaluator — 본인 업무용 빈 평가 기준표

본인 업무의 결과 하나를 골라 채운다. 입력의 완료조건은 가이드 05의 AC Tree에서 가져온다. 한 페이지에 끝낸다.

---

## Inputs

| 항목 | 내용 |
|---|---|
| 평가 대상 (결과 한 장) |  |
| AC (가이드 05에서) |  |
| claim (AI의 완료 주장) |  |
| evidence (실제 산출물·근거) |  |

claim과 evidence는 따로 적는다. 둘이 같은 대상을 가리키는지는 아래 의미 단계에서 본다.

## Stage 1 — 형식

- 판정 질문: (필수 칸·분량·형식이 다 채워졌는가)
- 결과: [통과 / 실패]
- 사유:
- 다음 행동: 통과면 Stage 2, 실패면 재작업

## Stage 2 — 의미 (claim vs evidence 대조)

- 판정 질문: (주장한 근거가 실제 증거에 있는가 / AC 용어와 어긋나지 않는가)
- 결과: [통과 / 실패]
- 사유:
- 다음 행동: 통과면 Stage 3, 실패면 재작업 또는 재질문

## Stage 3 — 목적

- 판정 질문: (이 결과만 보고 원래 결정·행동으로 넘어갈 수 있는가)
- 결과: [통과 / 실패]
- 사유:
- 다음 행동: 통과면 통과, 실패면 재질문 또는 종료

## Verdict

- 한 줄 판정: 통과 / 재작업 / 재질문 / 종료
- 다음 세션의 첫 행동: (동사 + 목적어)

## 종료 신호 (사람에게 보고)

- (예: 같은 사유로 재작업 3회 / 외부 답 대기)

## 재질문 신호 (질문 세션으로)

- (예: AC 단어 하나가 두 해석을 만든다 / 입력에 같은 용어가 두 뜻으로 섞여 있다)

---

## 제출 / 자기 검수 체크리스트

- [ ] claim과 evidence를 따로 적었다.
- [ ] 의미 단계가 "주장한 근거가 증거에 실재하는가"를 본다(claim-term guard).
- [ ] 한 줄 판정이 통과 / 재작업 / 재질문 / 종료 중 한 단어다.
- [ ] **같은 결과를 두 번 넣었을 때 판정 단어가 동일하다**(결정론 — 같은 입력 같은 판정).
- [ ] 종료 신호가 "3회 / N건 / 외부 답 대기" 같은 구체 조건이다.
- [ ] "더 좋아질 수 있음"을 재작업 사유로 쓰지 않았다(Satisfice).
- [ ] 외부 사례, 검증되지 않은 수치, 가격 또는 마케팅성 정보를 넣지 않았다.
