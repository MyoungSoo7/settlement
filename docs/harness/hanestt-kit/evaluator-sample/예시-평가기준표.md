# 예시 — 고객 문의 답변 검수 (평가 기준표)

채워진 한 장 예시다. 가이드 05의 AC를 입력으로 받아 결정론 게이트로 한 번 돌린 모양을 보여준다.

---

## Inputs

| 항목 | 내용 |
|---|---|
| 평가 대상 | 고객 환불 문의 답변 초안 한 장 |
| AC (가이드 05에서) | 문의의 핵심 조건을 모두 반영하고, 금지 표현이 없고, 발송 가능한 상태 |
| claim | "문의 조건 3개 다 반영했고 금지 표현 없습니다 — 발송 가능" |
| evidence | 답변 초안 본문 + 문의 원문 + 금지어 목록 |

## Stage 1 — 형식

- 판정 질문: 인사·본문·다음 안내 세 칸이 다 있고 분량이 정책 범위(10줄 이내)인가
- 결과: 통과
- 사유: 8줄, 세 칸 모두 있음
- 다음 행동: Stage 2

## Stage 2 — 의미 (claim vs evidence 대조)

- 판정 질문: claim이 말한 "조건 3개"가 실제 본문에 있는가 / 금지어가 정말 없는가
- 결과: 실패
- 사유: 문의 조건 중 "환불 처리 기간"이 본문에 없음. claim은 3개 반영이라 했지만 evidence에는 2개만 있음 (claim vs evidence 불일치)
- 다음 행동: 재작업

## Stage 3 — 목적

- 판정 질문: 이 답변만 보고 고객이 추가 문의 없이 다음 행동을 할 수 있는가
- 결과: (Stage 2에서 멈춤 — 평가 안 함)

## Verdict

- 한 줄 판정: **재작업**
- 다음 세션의 첫 행동: 생성 세션에 "환불 처리 기간 한 줄 추가" 요청

## 종료 신호 (사람에게 보고)

- 같은 "조건 누락" 사유로 재작업 3회 연속이면 종료하고 담당자에게 보고
- 환불 정책 자체가 불명확해 답을 못 정하면 종료 상태로 보존

## 재질문 신호 (질문 세션으로)

- 문의 원문에 "환불"과 "교환"이 한 단어로 섞여 있어 어느 쪽인지 모호하면 고객에게 재질문
- AC의 "발송 가능한 상태"가 "초안 완성"인지 "최종 발송 승인"인지 갈리면 담당자에게 확인

---

## 이 예시가 보여주는 것

claim은 "조건 3개 다 반영"이라고 통과를 주장했지만, evidence를 따로 읽어 대조하니 실제로는 2개만 있었다. 사람이 답변을 끝까지 읽고 느낌으로 판단한 게 아니라, "주장한 항목이 증거에 실재하는가"를 의미 단계에서 대조해 재작업으로 갈랐다. 그리고 재작업 범위는 "처음부터 다시"가 아니라 "한 줄 추가"로 좁혀졌다 — AC가 또렷했기 때문이다.

같은 초안을 새 세션에 다시 넣어도 판정 단어는 똑같이 "재작업"으로 나와야 한다. 다르게 나오면 Stage 2 판정 질문이 아직 모호하다는 신호다.
