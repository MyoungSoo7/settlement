# 호출 그래프와 스크래치 파일 흐름

## 8개 스킬의 호출 그래프

```
                  ┌─────────────────────────────┐
사용자 한 줄 ───→ │  /interview-harness          │
                  │  (마스터 오케스트레이터)      │
                  └──────────┬──────────────────┘
                             │
                Stage 1: Seed Gate
                             │
                ┌────────────┴─ Seed가 없거나 미완 ─────┐
                │                                       │
                │ Seed가 닫혀 있다 → 건너뜀               │
                │                                       ▼
                │                          ┌───────────────────────┐
                │                          │  /socrates              │
                │                          │  (Ralph 루프 1~5)        │
                │                          └──┬────────────────────┘
                │                             │
                │                  Wonder → Reflect → Refine → Restate
                │                     ▼          ▼         ▼          ▼
                │                ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
                │                │/wonder │ │/reflect│ │/refine │ │/restate│
                │                └────────┘ └────────┘ └────────┘ └────────┘
                │                             │
                │                       Seed v1 (YAML)
                │                             │
                ▼                             ▼
        Stage 2: Convergence Loop (1~5 사이클)
                             │
                             ▼
                  ┌─────────────────────────┐
                  │  /evolve-step            │
                  │  (사각지대 6 후보 → 채택) │
                  └──────────┬──────────────┘
                             │
                       Seed v_(n+1)
                             │
                             ▼
                  ┌─────────────────────────┐
                  │  /ontology               │
                  │  (idea/boundary/3 props) │
                  └──────────┬──────────────┘
                             │
                       ontology v_(n+1)
                             │
                             ▼
                  similarity ≥ 0.85? ── YES ──→ 종료 (convergence)
                             │
                             NO
                             │
                             ▼
                       다음 사이클 (Stage 2 처음으로)

                  Cycle > 5? ── YES ──→ 종료 (safety_valve)
```

## 한 사이클의 입력 narrow화

```
Cycle 1: 사용자 원문 (broad)
   → /socrates의 W/R/R/R 질문이 발산 모드 (의미 후보를 넓게 본다)
   → 사용자가 처음부터 답한다

Cycle 2+: 직전 Seed v_n이 입력 (narrow)
   → /socrates의 W/R/R/R 질문이 좁혀진다
   → 사용자 답도 빠르게 떨어진다
   → ontology가 점점 안정된다 → similarity가 올라간다
```

## 스크래치 파일 흐름

```
.claude/scratch/
├── socrates.md              ← canonical 파이프라인
│   ├── ## User Input        (사용자 원문)
│   ├── ## Cycle 1
│   │   ├── ### Wonder       (wonder가 씀)
│   │   ├── ### Reflect      (reflect가 씀)
│   │   ├── ### Refine       (refine이 씀)
│   │   └── ### Restate      (restate가 씀)
│   ├── ## Cycle 2, 3 …      (소크라테스가 새 사이클을 열 때 추가)
│   ├── ## Seed              ← 최신 Seed YAML 한 벌만 (매번 교체, 누적 X)
│   │                          socrates가 처음 만들고
│   │                          evolve-step이 매 사이클 교체
│   └── ## Ontology          ← 최신 Ontology 한 벌만 (매번 교체, 누적 X)
│                              ontology가 매 사이클 교체
│
├── evolve-step.md           ← audit 로그 (사이클별 누적)
│   ├── # Evolve-step — Cycle 1
│   │   ├── ## Seed v1
│   │   ├── ## Blind Spot Candidates (Q1 3개 + Q2 3개)
│   │   ├── ## Seed v2
│   │   └── ## Transferred Meaning
│   ├── # Evolve-step — Cycle 2
│   └── ...
│
└── interview-harness.md     ← 직전 ontology 스냅샷 + similarity 비교 로그
    ├── ## Previous Ontology (직전 사이클 ontology, 비교용)
    └── ## Similarity Log
        ├── Cycle 1 → 2: idea=1, boundary=0, properties=0.67 → 0.56
        └── Cycle 2 → 3: idea=1, boundary=1, properties=1.0 → 1.00 ✓
```

## 파일별 누적/교체 정책

| 파일 | 누적? | 매 사이클 무엇이 바뀌는가 |
|---|---|---|
| `socrates.md` 의 사이클 섹션들 | 누적 | 새 `## Cycle N`이 아래에 추가됨 |
| `socrates.md` 의 `## Seed` | **교체** | 매 사이클 통째로 갈아끼움. 항상 최신 한 벌만 |
| `socrates.md` 의 `## Ontology` | **교체** | 매 사이클 통째로 갈아끼움. 항상 최신 한 벌만 |
| `evolve-step.md` | 누적 | 사이클마다 새 `# Evolve-step — Cycle N` 블록 추가 |
| `interview-harness.md` | 교체 + 누적 | 직전 ontology는 교체, similarity 로그는 누적 |

핵심 설계: **canonical 한 벌(socrates.md의 Seed/Ontology)은 최신만, audit 로그(evolve-step.md, interview-harness.md의 similarity)는 누적.**

## sub 스킬의 단독 호출 보장

마스터 하네스는 sub 스킬을 *호출*만 한다. 본문 흐름을 흉내내지 않는다. 그 결과:

- 사용자가 `/wonder`만 단독 호출하면 의미 발산만 하고 끝난다.
- `/socrates`만 단독 호출하면 Seed v1까지만 만들고 끝난다.
- `/evolve-step`만 단독 호출하면 기존 Seed에 사각지대 점검 한 사이클만 돈다.
- `/ontology`만 단독 호출하면 기존 Seed에 경계만 정한다.

마스터 하네스가 깨져도 sub 스킬은 살아남는다. 분리는 그대로, 사용은 통합.
