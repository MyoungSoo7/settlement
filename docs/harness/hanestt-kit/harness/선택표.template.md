# 하네스 선택표 — 본인 업무용 빈 템플릿

본인 업무 하나를 골라 채운다. 한 페이지에 끝낸다.

---

## 업무

- 업무 이름:
- 업무 목표 (한 문장):
- 현재 가장 큰 실패 원인:

## 증거 표 (3축)

| 질문 | 본인 업무의 증거 | 점수 (1~3) |
|---|---|---|
| Q1. 같은 규칙을 매번 반복 설명하는가? | | |
| Q2. 메인 세션이 쉽게 지치는가? (탐색·초안·검수 혼재) | | |
| Q3. 순서·상태·재시도·복귀점을 사람이 매번 기억하는가? | | |

가장 강한 증거의 축이 본인 업무의 통제 지점이다.

## 선택 기준

| 질문 | 예 / 아니오 | 근거 |
|---|---|---|
| 반복 규칙·역할·금지사항·출력 형식만 고정하면 충분한가 | | |
| 탐색·초안·독립 검수를 분리해 메인 맥락을 보호해야 하는가 | | |
| 순서·상태·재시도·체크포인트를 코드/도구로 강제해야 하는가 | | |

## 선택

- **선택한 유형**: md형 / 분업형 / MCP형 (하나만)
- **선택 이유** (규칙 고정 / 역할 분리 / 순서 강제 중 하나 이상):
- **다음에 만들 산출물**:
- **지금은 하지 않을 것** (고르지 않은 두 가지에 대한 한 줄):

---

## 예시 (참고용)

```markdown
# 하네스 선택표

## 업무
- 업무 이름: 반복 문서 초안 작성
- 업무 목표: 입력 메모를 정해진 구조의 초안으로 바꾼다
- 현재 가장 큰 실패 원인: 매번 역할과 금지사항을 다시 설명한다

## 증거 표
| 질문 | 증거 | 점수 |
|---|---|---|
| Q1 | 매 요청마다 톤·금지어·출력 형식을 다시 설명한다 | 3 |
| Q2 | 한 세션에서 끝나므로 메인 피로는 작다 | 1 |
| Q3 | 단계 자체가 한 단계라 순서 사고가 없다 | 1 |

## 선택
- 선택한 유형: md형
- 선택 이유: 반복 규칙과 금지사항을 문서로 고정하는 것이 먼저다
- 다음에 만들 산출물: md 규칙서 v1 (한 장짜리 SKILL.md)
- 지금은 하지 않을 것: 분업형 역할 분리, MCP형 워크플로우 설계
```

---

## 제출 체크리스트

- [ ] 선택한 유형과 가장 강한 증거(점수 가장 높은 축)가 일치한다.
- [ ] md형 — 반복 규칙 고정 / 분업형 — 메인 세션 보호 / MCP형 — 순서·상태·재시도 강제 중 하나 이상의 기준으로 선택 이유를 적었다.
- [ ] MCP를 기본값으로 두지 않았다.
- [ ] 다음 산출물이 선택한 유형과 맞는다.
- [ ] "지금은 하지 않을 것"을 한 줄 적어 과한 하네스 설계를 피했다.
- [ ] 외부 사례, 검증되지 않은 수치, 마케팅성 정보를 넣지 않았다.
