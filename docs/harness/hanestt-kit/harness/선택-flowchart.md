# 선택 Flowchart — 3단계 질문

본인 업무 하나를 머릿속에 떠올리고, 다음 세 질문에 차례로 답한다.

## Q1. 규칙만 고정하면 충분한가?

반복 규칙, 역할, 금지사항, 출력 형식이 핵심 문제인가?

- **예** → **md형으로 시작.** md 규칙서 v1을 다음 산출물로 잡는다. (여기서 멈춰도 된다.)
- **아니오** → Q2로.

## Q2. 메인 맥락을 보호해야 하는가?

탐색·초안·비교·독립 검수가 한 세션에 쌓여 중요한 결정이 흐려지는가? 산출물 수명이 세션을 넘어야 하는가?

- **예** → **분업형으로 나눈다.** 역할 분리표 + handoff를 다음 산출물로 잡는다. Pass by Reference 패턴으로 sub-agent를 저자로 쓴다.
- **아니오** → Q3로.

## Q3. 순서·상태·재시도가 품질을 좌우하는가?

단계 순서, 상태, 재시도, 체크포인트를 사람이 매번 기억해야 하는가? 그 기억 부담이 *이미* 품질의 병목인가?

- **예** → **MCP형을 검토한다.** MCP 필요성 판단표를 다음 산출물로 잡는다.
- **아니오** → 요구와 기준을 다시 명확히 한다. 진짜 문제가 다른 곳에 있을 수 있다.

## 증거 기반 판정

각 질문에 "예"로 답할 때는 *증거*를 하나씩 적는다. 막연한 직관으로 분업형이나 MCP형을 고르지 않는다.

| 질문 | 본인 업무의 증거 | 점수 (1~3) |
|---|---|---|
| Q1 — 규칙 반복 | | |
| Q2 — 메인 피로 | | |
| Q3 — 실행 강제 | | |

점수가 가장 높은 축이 본인 업무의 통제 지점이다.

## "지금은 하지 않을 것"도 같이 정한다

선택을 정할 때 *고르지 않은 두 가지*도 한 줄로 적는다.

> MCP형을 고르지 않았다는 말은 부족하다는 뜻이 아니라, 현재 문제의 통제 지점이 다르다는 뜻이다.

이 한 줄이 과한 하네스 설계를 막는다.

## 같은 업무가 시간에 따라 옮겨갈 수 있다

오늘 md형으로 충분한 업무가 6개월 뒤에는 분업형이나 MCP형으로 올라갈 수 있다. **신호가 바뀔 때마다 선택표를 다시 본다.**

- 새로운 신호: 메인 답변이 자주 끊기거나 맥락이 부족해진다 → 분업형 검토.
- 새로운 신호: 단계 누락이나 순서 사고가 반복된다 → MCP형 검토.
