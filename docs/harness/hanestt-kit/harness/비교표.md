# md형 · 분업형 · MCP형 — 3축 비교표

## 한눈에

| 축 | md형 | 분업형 | MCP형 |
|---|---|---|---|
| 통제 지점 | 규칙 고정 | 맥락 보호 | 실행 강제 |
| 핵심 문제 | 같은 규칙을 매번 반복 설명한다 | 탐색·초안·검수가 메인 세션에 섞인다 | 순서·상태·재시도가 품질을 좌우한다 |
| 들어가는 곳 | 역할, 원칙, 금지사항, 출력 형식 | 메인/보조 세션, MCP/도구, handoff | 단계 그래프, 상태머신, 체크포인트 |
| 비용 | 가장 낮음 (문서 한 장) | 중간 (sub-agent 호출, handoff 설계) | 가장 높음 (코드, 도구 통합) |
| 무엇을 영속화하나 | 규칙 문서 | 산출물 파일 (Reference 패턴) | 워크플로우 상태 |
| 다음 산출물 | md 규칙서 v1 | 역할 분리표 + handoff | MCP 필요성 판단표 |

## 1) md형 — 규칙 고정

### 신호

- 매번 같은 역할/금지사항/출력 형식을 다시 설명하고 있다.
- 결과 형식이 일관되지 않는다.
- 자주 바뀌지 않는 규칙이 머릿속에만 있다.

### 들어가는 정보

역할, 원칙, 금지사항, 출력 형식, 예시 입출력, 도메인 용어.

### 한계

규칙을 길게 적어도 메인 세션 맥락 소모는 줄지 않는다. 탐색·초안·검수가 한 세션에 쌓이면 md만으로는 안 된다.

### 다음 산출물

`md 규칙서 v1` — 본인 업무용 한 장짜리 SKILL.md 또는 CLAUDE.md 섹션.

---

## 2) 분업형 — 맥락 보호

### 신호

- 메인 세션이 쉽게 지친다. 탐색·초안·비교·독립 검수가 한 대화에 쌓인다.
- 중요한 결정이 최근 대화에 끌려간다 (의도 vs 표면 어휘 이탈).
- 산출물이 세션 종료와 함께 사라진다.

### 들어가는 정보 (네 칸)

| 칸 | 책임 | 남길 정보 |
|---|---|---|
| 메인 세션 | 목표, 제약, 현재 상태, 최종 판단 | 결정 로그 + 다음 액션 |
| 보조 세션 | 탐색, 초안, 비교, 독립 검수 | sub가 작성한 파일 경로 |
| MCP/도구 | 반복 조회, 실행, 상태 확인 | (필요한 경우에만) |
| handoff | 세션 간 반환 형식 | 판단에 필요한 최소 정보 |

### 핵심 패턴

**Pass by Reference** — sub-agent를 저자로 쓰고, 메인은 인덱서/오케스트레이터가 된다. 상세는 `02-메인세션-보호가이드/` 참고.

### 한계

순서·상태·재시도를 사람이 매번 기억해야 한다면 분업형만으로는 안 된다.

### 다음 산출물

`역할 분리표` + `handoff 명세`.

---

## 3) MCP형 — 실행 강제

### 신호

- 단계 순서가 뒤섞이면 결과가 무너진다.
- 어디까지 통과했는지, 실패하면 어디로 돌아가는지 사람이 매번 기억해야 한다.
- 체크포인트와 재시도가 품질의 병목이다.

### 들어가는 정보

단계 그래프, 상태머신, 체크포인트, 재시도 정책, 외부 도구 호출 시퀀스.

### 핵심 관점

> "비결정적인 모델을 결정적인 실행으로 묶어내는 장치" — interview.md

규칙(md)이나 분업(역할 분리)으로는 강제할 수 없는 *실행 흐름*을 코드/도구 안쪽으로 넣는다.

### 한계

비용이 가장 높다. 규칙 반복이나 메인 맥락 소모가 진짜 문제인 업무에 MCP를 쓰면 과한 설계다.

### 다음 산출물

`MCP 필요성 판단표` — 아직 구현이 아니라, 순서·상태·재시도 강제의 *필요성*을 검증하는 한 장.

---

## 세 하네스의 관계 — 위계가 아니라 선택지

```
처음 (규칙 반복)
   ↓ md형 — 역할/금지/형식을 문서에 둔다

커짐 (탐색·초안·검수가 섞임)
   ↓ 분업형 — 메인은 결정만 잡고 나머지는 밖으로

반복 (단계·복귀점이 품질을 좌우)
   ↓ MCP형 — 순서/상태/재시도를 실행 흐름으로 강제
```

세 단계는 **순서가 아니다.** 본인 업무의 통제 지점이 어디인지에 따라 곧장 분업형 또는 MCP형으로 시작할 수도 있다. 다만 거의 모든 경우 md형을 먼저 점검하는 게 가장 작은 충분한 시작점이다.
