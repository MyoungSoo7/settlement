# 2002-1-3. Wonder · Reflect · Refine · Restate 네 스킬 만들기

## Practice Task and Prompt

- Task: `/wonder`, `/reflect`, `/refine`, `/restate` 네 개의 분리된 스킬을 만든다. 네 스킬은 같은 스크래치 파일을 통해 상태를 넘긴다.
- Prompt to paste into AI:

```text
Fast Campus 하네스 엔지니어링 실습 2002-1-3을 도와줘.

Curriculum mapping:
- Module: 2-1+. 네 단계 분해 (못 얻으면 / 무엇을 가정)
- Clip: [실습] Wonder · Reflect · Refine · Restate 네 스킬 만들기
- Key message: 질문이 하네스의 시작점이다
- Required output: .claude/skills/{wonder,reflect,refine,restate}/SKILL.md 네 개

Task:
아래 작성 템플릿대로 네 개의 SKILL.md를 만든다.
각 스킬은 4칸(역할 · 원칙 · 출력 형식 · 저장 위치)으로 구성한다.
"역할" 칸에는 소크라테스가 사용자에게 직접 던질 **질문 문장**을 그대로 적고,
스킬은 그 질문을 AskUserQuestion 도구로 사용자에게 묻는다 (모델이 본문에 자유 서술하지 않는다).

네 단계의 본질:
- Wonder  : 사용자가 꺼낸 단어/요청 안에 또 어떤 의미가 숨어 있는지 발산하듯 더 캐낸다.
- Reflect : 사용자와 모델이 떠올린 의미가 서로 다를 수 있다는 걸 비춰서 깨닫는다.
- Refine  : 서로 다른 의미를 같은 의미로 합쳐 shared meaning을 만든다.
- Restate : shared meaning을 토대로 사용자 goal을 한 줄로 다시 진술한다.

Boundaries:
- 네 스킬이 같은 공유 스크래치 파일 규약을 따른다 (경로와 섹션 이름이 일치).
- 사용자에게 묻는 질문은 모두 AskUserQuestion 도구로 던진다.
- 모르는 의미는 추측으로 채우지 않는다.

My input:
`.claude/scratch/socrates.md`
```

## 목적

다섯 단계 묻기를 네 개의 분리된 스킬로 나눈다. 한 덩어리 스킬은 흐름을 지키지만 단계별 단독 호출이 어렵다. 네 스킬을 분리하면 각자가 무엇을 위해 있는지가 분명해지고, 한 단계만 단독으로도 호출할 수 있다.

핵심은 두 가지다.

1. 각 스킬의 "역할" 칸은 기능 설명이 아니라, 소크라테스가 사용자에게 직접 던지는 **질문 문장 그 자체**가 들어간다.
2. 그 질문은 AskUserQuestion 도구를 통해 던진다. 모델이 본문에 자유 서술하는 게 아니라, 도구 호출로 사용자 답을 받는다.

## 권장 시간

스킬 1개당 약 5분, 네 개 합쳐 약 20분. 첫 번째(`/wonder`)를 워크드 예시로 만들고, 나머지 셋은 같은 패턴을 복제한다.

## 준비물

- 기존 한 덩어리 스킬: `../../../../.claude/skills/socrates/SKILL.md` (참고용)
- 슬라이드 14의 단계별 두 답 묶음 표 (본인이 머리에 담아 두는 가이드)
- 공유 스크래치 파일 경로와 섹션 규약 (예: `../../../../.claude/scratch/socrates.md`에 `## 사용자 입력`, `## Wonder`, `## Reflect`, `## Refine`, `## Restate` 다섯 섹션)

## 가이드 — 슬라이드 14의 단계별 두 답 묶음

이 표는 각 스킬을 만들 때 본인이 "왜 이 단계가 필요한지"를 다시 확인하는 용도다. SKILL.md 본문에는 넣지 않는다.

| 단계    | 못 얻으면 (효용)                              | 무엇을 가정 (전제)                          |
| ------- | --------------------------------------------- | ------------------------------------------- |
| Wonder  | 의미 후보가 안 나와 한 의미에 갇힌다          | 한 단어에 의미가 둘 이상 있다               |
| Reflect | 사용자와 모델의 의미 차이가 안 드러나 shared로 못 간다 | 사용자와 모델이 다른 의미를 떠올릴 수 있다 |
| Refine  | shared가 안 만들어져 같은 의미 합의가 없다    | 다른 의미끼리도 합치는 지점이 있다          |
| Restate | 같은 의미로 goal을 다시 진술하지 못한다       | shared가 형성되면 같은 의미로 더 또렷하게 진술할 수 있다 |

## 단계별 진행

1. **공유 스크래치 파일 규약 결정** — 네 스킬이 읽고 쓸 파일 경로와 섹션 이름을 먼저 정한다. 이 규약이 곧 출력 형식의 고정값이 되고, 다음 스킬의 입력이 된다.
2. **`/wonder`부터 시작** — 아래 워크드 예시를 그대로 베끼는 것부터 한다.
3. **역할 칸은 질문 문장으로** — "이 스킬은 ~을 한다"라고 설명하지 말고, 소크라테스가 사용자에게 던질 질문을 그대로 적는다. 모델은 그 문장을 AskUserQuestion 도구로 사용자에게 묻는다.
4. **출력 형식과 저장 위치 명시** — 답이 어떤 모양으로 떨어지는지, 그리고 스크래치 파일의 어느 섹션에 저장되는지 적는다.
5. **frontmatter 작성** — `description`은 "이 스킬이 언제 호출되어야 하는지"를 한 줄로 명시한다.
6. **나머지 셋(`/reflect`, `/refine`, `/restate`) 복제** — 같은 4칸 구조로 만든다. 각 스킬은 직전 스킬이 저장한 섹션을 입력으로 읽고, 자기 섹션을 출력으로 쓴다.
7. **검증** — 본인이 단어 하나(예: "성과", "리더십" 등 모호한 단어)를 골라 `/wonder` → `/reflect` → `/refine` → `/restate` 순서로 호출. 산출이 기존 `/socrates`와 형식이 일치하면 분리 성공.

## 작성 템플릿 — 한 스킬당 한 장

```markdown
---
name: {{wonder | reflect | refine | restate}}
description: {{이 스킬이 언제 호출되어야 하는지를 한 줄로 — 트리거 단어 포함}}
---

# {{Wonder | Reflect | Refine | Restate}}

## 역할 — 소크라테스의 질문
> "{{사용자에게 직접 던질 질문 문장}}"
> "{{같은 단계 안에서 한 겹 더 들어갈 때 던질 추가 질문}}"

이 질문은 모두 AskUserQuestion 도구로 사용자에게 묻는다.

## 원칙
1. 한 번에 한 질문만 던진다.
2. 사용자가 꺼낸 답을 평가하지 않는다.
3. 추측으로 답을 만들어 주지 않는다 — 모르면 다시 묻는다.

## 출력 형식
{{의미 후보 3개 리스트 / Y·N 확인 / 한 줄 정의 등}}

## 저장 위치
`.claude/scratch/socrates.md`의 `## {{단계명}}` 섹션. 조건이 충족되면 저장하고 종료, 다음은 `/{{다음 스킬}}`로 넘긴다.
```

## 예시 — `/wonder` 한 장 (워크드 예시)

```markdown
---
name: wonder
description: 사용자가 던진 단어/요청 안에 또 어떤 의미가 숨어 있는지 발산하듯 더 캐낸다. "wonder", "/wonder" 호출.
---

# Wonder

## 역할 — 소크라테스의 질문
> "방금 말씀하신 그건 또 어떤 의미를 가질 수 있을까요?"
> "그 안에 숨어 있는 다른 뜻은 없습니까?"
> "한 겹 더 들어가면 무엇이 보이나요?"

사용자가 의미 하나를 꺼내도 거기서 멈추지 않고, 그 단어가 본질적으로 무엇을 가리키는지 계속 발산하듯 더 묻는다. 의미 후보가 셋 이상 모일 때까지 반복한다.

질문은 모두 AskUserQuestion 도구로 사용자에게 던진다.

## 원칙
1. 한 번에 한 질문만 던진다.
2. 사용자가 꺼낸 의미를 평가하지 않는다 — 더 캐낸다.
3. 추측으로 의미를 만들어 주지 않는다. 사용자가 직접 꺼내야 한다.

## 출력 형식
의미 후보 3개 이상의 마크다운 리스트. 각 후보에 한 줄 설명.

- 의미 A — 한 줄 설명
- 의미 B — 한 줄 설명
- 의미 C — 한 줄 설명

## 저장 위치
`.claude/scratch/socrates.md`의 `## Wonder` 섹션. 의미가 셋 이상 모이면 저장하고 종료, 다음은 `/reflect`로 넘긴다.
```

## 완료 기준

- 네 개의 SKILL.md가 `../../../../.claude/skills/wonder`, `../../../../.claude/skills/reflect`, `../../../../.claude/skills/refine`, `../../../../.claude/skills/restate`에 존재한다.
- 각 SKILL.md는 4칸(역할 · 원칙 · 출력 형식 · 저장 위치)으로만 구성되어 있다.
- "역할" 칸에 기능 설명이 아닌 **질문 문장**이 들어 있고, 그 질문이 AskUserQuestion 도구로 사용자에게 던져진다.
- 네 스킬이 같은 공유 스크래치 파일 경로를 사용하고, 섹션 이름이 일치한다.
- 본인이 단어 하나를 골라 네 스킬을 순서대로 호출했을 때, 기존 `/socrates`와 형식이 일치하는 산출이 떨어진다.
- 네 스킬 중 하나만 단독 호출해도 바로 동작한다.
