# 사내 교육 신청 페이지 개편 회의록 (시운전용 샘플)

- 일시: 2026-05-22 (금) 14:00–14:50
- 참석: 박PM, 김디자이너, 이FE, 정BE, 한CS
- 목적: 신청 직후 환불·일정 변경 문의가 CS에 다수 유입 → 페이지 개편 방향 합의

## 발언 기록

- 박PM: "오늘은 방향만 합의한다. 최종 담당자·일정은 다음 주 월요일 정기 미팅에서 확정한다."
- 박PM: 목표는 "신청 직후 환불·일정 문의를 줄인다" 한 줄로 잡는다.
- 한CS: 지난 분기 신청 후 환불·일정 변경 문의가 다수 유입. "정확한 수치는 다시 뽑아 와야 한다"는 단서 — 한CS 추정 60% 수준이라고 했으나 출처 미확인.
- 김디자이너: "일정·환불 안내가 페이지 맨 아래 작은 글씨로 있어서 사실상 안 읽힌다." 신청 직전 강조 화면이 필요하다는 의견.
- 이FE: 자동 안내 메일에 환불·일정 기준을 추가하는 쪽이 비용이 가장 적다.
- 정BE: 결제 모듈 모달은 외부 SDK라 위치 이동이 불가하다. 이번 범위에서는 건드리지 않는다.
- 박PM: "페이지 전체 리뉴얼은 이번 범위가 아니다." 두 갈래(강조 화면 + 자동 메일)만 본다.
- 한CS: 환불 기준 자체 명문화 여부는 운영팀 별도 확인 사안. "환불 기준 자체가 명문화돼 있지 않다"는 현재 상태.
- 김디자이너: 시안은 다음 주 화요일까지 가져온다 (잠정).
- 정BE: 시안 확정 후 3일 안에 개발 일정 산정 (잠정).

## 합의된 범위

- (1) 신청 직전 핵심 정보 강조 화면 추가.
- (2) 자동 안내 메일에 환불·일정 기준 추가.

## 범위 밖

- 페이지 전체 리뉴얼.
- 결제 모듈 모달 위치 변경 (외부 SDK 제약).
- 환불 기준 자체 명문화 (운영팀 별도 확인 사안).

## 미결정

- 최종 담당자·일정 (월요일 정기 미팅에서 확정 예정).
- CS 문의 60% 수치 출처.
- 환불 기준 명문화 가능 여부 (운영팀 회신 대기).
- 시안·메일 템플릿 작업 일정의 최종 승인.
