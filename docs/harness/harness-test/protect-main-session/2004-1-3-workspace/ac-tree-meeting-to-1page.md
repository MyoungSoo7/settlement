## 4. 세 검사

- **Parent 합**: ask 2(L1·L2) + build 2(L3·L4) + review 2(L5·L6) → "회의록을 7칸 1page로 변환" 한 줄이 닫힘. **OK**
- **겹침**: L3은 6칸을, L4는 7번째 칸을 각각 채움. L5는 보존 표현 인용 가드, L6은 도메인 제약 가드 — 서로 다른 가드. **OK**
- **Orphan**: AC raw R1~R16이 모두 위 6개 리프 중 하나로 흘러들거나 "4장에 안 가져옴(R9·R10)"으로 분류됨. **OK**

## 5. 분업 가치 판정

- **분업 가치 있음 (sub agent로 분리)**
  - **L1 ask** — 회차별 의사결정자 답이 누적돼야 함(월요일 미팅·운영팀 회신 등 외부 회차). 1-shot 안에선 회차 추적이 안 됨. → 4장에서 분리 가치 큼.
  - **L5 review** — 1-shot 안에선 같은 세션의 관성 때문에 자기 결과를 자기가 통과시키는 경향. 보존 표현 인용 정확도는 별도 세션이 잡는 게 안전.
  - **L6 review** — 도메인 제약 4항목(외부 사실 단정 / 미확인 수치 / 범위 밖 / 미결정 결론 톤)이 서로 성질이 다른 가드라 한 세션이 동시에 못 잡는 경우가 있음. 분리 가치 있음.

- **분업 가치 낮음 (1-shot으로 충분 — 4장에 안 가져옴)**
  - **L3·L4 build** — 처리 순서가 짧고(6칸 + 1칸), 입력이 한 회의록으로 닫힘. 1-shot으로 충분. 단 회의록이 길거나 검수자가 별도 build 세션을 원하면 가져옴.
  - **L2 ask** — 인용 추출은 L3 build에 흡수해도 됨. 별도 세션을 쓸 만큼 분량이 크지 않음.
