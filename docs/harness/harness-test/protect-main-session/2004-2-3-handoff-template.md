# 2004-2-3. 세션 분리 규칙과 handoff 양식 만들기

## 이 산출물
모든 보조 세션(ask/build/review)이 메인 세션에 한 장으로 돌려주는 handoff 양식 v1이며, `2004-3-3` deterministic review gate와 exit rule 실습의 입력이 된다.

## 목적

세션이 나뉘면 결과 회수 양식이 미리 정해져 있어야 한다. 4세션이 모두 같은 양식으로 채워서 돌려주면 메인 세션은 같은 위치에서 다음 결정을 한다. 본 실습에서는 골격 한 장 + 본인 도메인 예시 한 장을 만든다.

## 권장 시간

25분

## 준비물

- 2004-1-3에서 만든 작업 트리에서 고른 리프 1개 (가능하면 build 세션 후보)
- 그 리프가 끝났다고 가정했을 때의 가상 결과 한 장 (표 또는 한 줄)
- 다음 세션 후보 (보통 review 세션)

## 입력

아래 입력을 먼저 채운다.

| 항목 | 내용 |
|---|---|
| 리프 이름 |  |
| 리프 (목적·입력·범위·금지·완료조건) |  |
| 보조 세션 종류 (ask/build/review) |  |
| 가상 결과 한 장 |  |
| 다음 세션 후보 |  |

## AI에게 시키기

위 입력표를 채워 그대로 붙인 뒤, 한 줄을 덧붙인다.

> 이 입력으로 handoff 양식 v1과 예시 한 장을 작성해줘. 아래 작성 템플릿의 모든 칸을 채우고, 결과 요약은 한 화면 분량으로 줄여. 각 주장에는 근거(출처) 칸에 파일·줄을 달아 — 4-3 2단 증거 검사가 이 줄을 직접 대조한다. 본문 대화를 옮기지 마.

## 단계별 진행

1. 양식 골격을 본인 md 파일에 한 번 적는다.
2. 리프 핸드오프 목적·입력·범위·금지·완료조건을 윗부분에 옮긴다.
3. 가상 결과를 결과 요약에 줄여 적고, 각 주장의 출처(파일·줄)를 근거 칸에 적는다.
4. 판정 (통과/재작업/재질문/종료)과 사유를 적는다.
5. 남은 질문과 다음 세션의 첫 행동 한 줄을 적는다.
6. 다음 세션 입장에서 양식만 읽고 작업을 시작할 수 있는지 본다.

## 작성 템플릿

````markdown
# Handoff — <리프 ID> · <세션 종류> → 메인

## From task (4-1 리프)
- 목적:
- 입력:
- 범위 (하지 않을 일 포함):
- 금지:
- 완료조건:

## Result (이 세션이 채운 부분)
- 결과 요약:
- 근거(출처):
- 판정: 통과 / 재작업 / 재질문 / 종료 중 하나
- 남은 질문:
- 다음 세션의 첫 행동:
````

## 예시

````markdown
# Handoff — 본문 채우기 · build 세션 → 메인

## From task
- 목적: 회의록 확정 내용으로 1page 본문을 채움
- 입력: 회의록 원문(sample-meeting.md) + ask 세션 결정 답 + 보존 인용 3줄
- 범위: "확인 필요 항목"을 뺀 본문 전체. 회의록에 없는 사실 추가 금지
- 금지: 미확인 수치·일정·담당자를 결론처럼 쓰기
- 완료조건: 제목·배경·문제·사용자·해결 방향·성공 기준이 한 줄 이상씩

## Result
- 결과 요약:
  | 칸 | 채운 내용 |
  |---|---|
  | 제목 | 사내 교육 신청 페이지 개편 v1 |
  | 문제 | 일정·환불 안내가 안 읽혀 신청 후 CS 문의 |
  | 성공 기준 | "신청 직후 환불·일정 문의를 줄인다" |
- 근거(출처):
  - 성공 기준  ← sample-meeting.md:10
  - 문제       ← sample-meeting.md:16
  - 범위 밖(전체 리뉴얼 제외) ← sample-meeting.md:15
- 판정: 통과 (자체 검수 기준)
- 남은 질문: 성공 기준의 측정 단위·기간은 미확정
- 다음 세션의 첫 행동: review 세션에 본문 초안 + 근거(출처)를 함께 보낸다
````

## 완료 기준

- 양식의 모든 칸이 채워져 있다.
- 판정이 통과/재작업/재질문/종료 중 한 단어로 정해져 있다.
- 결과 요약이 한 화면 안에 들어온다.
- 다음 세션의 첫 행동이 동사 + 목적어로 적혀 있다.

## 자기 검수 체크리스트

- [ ] handoff가 한 화면에 다 들어온다.
- [ ] "잘 됐다 / 안 됐다" 같은 모호한 판정 표현이 없다.
- [ ] 결과의 각 주장에 출처(파일·줄)가 근거 칸에 붙어 있다 — 4-3 2단 증거 검사가 이 줄을 직접 대조한다.
- [ ] 결과 요약에 보조 세션의 대화 흔적(메타 설명)이 남아 있지 않다.
- [ ] 다음 세션의 첫 행동이 "검토 부탁" 같은 모호 표현이 아니다.
- [ ] 같은 양식을 ask/build/review 세 세션이 함께 쓸 수 있다.
