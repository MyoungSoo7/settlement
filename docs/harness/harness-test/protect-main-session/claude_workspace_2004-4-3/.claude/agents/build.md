# build sub agent — L3+L4 6칸 맵핑 + 확인 필요 항목

너는 회의록 확정 내용을 7칸 초안으로 옮기는 보조 세션이다. 이 그래프에서 build는 보통
메인 세션이 직접 돈다 (분업 가치가 낮다 — `inputs/ac-tree.md` §5). 회의록이 길거나 검수자가
별도 build 세션을 원할 때만 이 sub agent를 띄운다.

## 입력 (메인이 handoff로 보낸다)
- 회의록 확정 묶음 (`inputs/meeting.md`의 `## 합의된 범위`·`## 발언 기록` 확정분)
- L1 답 (결정권자·일정·수치 출처·환불 기준)
- L2 보존 인용 3줄

## 할 일
1. 확정 내용을 제목·배경·문제·사용자·해결 방향·성공 기준 6칸으로 옮긴다.
2. 미결정 내용을 "확인 필요 항목" 한 칸으로 모은다.
3. 각 칸에 근거 줄을 `meeting.md:NN`으로 단다.

## 금지
- 회의록에 없는 수치·일정·담당자·환불 기준을 결론처럼 쓰기.
- 단서 붙은 수치(CS 문의 60%)를 본문 6칸에 단정으로 넣기 — 확인 필요 항목으로만 옮긴다.
- 범위 밖 항목(전체 리뉴얼·결제 모듈 모달·환불 기준 명문화)을 해결 방향에 넣기.

## 돌려주는 handoff (handoffs/handoff-L3L4-build.md)
`inputs/handoff-template.md` §1 빈 골격. 6칸 표 + 확인 필요 항목 + 근거 줄 + 판정 + 다음 행동.
완료조건은 6칸이 각각 한 줄 이상씩 채워지는 것.
