# 03. 문서형 하네스 한계 진단표 (2003-3-3 산출물)

> 입력: `02-2003-2-3-output.md`. md 템플릿이 어디까지 버티고 어디서 무너지는지 정리한다.

## 입력표

| 항목 | 내용 |
|---|---|
| 진단할 업무 | 사내 교육 신청 페이지 개편 회의록 → 1page 기획안 |
| 반복되는 규칙 | 회의록·확인된 요구사항만 사용 / 확정·미결정 분리 / 보존 표현 인용 |
| 반복되는 출력 포맷 | 제목·배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목 7칸 |
| 반드시 지켜야 할 순서 | 원문 보존 → 확정/미결정 분리 → 7칸으로 옮김 → 보존 표현 인용 검사 |
| 추적해야 할 중간 상태 | 확인 필요 항목에 대한 답변 회차(월요일 미팅 → 운영팀 회신 → CS 수치 회신) |
| 실패 후 재질문 또는 재시도 지점 | "반드시 보존할 표현" 누락 / 미결정 항목이 결론처럼 적힌 경우 |
| 문서형으로 충분하다고 보는 범위 | 반복 규칙, 7칸 출력 포맷, 금지사항, 짧은 예시 기준 |
| 다음 하네스가 필요할 수 있는 범위 | 확인 필요 항목 답변 추적, 보존 표현 누락 시 재시도, 운영팀·CS 등 외부 회신 단계의 순서 강제 |

# 문서형 하네스 한계 진단표

## 진단 대상
- 업무명: 사내 교육 신청 페이지 개편 회의록 → 1page 기획안
- 입력: 회의록 원문과 확인된 요구사항
- 출력: 제목·배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목이 있는 1page 초안
- 원하는 반복 결과: 입력 회의록이 달라도 같은 7칸으로 초안이 정리된다.

## 문서형으로 강한 부분
| 항목 | 내용 | md에 넣을 위치 | 이유 |
|---|---|---|---|
| 반복 규칙 | 회의록과 확인된 요구사항만 사용 | 처리 원칙 | 매번 같은 판단 기준이 필요하다 |
| 출력 포맷 | 제목·배경·문제·사용자·해결 방향·성공 기준·확인 필요 항목 | 출력 형식 | 사람이 검수하기 쉽다 |
| 금지사항 | 회의록에 없는 수치·일정·담당자·환불 기준 추가 금지 | 금지 | 추측을 막는다 |
| 예시 기준 | 짧은 항목형 + 보존 표현은 큰따옴표 인용 | 예시 | 톤과 길이를 맞춘다 |

## 문서형으로 약한 부분
| 항목 | 내용 | 문서만으로 약한 이유 | 다음 하네스 후보 |
|---|---|---|---|
| 순서 | 원문 보존 → 확정/미결정 분리 → 7칸 옮김 → 보존 표현 검사 | 문서는 순서를 안내할 수 있어도 실행을 강제하지 않는다 | 분업형 |
| 상태 | 확인 필요 항목 회차별 답변(월요일·운영팀·CS) | 답변이 여러 차례 오가면 상태 저장이 필요하다 | MCP형 |
| 재질문 | 결정권자·일정·수치 출처가 비어 있을 때 | 질문 후 답변을 다시 입력으로 넣는 흐름이 필요하다 | 분업형 |
| 재시도 | 보존 표현이 인용 부호 없이 의역된 경우 | 실패 감지와 재실행 기준이 문서 안에서 강제되지 않는다 | MCP형 |

## 결정
- 문서형으로 만들 범위: 반복 규칙, 7칸 출력 포맷, 금지사항, 보존 표현 인용 규칙
- 다음 하네스가 필요한 범위: 확인 필요 항목 답변 추적, 보존 표현 누락 시 재시도, 외부 회신 단계 순서
- 지금 단계에서 보류할 판단: 페이지 전체 리뉴얼 범위, 환불 기준 명문화

## 검수 점검
- [x] 반복 규칙과 출력 포맷은 문서형 범위에 있다.
- [x] 순서·상태·재시도는 문서 밖 실행 구조 후보로 검토했다.
- [x] 문서형으로 충분한 이유 또는 부족한 이유가 한 줄로 설명된다.
