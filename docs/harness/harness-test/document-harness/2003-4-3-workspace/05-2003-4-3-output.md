# 05. 프로젝트 1 산출물: 회의록 → 1page 기획안 하네스 워크스페이스 (2003-4-3)

> 입력: 1-3 체크리스트, 2-3 md 템플릿 v1, 3-3 한계 진단표, 4-1 작성 프랙티스, 4-2 설계도, 본인 회의록 1건. 결과: 본 워크스페이스 (`../../../../../.claude` + `examples`).

## 입력표

| 항목 | 내용 |
|---|---|
| 메타 스킬 이름 | `using-1page-harness` |
| 도메인 스킬 이름 | `meeting-to-1page` |
| 도메인 스킬 description (요지) | 회의록 → 1page 기획안 7칸 변환. 트리거: "회의록 정리", "1page 기획안", "회의록 → 1page", "사내 교육 신청 페이지 개편". |
| 회의록 원문 (시운전용) | `examples/sample-meeting.md` (사내 교육 신청 페이지 개편, 2026-05-22) |
| 반드시 보존할 표현 (3줄) | "신청 직후 환불·일정 문의를 줄인다" / "페이지 전체 리뉴얼은 이번 범위가 아니다" / "환불 기준 자체가 명문화돼 있지 않다" |
| 비어 있으면 질문할 항목 | 최종 담당자·일정 / CS 문의 60% 수치 출처 / 환불 기준 명문화 여부 / 시안·메일 작업 일정 최종 승인 |
| 출력 형식 (1page 칸) | 제목 · 배경 · 문제 · 사용자 또는 대상 · 해결 방향 · 성공 기준 · 확인 필요 항목 (7칸) |
| 사람이 판단할 영역 | 최종 일정·담당자 확정, 외부 사례 보강 여부, 환불 기준 명문화 |
| SessionStart hook 안내 규약 | 회의록·요구사항 밖 결론 금지 / 보존 표현 큰따옴표 인용 유지 / 미결정은 확인 필요 항목 분리 / 파일명 1page·one-page·2003-4-3 키워드 포함 / exit 2 시 위반 부분만 보정 |
| PostToolUse hook 검사 (3종) | ① 보존 표현 3줄 인용 ② 확인 필요 항목 섹션 존재 ③ "60%" 미확인 수치 단정 차단 |

## 폴더 구조

```

│   ├── settings.json                          ← SessionStart + PostToolUse(Write|Edit)
│   ├── hooks/

│   │   └── check-1page-output.sh              ← 산출물 가드 (검사 3종)
│   └── skills/
│       ├── using-1page-harness/SKILL.md       ← 메타 스킬 (협업 규약·재작성 양식)

├── examples/
│   └── sample-meeting.md                      ← 시운전용 회의록 (2026-05-22)

```

|---|---|---|
| `../../../../../AGENTS.md` / `../../../../../CLAUDE.md` 초기 지침 | SessionStart hook + 메타 스킬 본문 | 사전 안내 |
| `skills/<도메인>/SKILL.md` | `.claude/skills/meeting-to-1page/SKILL.md` | 도메인 작업 규칙 |

| `skills/using-superpowers/SKILL.md` 메타 | `.claude/skills/using-1page-harness/SKILL.md` | 협업 규약 |

## 작성 흐름 (실제 수행한 순서)

1. `../../../../../.claude` 폴더와 `examples` 폴더 정렬.
2. 메타 스킬 `using-1page-harness/SKILL.md` 작성 — 협업 규약·hook 동작·재작성 행동 양식·도메인 스킬 호출 순서만.
3. 도메인 스킬 `meeting-to-1page/SKILL.md` 작성 — 보존 표현 3줄·7칸 스키마·처리 순서·예시 입력/출력.
4. `session-start.sh` 작성 + 실행 권한.
5. `check-1page-output.sh` 작성 + 실행 권한.
6. `settings.json`에 SessionStart, PostToolUse(matcher: "Write|Edit") 두 hook 등록.
7. `examples/sample-meeting.md`로 시운전.
8. PostToolUse hook 통과 / 실패 시 위반 부분만 Edit으로 보정.

## 시운전 결과

- 입력: `examples/sample-meeting.md`.
- 출력: 1page 7칸 모두 채움. 보존 표현 3줄이 큰따옴표 인용으로 살아 있음.
- 미결정 4건은 모두 `## 확인 필요 항목` 섹션에만 모임. 결론 칸에 섞이지 않음.
- PostToolUse hook: 첫 Write에서 통과 (exit 0).
- 같은 회의록 재투입: 같은 칸 구성 그대로. deterministic 한 줄 시험 통과.
