# 1page 기획안 — 사내 교육 신청 페이지 개편 (2003-4-3 시운전 run2)

원천 회의록: `examples/sample-meeting.md` (2026-05-22 회의)

---

## 제목
사내 교육 신청 페이지 — "신청 직후 환불·일정 문의를 줄인다" v1

## 배경
- 지난 분기 신청 후 환불·일정 변경 문의가 CS로 다수 유입.
- 현재 일정·환불 안내는 페이지 맨 아래 작은 글씨로만 존재해 사실상 노출되지 않음.
- 2026-05-22 회의에서 방향만 합의, 최종 담당자·일정은 다음 주 월요일 정기 미팅에서 확정 예정.

## 문제
- "일정·환불 안내가 페이지 맨 아래 작은 글씨로 있어서 사실상 안 읽힌다" (김디자이너 발언).
- 신청 직전 핵심 정보가 노출되지 않아 신청 후 CS 환불·일정 문의로 이어짐.

## 사용자 또는 대상
- 사내 교육 신청자
- CS팀 (환불·일정 문의 1차 응대)

## 해결 방향
1. 신청 직전 핵심 정보(일정·환불 기준) 강조 화면 추가.
2. 자동 안내 메일에 환불·일정 기준 추가 (이FE: 비용이 가장 적은 갈래).

범위 밖으로 합의된 항목 (해결 방향에 포함하지 않음):
- "페이지 전체 리뉴얼은 이번 범위가 아니다" (박PM).
- 결제 모듈 모달 위치 변경 — 외부 SDK 제약으로 이번 범위 제외 (정BE).
- 환불 기준 자체 명문화 — 운영팀 별도 확인 사안.

## 성공 기준
- "신청 직후 환불·일정 문의를 줄인다" (박PM이 정한 목표 한 줄).

## 확인 필요 항목
- 최종 담당자·일정 — 다음 주 월요일 정기 미팅에서 확정 예정.
- CS 문의 비중 수치 출처 — 한CS 추정 60% 수준이라고 했으나 출처 미확인, 다시 뽑아 와야 함.
- 환불 기준 명문화 가능 여부 — 운영팀 회신 대기. 현재 상태는 "환불 기준 자체가 명문화돼 있지 않다".
- 김디자이너 시안 (잠정 다음 주 화요일) · 정BE 개발 일정 산정 (잠정 시안 확정 후 3일) — 최종 승인 전.
